Model 类综述 #

基于 11 篇论文综合。

1. 现状快照 #

2025–2026 年的模型设计正经历三重范式转型。MoE 架构主导化——1T 级 MoE 配合 MLA 压缩 KV cache 已成开源旗舰标配，稀疏度从 32 推到 48 [L2:2507.20534]，同一主干三代复用（K2→K2.5→K2.6）表明架构本身趋近饱和、杠杆转向训练与部署 [L2:kimi-k2-6]。全模态端到端——Thinker-Talker MoE 框架将文本/音频/视频统一在一条推理链路中，开源实现在 32/36 音频基准上取得 SOTA [L2:2509.17765]，而 DiT-centric 路线在音视频联合生成质量上全面领先 [L2:2604.14148]。注意力机制多元化——线性注意力首次在公平对比下全面超越全注意力 [L2:2510.26692]，input-dependent 深度聚合替代固定残差连接 [L2:2603.15031]，RoPE 低频维度的 massive values 被揭示为上下文理解的关键功能信号 [L2:2502.01563]。

2. 主线与分支 #

2.1 MoE 架构与极限稀疏 #

主线：以 Kimi K2 为代表的 1T 级 MoE 成为开源 agentic 模型的架构基底。K2 使用 384 routed experts top-8 + 1 shared expert，稀疏度 48 在 scaling law 下比稀疏度 8 节省 1.69× FLOPs [L2:2507.20534]。注意力层采用 MLA（Multi-head Latent Attention），每 token KV cache 仅 576 floats（kv_lora_rank=512 + qk_rope_head_dim=64），使得 1T 参数模型在 262K context 下仍可部署 [L2:2602.02276]。

K2.5 在不改动主干的前提下接入 MoonViT-3D 视觉塔，通过"早期融合 + 10:90 低视觉比例"联合预训约 15T tokens，证明了 1T 级 MoE + 多模态的可行性 [L2:2602.02276]。K2.6 进一步将工程重心完全放在 post-training 和部署上：默认以 INT4 compressed-tensors 发布（仅量化 routed experts，attention/shared/head 保留 FP16），将部署门槛从多节点拉到单 8×H200 节点 [L2:kimi-k2-6]。长程 agentic RL 把单次 rollout horizon 推到 4000+ 工具调用 / 12–13 小时连续执行，Toolathlon +80%、MCPMark +89% [L2:kimi-k2-6]。

Qwen3-Omni 的 Thinker 也采用 MoE 设计（30B total / 3B active，~10:1 sparsity），但选择 GQA 而非 MLA，在多模态语境下更倾向于用 GQA 来压缩 KV [L2:2509.17765]。其 MoE 的具体 expert 数量、routing 机制和 auxiliary loss 均未披露 [L2:2509.17765]，与 K2 系列的全面开放形成对比。

分支决策：K2 系列选择 MLA + sigmoid noaux_tc 路由 + 384 experts；Qwen3-Omni 选择 GQA + 未公开 MoE 配置。两者在不同规模和任务类型上各有取舍——MLA 的 KV cache 压缩更激进（576 floats/token），但 kernel 实现复杂度更高 [L2:2507.20534]。

2.2 注意力机制革新 #

线性注意力突破：Kimi Linear 的 KDA（Kimi Delta Attention）把 Gated DeltaNet 的 scalar forget gate 替换为 per-channel diagonal gate，并通过 DPLR 约束（a=b=k）使 chunkwise kernel 比通用 DPLR 快 ~2× [L2:2510.26692]。层级混合设计中每 3 层 KDA 后插 1 层 MLA（NoPE），实测 3:1 比例最优；纯 MLA（0:1）最差，过疏（15:1）退化 [L2:2510.26692]。MLA 层采用 NoPE（无位置编码），位置信息完全由 KDA 层的 data-dependent gate 和 short convolution 提供，使 context 拉到 1M 而无 RoPE 外推问题 [L2:2510.26692]。推理侧 decode 吞吐提升 +75%，KV cache 减少 ~75% [L2:2510.26692]。

深度残差革新：Attention Residuals 把标准 PreNorm 的"恒定权重 1 残差累加"替换为"沿深度做 softmax-attention 加权求和"——每层学习一个伪 query 对所有先前层输出做选择性聚合 [L2:2603.15031]。Block-AttnRes 版本训练 overhead < 4%，在 Kimi Linear 48B-A3B 上验证 GPQA +7.5，scaling law 显示相同 loss 下节省 ~1.25× compute [L2:2603.15031]。DenseFormer 的固定标量聚合被证明无效（val loss 1.767 vs baseline 1.766），input-dependent 的 softmax 选择是关键 [L2:2603.15031]。

RoPE 机理揭示：RoPE 位置编码导致 Q/K 低频维度出现 concentrated massive values，这些值对上下文知识理解至关重要——破坏后 GSM8K 从 76.9% 暴跌至 4.0%，而参数知识任务仅温和下降 [L2:2502.01563]。这一发现直接指导量化策略：AWQ/SmoothQuant（保护 massive values）在上下文任务上优于 GPTQ（不保护）[L2:2502.01563]。所有 RoPE 模型均存在此现象，非 RoPE 模型（GPT-2, OPT, Jamba）均不存在 [L2:2502.01563]。

2.3 全模态端到端模型 #

Thinker-Talker 框架演进：Qwen2.5-Omni 是首个开源 omni-modal 端到端模型，采用 dense Thinker（7B）+ dense Talker（0.6B）+ 单 codebook + DiT vocoder 架构 [L2:2503.20215]。Qwen3-Omni 对其进行系统性五轴重构 [L3:2509.17765]：Thinker/Talker 均升级为 MoE；音频编码器从 Whisper-large-v3 初始化改为 AuT 从头训练（20M 小时监督数据、动态 1–8s 窗口、12.5 Hz token rate）[L2:2509.17765]；语音编解码从单 codebook 升级到层级 RVQ；vocoder 从 DiT 改为 Causal ConvNet 使首包延迟从 ~640ms 降至 234ms [L2:2503.20215]；Talker 从紧耦合（hidden state passing）改为解耦（discrete tokens + multimodal features），使 RAG/safety/FC 能介入 text→speech 路径 [L2:2509.17765]。

"无退化"训练策略：Qwen3-Omni 的 S1 阶段"adapter-first / encoder-second"两步分训 + S2 阶段从一开始就混入跨模态数据，是"同 FLOPs 下 omni-base 在 text/vision 均 ≥ 单模态 base"的关键 [L2:2509.17765]。但其受控消融显示 MVBench 从 71.87 降至 69.50（−2.4），作者未讨论该退化 [L3:2509.17765]，使"零退化"的叙事存在实证漏洞。

2.4 视频与音视频生成 #

DiT + GRPO 路线：LongCat-Video（13.6B DiT）将所有视频生成任务统一为 video continuation（T2V=0 条件帧，I2V=1 帧，VC=多帧），配合 Block Causal Attention + KV caching 实现分钟级长视频 [L2:2510.22200]。训练创新包括为 Flow Matching 设计改进的 GRPO（固定 SDE 时间步解决 credit assignment、loss 重加权消除梯度消失、multi-reward 防 hacking），推理侧 coarse-to-fine（480p→720p 实现 10.6× 加速且质量提升）+ 3D Block Sparse Attention（93.75% sparsity，near-lossless）[L2:2510.22200]。VBench 2.0 Commonsense 维度 70.94% 全场最佳（含闭源）[L2:2510.22200]。

原生音视频联合生成：Seedance 2.0 采用 dual-branch MMDiT 架构，视频和音频各有一支 DiT branch 通过 cross-modal joint module 在每步去噪时深度交互 [L2:2604.14148]。T2V 所有维度全面领先（Motion Quality 3.75 vs runner-up 3.10），音频维度是最大护城河（AV Sync 3.75 vs runner-up 2.91），Arena.AI 公开盲测 Elo 1450 以 720p 击败 1080p 对手 [L2:2604.14148]。20/22 多模态任务支持，其中 7 个独家 [L2:2604.14148]。但其架构/训练完全不披露，可复现性为零 [L2:2604.14148]。

两者同属 RL-on-generation 范式——LongCat-Video 用多 reward GRPO，Seedance 用 DanceGRPO/RewardDance——但 LongCat 的理论洞察（GRPO as stochastic noise search）比 Seedance 的产品导向更具方法论贡献 [L3:2509.17765]。

2.5 训练方法论创新 #

MuonClip：解决 Muon 优化器在大规模 MoE 训练中的 attention logit 爆炸问题。Muon 的 msign 操作产生 equal singular values 的权重更新，导致 Q/K 的 spectral norm 累加增长被 bilinear form 平方放大。QK-clip 在 max logit 超过阈值 τ 时按 √γ rescale Q/K 投影权重，且是 self-deactivating 的——K2 训练中仅 12.7% heads 在前 70k 步触发，30% 训练后完全自然衰减 [L2:2507.20534]。

长程 agentic RL：K2.6 将 RL rollout horizon 从分钟级推到 4–13 小时（4000+ 工具调用），reward 依赖最终性能指标而非单次对错 [L2:kimi-k2-6]。核心壁垒在于 trajectory-level reward shaping + 超长 rollout 的基础设施——单条 rollout 占 GPU 数小时，需要分层 judge 或外部可执行 oracle [L2:kimi-k2-6]。

GSPO 与 GRPO 变体：Qwen3-Omni Thinker 后训练使用 GSPO（rule-based + model-based rewards + teacher→student KL 对齐）[L2:2509.17765]；LongCat-Video 为 Flow Matching 量身定制 GRPO（固定 SDE 时间步 + policy/KL loss 重加权 + max group std + multi-reward 加权）[L2:2510.22200]。两者都验证了 RL 后训练在各自领域的有效性。

3. 核心 trade-off 轴 #

轴 1: KV cache 压缩 vs 表达力 #

MLA 将 KV cache 压到 576 floats/token（K2 系列），GQA 下 KV cache 尺寸取决于 KV heads 数（Qwen3-Omni Thinker），KDA 层则为固定 128×128 state 不随序列增长（Kimi Linear）[L2:2507.20534] [L2:2509.17765] [L2:2510.26692]。Kimi Linear 的混合方案（20 KDA + 7 MLA 层）在长 context decode 吞吐 +75% 的同时保持质量匹配全注意力 baseline [L2:2510.26692]。三种方案构成一个连续谱：固定 state（KDA）→ 潜变量压缩（MLA）→ 标准 GQA/MHA，越靠左 decode 越快但实现越复杂。

轴 2: MoE 稀疏度 vs 通信开销 #

K2 的 sparsity 48（384/8）比 sparsity 8 节省 1.69× FLOPs，但带来更多 all-to-all communication cost in expert parallelism [L2:2507.20534]。Qwen3-Omni 的具体稀疏度虽未公开，但 10:1 sparsity ratio 暗示更温和的设计，降低 EP 通信但需更多 active FLOPs [L2:2509.17765]。K2.6 通过 INT4 routed-only 量化（~258 GB 总权重）把 1T MoE 从 16+ GPU 拉到单 8 卡节点，从部署侧缓解高稀疏度的内存代价 [L2:kimi-k2-6]。

轴 3: 首包延迟 vs 语音质量 #

Qwen2.5-Omni 的 DiT vocoder 需 4-block 窗口等待 lookahead，首包 ~640ms+ [L2:2503.20215]；Qwen3-Omni 切换到 Causal ConvNet 降至 234ms [L2:2509.17765]。但 234ms 仅在 1-concurrency 下成立，6-concurrency 下增至 1172ms（5×），论文的"延迟基本不受影响"声明与数据不符 [L3:2509.17765]。

轴 4: 量化精度 vs 部署门槛 #

K2.6 的 INT4 routed-only 量化把 1T MoE 部署门槛从多节点拉到单节点，但未给出 BF16 vs INT4 在长程 4000 步 rollout 上的消融 [L2:kimi-k2-6]。Massive values 研究表明，保护 Q/K 低频维度的量化方法（AWQ, SmoothQuant）在上下文理解任务上显著优于不保护的方法（GPTQ）[L2:2502.01563]——这对 K2.6 的 routed-only 策略是隐含风险：虽然 attention 权重保留了 FP16，但 MoE expert 的 INT4 量化误差是否在 4000+ 步 agentic rollout 中累积仍未可知 [L2:kimi-k2-6]。

轴 5: 多模态联合训练 vs 单模态退化 #

Qwen3-Omni 通过"S1 adapter-first / encoder-second + S2 早期多模态混入"实现同 FLOPs 下 omni ≥ 单模态 [L2:2509.17765]，但受控消融中 MVBench 仍退化 2.4 分 [L3:2509.17765]。K2.5 反直觉地发现 10:90 低视觉比例优于常规 50:50，且 zero-vision SFT 优于 text+vision SFT [L2:2602.02276]。两者结论均仅在特定规模上验证，7B 或 100B+ 的可迁移性未知。

轴 6: LLM-centric vs DiT-centric 多模态 #

Qwen3-Omni 以 LLM 为中心（Thinker MoE 统一理解+生成），在跨模态推理任务上优势明显（WorldSense 54.0, DailyOmni 75.8 均 SOTA）[L2:2509.17765]。Seedance 2.0 以 MMDiT 为中心（dual-branch 视频+音频联合去噪），在生成质量和 AV 同步上全面领先 [L2:2604.14148]。这是不同生态位的选择而非简单优劣——LLM 路线擅长理解+推理，DiT 路线擅长高质量生成 [L3:2509.17765]。

4. 冲突与争议 #

位置编码之争：K2 系列沿用 RoPE + YaRN 扩展到 262K context [L2:2602.02276]；Kimi Linear 则完全放弃 RoPE，在 MLA 层用 NoPE，位置信息完全靠 KDA 的 data-dependent gate 提供，context 拉到 1M [L2:2510.26692]。两个方案来自同一团队（Moonshot），暗示 RoPE 并非唯一选项。但 K2.6 仍沿用 RoPE [L2:kimi-k2-6]，说明 NoPE 路线尚未完全取代 RoPE——在工具调用和 agentic 场景下，RoPE 的成熟生态和推理框架兼容性可能是更优先的工程约束。

线性 vs 全注意力：Kimi Linear 在 1.4T 公平对比下首次全面超越全注意力 [L2:2510.26692]，但论文承认 pure linear attention 仍不可行——需要 MLA 层兜底 [L2:2510.26692]。Massive values 研究揭示 RoPE 低频维度在全注意力模型中承载核心语义功能 [L2:2502.01563]，这与 KDA 的"不需要 RoPE"哲学形成张力：如果语义信号被绑在 RoPE 的低频结构中，去掉 RoPE 的 KDA 方案用什么等价机制补偿？KDA 的 data-dependent gate 可能隐式承担了此角色，但缺乏对应的机理分析。

Thinker-Talker 耦合 vs 解耦：Qwen2.5-Omni 的紧耦合设计（Thinker hidden + sampled token embed 直接传给 Talker）阻止了外部模块介入 [L2:2503.20215]；Qwen3-Omni 改为解耦（discrete tokens + multimodal features），性能反而提升 [L2:2509.17765]。跨篇分析指出这理论上允许 Thinker 热替换（如接入 K2 级 agentic 模型），但缺乏任何实验验证 [L3:2509.17765]。

视频生成路径分歧：LongCat-Video 走"统一 video continuation + 多 reward GRPO 后训练"的开源路线 [L2:2510.22200]；Seedance 2.0 走"闭源 dual-branch MMDiT + 字节内部数据生态 + RewardDance/DanceGRPO"的产品路线 [L2:2604.14148]。两者都验证了 GRPO/RL 在视频生成后训练中的有效性，但 Seedance 2.0 的架构和训练细节完全不披露 [L2:2604.14148]，而 LongCat 开源了 3D Block Sparse Attention 的 Triton 实现 + ring attention 支持 [L2:2510.22200]。开源生态中，LongCat 的 Commonsense 70.94% 已可与闭源竞争 [L2:2510.22200]，但 Human Fidelity 80.20% 是测试模型中最低的 [L2:2510.22200]。

"非退化"叙事的可靠性：Qwen3-Omni 将"多模态训练不损害单模态性能"作为核心卖点，但自身消融中 MVBench 退化 2.4 分未被讨论 [L3:2509.17765]。此外，Thinking 变体在 ASR 任务上明显退化（Librispeech clean WER 1.22 → 2.22），说明链式思考推理会破坏感知任务 [L2:2509.17765]——"全能模型"的叙事需要加显著条件限定。

5. 未探索角落 #

  • KDA 在 100B+ 规模的验证：Kimi Linear 仅在 48B 上验证，大规模 dense / 更大 MoE 模型是否仍能保持质量优势未知 [L2:2510.26692]
  • Attention Residuals 在标准 dense 大模型上的验证：目前仅有 MoE Kimi Linear 一例 production 验证，70B+ dense 模型未经检验 [L2:2603.15031]
  • 混合线性注意力 × 多模态 Thinker：将 KDA 的 fixed-size recurrent state 引入 Qwen3-Omni 的 Thinker 层可大幅降低长音频/视频场景的 KV cache 压力，但 TM-RoPE 的时间对齐机制是否兼容 KDA 的 NoPE 设计是核心挑战 [L3:2509.17765]
  • "无退化"在其他规模的可复现性：Qwen3-Omni 的多模态无退化仅在 30B-A3B 验证，建立形式化的多模态 co-training 理论（何种数据分布和混合比例下模态间互信息增加而不损失单模态性能）仍是 open question [L3:2509.17765]
  • Massive values 的主动利用：目前仅用于量化策略指导，如何在训练中主动塑造 massive value 分布以提升特定能力、非 RoPE 位置编码是否有等价功能信号、可否设计 massive-value-aware 注意力变体均未探索 [L2:2502.01563]
  • INT4 量化对长程 agentic rollout 的累积影响：K2.6 未给出 BF16 vs INT4 在 4000 步 rollout 上的消融，量化误差是否在超长 tool-use 链中累积是实用性关键 [L2:kimi-k2-6]
  • 多 reward GRPO 用于语音生成后训练：LongCat-Video 证明多 reward GRPO 比单一 reward 更稳定，将类似设计（WER + 韵律 + 情感 + 多语言一致性）用于 Talker 后训练可能优于 Qwen3-Omni 当前的 DPO 链路 [L3:2509.17765]
  • Thinker 可替换性的实证验证：Qwen3-Omni 的 Talker 解耦设计理论上允许替换 Thinker，但实际中隐式的 feature distribution 依赖可能导致热替换失效 [L3:2509.17765]
  • 视频生成模型的 weight quantization：LongCat-Video 13.6B 模型未探索 INT8/INT4 量化以降低部署门槛 [L2:2510.22200]
  • GQA/MQA 等 Q/K 共享机制如何与 massive values 交互 [L2:2502.01563]

6. 时代定位 #

Model 类在 2025–2026 处于"架构饱和 + 训练/部署创新主导"的转折点。K2 系列三次迭代（K2→K2.5→K2.6）全部复用同一 backbone，所有创新放在训练管线（MuonClip、长程 RL、PARL agent swarm）和部署优化（INT4、agent orchestration）上 [L2:kimi-k2-6]。K2.6 的长程 agentic RL（单次 4000+ 工具调用、12–13 小时自主执行）把 RL 的 trajectory horizon 拉到前所未见的水平，核心洞察是"当底模架构达到够用的饱和点，下一步杠杆是把 RL rollout horizon 从分钟级推到小时级" [L2:kimi-k2-6]。

但并行地，Kimi Linear 和 Attention Residuals 代表了架构层面的持续探索——线性注意力 + NoPE 的组合 [L2:2510.26692] 和 input-dependent 深度聚合 [L2:2603.15031] 可能成为下一代架构的标配组件，为最终的 K3 级架构 redesign 储备技术选项。

多模态侧存在两条根本不同的技术路线。Qwen3-Omni 代表的 LLM-centric 路线以 Thinker MoE 为中心，覆盖感知+理解+生成完整链路（119 种文本语言 + 19 种语音输入 + 10 种语音输出），在跨模态推理上占优 [L2:2509.17765]。Seedance 2.0 代表的 DiT-centric 路线以 MMDiT 为中心，在生成质量和 AV 同步上全面碾压但不做感知和理解 [L2:2604.14148]。两条路线并非互斥——Seedance 证明"视频/音频联合生成当前最佳路径是 dedicated MMDiT，而非扩 LLM 上下文吃所有模态" [L2:2604.14148]，而 Qwen3-Omni 证明 LLM 路线在理解维度不可替代 [L3:2509.17765]。

视频生成侧，竞争已从"能否生成"转向"多模态任务覆盖广度 + 细粒度评测维度均衡度" [L2:2604.14148]。开源模型（LongCat-Video）在 Commonsense 维度上已可与闭源竞争 [L2:2510.22200]，但全维度均衡仍由闭源产品把持。

与相邻 category 的交叉：model 的 MoE routing、MLA/KDA kernel 设计直接影响 kernel 层优化空间；MuonClip、GRPO 变体和 Attention Residuals 是 algorithm 层贡献；Thinker-Talker 解耦、Agent Swarm 编排和 Claw Groups 跨入 agent 层；INT4 量化和 KV cache 压缩直接决定 cluster 层的部署密度。

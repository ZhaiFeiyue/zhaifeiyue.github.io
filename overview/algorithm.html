Algorithm 类别全景：长上下文与统一多模态 #

1. 现状快照 #

Algorithm 类别当前被两条主线主导。第一条主线（6/7 篇）围绕长上下文 LLM 的位置偏置诊断与修复展开：从 2024 年的 lost-in-the-middle 数据修复方案 [L2:2404.16811]，到 2025–2026 年逐步深入的多跳位置偏置 [L2:2412.10079]、隐式关联召回失败 [L2:2502.05167]、解码空间的后验显著度衰减 [L2:2506.08371]、单针 LITM 在前沿模型上的消除 [L2:2511.05850]、以及通用临界点量化 [L2:2601.15300]。第二条主线（1/7 篇）聚焦统一多模态视频理解-生成-编辑架构 [L2:2510.08377]。长上下文算法已从"能否支持 128K"进入"在 128K 里到底能用多少"的精细审计阶段；统一视频算法则处于范式建立期。

2. 主线与分支 #

2.1 长上下文位置偏置——诊断与评测 #

这是本类别最密集的子簇，4 篇工作从不同维度刻画了 LLM 在长上下文中的信息利用失败：

Baker et al. (2412.10079) 把 Liu et al. 2024 的单跳 LITM U 形曲线推广到多跳 QA，发现第二层退化变量——多个证据片段之间的相对距离（lost-in-between）。实验在 GPT-3.5 / MPT-7B / Llama-2-longlora 上证明 adjacent 配置稳定优于 separated 配置 1–7 pp，且 CoT 在非指令微调模型上反而导致准确率崩溃至 ≤1% [L2:2412.10079]。

Modarressi et al. (2502.05167, NoLiMa) 揭示了所有主流长上下文 benchmark 的致命缺陷——问题与 needle 之间的 ROUGE-L precision 高达 0.5–0.96，模型只需靠字面匹配而非语义理解即可得高分。NoLiMa 将 ROUGE-L 压到 0.07，迫使模型依赖隐式关联推理，结果 13/13 个前沿 LLM 的有效长度全部 ≤8K（声称 128K–1M 的模型实际有效长度缩水 16–128 倍）[L2:2502.05167]。

McKinnon (2511.05850) 从对立面证明，对于最简单的单针事实检索任务，Gemini 2.5 Flash 在 1M token 窗口（92% 填充率）内已实现 100% 准确率——LITM 的 U 形曲线在该子任务上完全消失 [L2:2511.05850]。但该结论严格限于单针 + 事实查询 + 单一模型，多针/多跳场景仍有退化。

Wang et al. (2601.15300) 提出 Natural Length Distribution Analysis，用 5 种独立检测器交叉验证，量化了 Qwen2.5-7B 在 128K 上下文下出现悬崖式智能衰退的临界点 Lc ≈ 43.2%（≈55K tokens），F1 骤降 45.5%。三瓶颈理论框架（RoPE 周期性 / attention 弥散 / 信息瓶颈）将 cliff 归因于 attention 弥散比 RoPE aliasing 更早触发 [L2:2601.15300]。

2.2 长上下文位置偏置——数据驱动修复 #

An et al. (2404.16811, FilM-7B / IN2) 提出最干净的数据驱动方案：通过 GPT-4 合成 1.75M 条 QA 数据，强制目标信息段在 4K–32K 上下文中均匀分布到任意位置（reject sampling），仅靠 instruction tuning 就将 Mistral-7B 的 VaL Probing 从 47.3 提升到 85.9（超过 GPT-4-Turbo 的 79.0），min-max gap 从 56.2 压到 13.9 [L2:2404.16811]。核心论点是：lost-in-the-middle 是数据问题，不是架构问题——不改注意力、不改位置编码即可修复 [L2:2404.16811]。

2.3 长上下文位置偏置——推理时解码修复 #

Xiao et al. (2506.08371, PCD) 从解码空间切入，发现长上下文下 gold token 的后验排名退化到 3–6 位但始终在 top-8 内（Posterior Salience Attenuation），提出 Positional Contrastive Decoding：对 RoPE 低频分量做过旋转得到 local-aware logits，与标准 logits 做对比解码，不需要任何训练即可把 InfiniteBench KV Retrieval 8K 段从 72% 提到 79%。理论证明 contrastive score 衰减率被压缩到 $(\ln B'/\ln B)^{2/d}<1$ [L2:2506.08371]。

2.4 统一多模态视频算法 #

Wei et al. (2510.08377, UniVideo) 提出双流架构（冻结 MLLM + MMDiT），通过 MLP 连接器 + self-attention 对齐，首次将视频理解、生成和编辑统一到单一多模态指令框架中。三阶段训练（连接器对齐 → 微调 → 多任务）仅需 35K 步，in-context 生成的主体一致性 SC=0.88 大幅超越 Kling1.6（0.68）和 Pika2.2（0.45），且展现出零样本泛化到未见编辑任务的能力 [L2:2510.08377]。

3. 核心 Trade-off 轴 #

轴 1：位置鲁棒性 vs 绝对准确率 #

文档预处理（摘要化 / KG 三元组抽取）能压平多跳 QA 的位置曲线，但绝对准确率下跌 1/3–1/2——GPT-3.5 HotpotQA 从 73.9% 掉到 KG 54.3% / Summary 59.5% [L2:2412.10079]。这个 trade-off 在弱模型上反转：Llama-longlora 反而从 KG 裁剪中受益 [L2:2412.10079]。

轴 2：训练成本 vs 推理成本 vs 效果 #

三条长上下文修复路径的成本结构截然不同：

  • 训练侧（IN2）：GPT-4 调用 \$50K–100K + 128×A100×56h ≈ \$14K 训练，一次性投入后推理无额外开销，VaL Avg +38.6 [L2:2404.16811]；
  • 推理侧（PCD）：零训练成本，但每 token 需双 forward（throughput 减半），InfiniteBench +7% / LongBench 仅 +0.89 [L2:2506.08371]；
  • RAG / prompt engineering：Segment Reranking 在 LongBench 上崩盘至 12.18（vs base 25.98），说明 prompt 重排不可靠 [L2:2506.08371]。

轴 3：声称上下文长度 vs 有效上下文长度 #

NoLiMa 揭示的核心矛盾——GPT-4o 声称 128K、有效仅 8K；Llama 3.3 70B 声称 128K、有效仅 2K；Gemini 1.5 Pro 声称 2M、有效仅 2K [L2:2502.05167]。而 Gemini 2.5 Flash 在单针任务上已完全消除这个差距 [L2:2511.05850]；Qwen2.5-7B 的 cliff 出现在 ~40% 标称长度处 [L2:2601.15300]。差距大小取决于任务难度层级：字面匹配检索 → 隐式关联推理 → 多跳推理，每升一级有效长度可能缩水一个数量级。

轴 4：理解能力保持 vs 生成能力获取 #

UniVideo 通过冻结 MLLM 避免了生成训练损害理解能力（MMBench 83.5 保持竞争力），但代价是无法做端到端联合优化 [L2:2510.08377]。FilM-7B 的 IN2 训练在常识推理上有显著退化（HellaSwag -4.5 / ARC-C -3.4），OpenOrca 11% anchor 未能完全兜住 catastrophic forgetting [L2:2404.16811]。

4. 冲突与争议 #

争议 1：LITM 已经消失了吗？ #

McKinnon 2511.05850 证明 Gemini 2.5 Flash 在单针事实检索上 LITM 完全消失 [L2:2511.05850]，但 NoLiMa 2502.05167 证明即使是同期最强的 GPT-4o，在去除字面匹配后有效长度也仅 8K [L2:2502.05167]；Wang 2601.15300 进一步量化了 Qwen2.5-7B 的 cliff 在 43% 标称长度 [L2:2601.15300]。结论：LITM 在最简单的子任务上确实被训练课程消除，但在需要隐式关联推理或多跳推理的任务上依然严重——"消失"的边界取决于任务对字面匹配的依赖程度。

争议 2：长上下文失败的根因是架构还是数据？ #

FilM-7B (2404.16811) 的核心论点是数据问题——不改架构仅靠合成数据即可修复 LITM [L2:2404.16811]。但 NoLiMa (2502.05167) 的 Last-2K 对齐实验证明固定 RoPE 相对距离后性能仍随上下文长度下降——指向 attention SNR 而非 position encoding decay [L2:2502.05167]。PCD (2506.08371) 则从频率分工角度论证 RoPE 低频分量的 attention 衰减是可被解码侧修复的 [L2:2506.08371]。未解决：三种归因（数据偏置 / attention SNR / RoPE 频率衰减）可能是同一问题在不同抽象层的投影，但目前没有工作统一验证。

争议 3：Cliff 还是 Slope？ #

Wang 2601.15300 声称 Qwen2.5-7B 存在"悬崖式"智能衰退（F1 在 10% ratio 窗口内骤降 45.5%），但从 Figure 1 的实际趋势线来看更接近"缓降 + 局部 dip"，且 5 个检测器中 4 个共享 3-stage filter、Definition 3 的 30% 阈值是任意定的 [L2:2601.15300]。这一方法论争议直接影响"40% 法则"的适用性。

5. 未探索角落 #

以下方向在现有 7 篇工作中被识别但未被任何一篇实质推进：

  • 多针 NoLiMa 扩展：当前 NoLiMa 是单 needle 设计，真实 RAG 常返回多个低相关 chunk + 1 个高相关 chunk，多针设定下有效长度可能被进一步压低 [L2:2502.05167]。
  • 64K–1M 的 IN2 外推：FilM 仅在 32K 内验证，能否外推到更长上下文是最大的未验证空间 [L2:2404.16811]。
  • PCD 与长上下文训练的叠加性：先经过 LongRoPE/ProLong 训练再叠 PCD 解码，收益是否可叠加还是被训练侧吸收 [L2:2506.08371]？
  • 非 RoPE 架构的 cliff 行为：Wang 2601.15300 的三瓶颈框架中 attention 弥散和信息瓶颈部分未真实测量，对 ALiBi/Mamba+Attn 混合架构的预测（Table 4）都是 conjecture [L2:2601.15300]。
  • Attention head/layer 级的机制解释：NoLiMa 的"attention SNR"是行为级论断 [L2:2502.05167]，PCD 的频率分工假设缺少 per-band ablation [L2:2506.08371]，都需要 head/layer 级的可解释性研究来升级为机制。
  • 多跳位置偏置在新一代模型上的验证：Baker 2412.10079 仅测了 GPT-3.5 / MPT-7B / Llama-2（2023–2024 模型），GPT-4o / Llama-3.1-128k / DeepSeek-V3 上 lost-in-between 是否仍然成立完全未知 [L2:2412.10079]。
  • UniVideo 向更长视频和端到端训练的扩展：当前限于 129 帧 / 854px，且 MLLM 全程冻结——原生端到端训练能否实现更深的理解-生成协同 [L2:2510.08377]？

6. 时代定位 #

Algorithm 类别在 2024–2026 年处于长上下文 LLM 从"窗口扩张赛"转入"质量审计赛"的关键转折期。2024 年 128K–1M token 已成标配，但真正可用的有效长度远低于营销数字：

  • 评测工具演进：从 Needle-in-the-Haystack（已饱和）→ VaL Probing [L2:2404.16811] → 多跳位置扰动 [L2:2412.10079] → NoLiMa 去字面匹配 [L2:2502.05167] → Natural Length cliff 检测 [L2:2601.15300]，评测难度逐级递增，每一代都暴露上一代 benchmark 掩盖的缺陷。
  • 修复方案分层：训练侧（IN2 数据合成）解决数据层偏置 [L2:2404.16811]，推理侧（PCD 对比解码）解决排序层衰减 [L2:2506.08371]，服务侧（context budget 软上限）做工程降级 [L2:2601.15300]——三层互补但尚未有工作验证叠加效果。

与相邻类别的关系：

  • → Kernel：长上下文算法不引入新算子，但对 FlashAttention 等 kernel 提出了正确性新要求（中间位置不能有 quiet under-performance）[L2:2511.05850]；PCD 可受益于"双 RoPE base 融合 kernel"[L2:2506.08371]。
  • → Framework：推理框架（vLLM / SGLang）应暴露"有效上下文长度"作为一等公民元数据 [L2:2502.05167]，并可在 scheduler 层引入 context budget 软上限 [L2:2601.15300]。
  • → Agent：RAG agent 不能假设 LLM 会平等使用所有片段——需要主动做 evidence packing 排序，按 adjacent-gold 原则优化组合 [L2:2412.10079]；agent 运行时应确保工具定义等 recall-critical 内容在词汇上接近 query [L2:2502.05167]。
  • → Model：UniVideo 验证了"冻结 MLLM 作为通用编码器 + MLP self-attention 对齐"的可复用设计模式 [L2:2510.08377]。

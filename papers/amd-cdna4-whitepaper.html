AMD CDNA 4 Architecture White Paper

Summary

Core Contribution

AMD CDNA 4架构全面押注AI精度下探，引入MXFP Microscaling格式(MXFP8/6/4)，矩阵资源翻倍，LDS扩容至160KB，在FP8达5 PFLOPS、MXFP4达10 PFLOPS，同时大胆移除TF32硬件并削减FP64 Matrix性能。

Summary

CDNA 4是AMD对AI优先策略最明确的表态。架构从CDNA 3的8 XCD(5nm)+4 IOD(6nm)演进为8 XCD(3nm N3P)+2 IOD(6nm N6)——XCD升级至3nm获得更高密度和能效，IOD从4个精简为2个以降低功耗和延迟。256个CU(32/XCD)虽少于MI300X的304个，但per-CU矩阵吞吐翻倍(16-bit及以下数据类型)，总AI性能实现大幅跃升。

核心创新是MXFP Microscaling格式支持——业界首批实现OCP MX标准的GPU，32个元素共享一个8-bit指数，提供MXFP8(E5M2/E4M3)、MXFP6(E3M2/E2M3)、MXFP4(E2M1)三个精度层级。MXFP6是AMD独有的中间精度，提供FP8和FP4之间的精度-性能折中。MI355X在FP8达5.0 PFLOPS，MXFP4/MXFP6达10 PFLOPS，较MI325X提升约3.85x(FP8基准)。

LDS从64KB扩容至160KB(2.5x)，读带宽从128B/cycle翻倍至256B/cycle，大幅提升数据供给能力。Transcendental单元也2x以匹配矩阵吞吐增长。显存升级至288GB HBM3E(8.0TB/s)。

大胆取舍: TF32硬件移除(转为BF16软件模拟)，FP64 Matrix性能减半(78.6 TFLOPS vs CDNA3的163.4 TFLOPS)，腾出面积给MX格式和矩阵资源翻倍。这标志着AMD从HPC/AI双重定位明确转向AI优先。

Key Findings

  • 8 XCD(3nm N3P) + 2 IOD(6nm N6)，IOD从4个缩减为2个
  • 256 CU, FP8: 5.0 PFLOPS, MXFP4/MXFP6: 10 PFLOPS
  • MXFP Microscaling: OCP MX标准, MXFP8/6/4三档精度
  • LDS 160KB/CU (2.5x), 读带宽256B/cycle (2x)
  • 288GB HBM3E, 8.0TB/s
  • TF32硬件移除, FP64 Matrix减半 — AI优先的明确信号
  • Matrix Core 16-bit及以下吞吐翻倍

Limitations

  • TF32硬件移除，转为BF16软件模拟，兼容性和性能可能受影响
  • FP64 Matrix性能减半(78.6 vs 163.4 TFLOPS)，HPC用户受损
  • CU数量减少(256 vs 304)，依赖per-CU吞吐翻倍弥补
  • TDP 1400W(DLC)功耗极高
  • 互联带宽(1,075GB/s)落后NVLink5(1,800GB/s) 67%
  • 缺乏NVL72等价的机架级GPU直联方案，大规模训练扩展性受限

Infrastructure Impact

  • Algorithm: MXFP6提供FP8/FP4之间的精度-性能新折中，需要量化算法适配
  • Kernel: LDS 160KB + 256B/cycle允许更大tile size和更高数据复用
  • Framework: MX格式需要PyTorch/JAX框架层面的量化支持
  • LLM: 288GB显存 + 10 PFLOPS MXFP4，单卡可推理更大模型，更低精度
  • System: IOD缩减为2个简化了系统设计，但互联带宽差距需要网络层弥补
Details

Deep Analysis (hardware)

1. Architecture Overview

  • 产品: AMD Instinct MI350X (AC) / MI355X (DLC) / CDNA 4
  • 工艺: XCD: TSMC 3nm (N3P), IOD: TSMC 6nm (N6)
  • 封装: Chiplet 3D堆叠 — 8 XCD + 2 IOD
  • 目标市场: AI训练/推理 (明确AI优先，HPC次要)

2. Generational Delta (vs CDNA 3 / MI300X/MI325X)

新增: MXFP8/MXFP6/MXFP4 Microscaling格式, Matrix Core翻倍(16-bit以下)

移除: TF32硬件(转为BF16软件模拟), 2个IOD(从4→2)

增强: XCD 3nm(vs 5nm), LDS 160KB(vs 64KB, 2.5x), LDS读带宽256B/cycle(vs 128B/cycle, 2x), 显存288GB(vs 256GB), 带宽8.0TB/s(vs 6.0TB/s), Transcendental单元2x

削弱: FP64 Matrix减半(78.6 vs 163.4 TF), CU减少(256 vs 304)

3. Compute Unit Deep Dive

  • 每XCD: 32 CU (vs CDNA3的38), 4MB L2 Cache
  • 每CU: 1个Matrix Core Engine (16-bit以下吞吐翻倍), 4个Vector单元, Scalar单元
  • LDS: 160KB/CU (2.5x扩容), 读带宽256B/cycle (2x)
  • L1 Data Cache: 32KB/CU
  • Wavefront: 64 work-items
  • MFMA支持: FP64/FP32/FP16/BF16/FP8/INT8/MXFP8/MXFP6/MXFP4
  • Transcendental单元: 2x (匹配矩阵吞吐增长)

5. Memory Subsystem

  • HBM3E: 288GB, 8.0TB/s
  • Infinity Cache: 256MB
  • L2: 32MB (4MB/XCD × 8)
  • L1 Data Cache: 32KB/CU
  • LDS: 160KB/CU, 256B/cycle读带宽

6. Interconnect

  • Infinity Fabric: 7x links + 1x PCIe Gen5, 1,075 GB/s
  • PCIe Gen5: x16
  • XCD-IOD: 3D堆叠互联
  • IOD缩减: 从4个到2个，降低功耗和延迟

7. MXFP Microscaling Format Detail

  • 标准: OCP MX (Open Compute Project Microscaling)
  • 机制: 32个元素共享1个8-bit指数(shared exponent)
  • MXFP8: E5M2/E4M3 element + 8-bit shared exponent
  • MXFP6: E3M2/E2M3 element + 8-bit shared exponent (AMD独有中间精度)
  • MXFP4: E2M1 element + 8-bit shared exponent
  • 优势: 比per-tensor量化更细粒度，比per-element FP格式更高效

8. Software & ISA

  • ROCm (持续演进)
  • MFMA指令族: 新增MXFP8/MXFP6/MXFP4矩阵指令
  • TF32: 转为BF16软件模拟路径
  • HIP + LLVM编译器
  • vLLM/SGLang/PyTorch MX格式支持

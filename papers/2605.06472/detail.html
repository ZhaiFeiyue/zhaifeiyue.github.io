PBKV: Prediction-Based KV-Cache Management for Dynamic Agent Workflows #

Haoyu Zheng, Fangcheng Fu, Jia Wu, Binhang Yuan, Yongqiang Zhang, Hao Wang, Yuanyuan Zhu, Xiao Yan, Jiawei Jiang | 2026-05 | arXiv 2605.06472 Category: agent | Tags: kv-cache-management, multi-agent-serving, workflow-prediction, cache-eviction, algorithms-with-predictions Read: 2026-05-25

§1 TL;DR #

PBKV 用 GraphSAGE 多步预测 agent 调用序列，驱动「分层淘汰 + 保守预取」的 KV 缓存策略，在动态多 agent 工作流 serving 中实现 1.85× 加速和 2.55× 缓存命中率提升，并提供基于 Lipschitz 连续性的优雅退化保证。

§2 Q1 / Q2 / Q3 #

Q1 痛点：LRU 与静态 DAG 假设均无法应对动态 agent 工作流 #

多 agent 工作流（Planner → Coder → Tester → 重试循环）中上下游 agent 共享大量 KV-Cache（系统 prompt、工具描述、对话历史），理论命中率可达 90%+。但实际受限于 GPU 显存，必须做淘汰决策：

  • LRU 的失配：LRU 假设时间局部性，但工作流级缓存复用由 DAG 结构决定——一个缓存节点可能在多个 agent 调用之间空闲然后被复用。LRU 会过早淘汰这些结构上仍有价值的节点。实测命中率仅 27-28%。
  • KVFlow 的静态假设：KVFlow 将 Belady 算法适配到静态 Agent Step Graph 上，按"距执行步数"打分淘汰。但真实工作流是动态的——Tester 失败触发重试循环、Retriever 根据内容生成子查询——运行时 agent 调用序列取决于上下文，KVFlow 无法处理。
  • 预测与代价的不对称：agent 工作流天然难预测（随机解码沿路径传播不确定性），而预测错误的代价极高——一次 cache miss 需要对累积的数万 token 做完整 re-prefill。

Paper Figure 1：代码生成任务的 agent 调用图。Tester 根据运行结果条件性地触发 Analyzer → Coder 的重试循环，使调用序列在运行时才确定。

这张图展示了为何静态 DAG 假设失败：重试循环的触发取决于 Tester 的 LLM 输出，基于预定义执行顺序的缓存策略无法为这类动态分支做出正确淘汰决策。

Q2 方法：预测驱动的三组件 KV-Cache 管理 #

PBKV 的核心设计原则：预测即将到来的 agent 调用 → 评估现有缓存的复用价值 → 驱动淘汰与预取。三个组件：

  1. 多步预测器：融合 GraphSAGE 拓扑先验（调用图结构）、注意力历史聚合（工作流前缀）、prefill 语义信号（LLM 隐藏状态），单次前向传播输出 $K$ 步 agent 分布。~350K 参数，1024 请求推理仅需 1.56ms。
    1. 分层淘汰：(a) 生命周期感知——优先回收已终止工作流的"退役缓存"（确定性零价值）；(b) 评分驱动——对活跃缓存按 $K$ 步期望复用次数 $\mathrm{Score}(c)$ 排序淘汰。分层设计确保确定性保护嵌入概率系统——预测准确时获益，不准确时退化至生命周期感知水平而非 LRU。
      1. 保守预取：仅在空闲 GPU 空间和纯 decode 批次的空闲 PCIe 带宽内，从 host 预取高价值缓存。不淘汰活跃缓存为预取腾空间，确保预测错误时无负面影响。

        2. 核心技术壁垒：「确定性护栏 + 概率系统」的分层架构。退役缓存淘汰提供不依赖预测的确定性基线（单独贡献 1.66× 命中率提升），评分驱动淘汰的 Lipschitz 退化保证（Theorem 5.1）确保增益随预测精度连续变化而不断崖式下降，保守预取的空间/带宽双约束确保错误预测不反噬。三层联合使 PBKV 在预测精度从 0.94 到随机的整个谱上都严格优于 LRU——这是 KVFlow 和 PASTE 等同类工作未能实现的稳健性。

        Q3 结果 #

        • 动态工作流：相比 LRU 延迟降低最高 1.85×，命中率提升 2.55×
        • 静态工作流：相比 SOTA KVFlow 延迟降低 1.26×，命中率提升 1.39×
        • 预测器精度：1-step 0.94，3-step 0.77（1K 训练轨迹，HoVer+LangChain）
        • 调度开销可忽略：预测推理 1.18ms（<10% decode step），评分更新 1.53μs
        • 理论保证：淘汰代价遗憾与预测误差呈 Lipschitz 连续，乘子 $1/(2(1-\gamma))$ 独立于系统规模

        Agent Scope #

        • 任务类别：serving 基础设施层——为动态多 agent 工作流提供 KV-Cache 管理，非 agent 本身
        • 交互模式：服务多轮 agent 调用链（每个 agent 是一次 LLM 请求），工作流长度 3-10+ agent
        • 自治性：缓存管理完全自动，预测器自主推理；不涉及 agent 层面的规划或决策
        • 覆盖范围：动态工作流（含条件分支、重试循环）和静态工作流均适用

        §3 架构 / 方法图 #

        系统全景 #

        Paper Figure 2：PBKV 系统架构。对每个活跃工作流 $w$，预测器产生 $K$ 步预测，驱动共享评分函数 $\mathrm{Score}(c)$，同时服务于分层淘汰和保守预取策略，底层为 Radix Tree (GPU) + HiCache (host) 的两级存储。

        系统分三层：预测层（轻量 GNN + attention + MLP，独立 CUDA stream，与 decode 重叠），策略层（淘汰/预取决策，CPU 调度线程），存储层（SGLang Radix Tree + HiCache）。策略层不占 GPU 关键路径。

        预测器架构 #

        Paper Figure 3：预测器架构。三路信号—— GraphSAGE 拓扑嵌入 $\mathbf{h}_{\mathrm{cur}}$、注意力工作流前缀摘要 $\mathbf{h}_{\mathrm{path}}$、prefill 语义信号 $\mathbf{h}_{\mathrm{txt}}$——拼接后经 MLP 联合预测 $K$ 步 agent 分布。

        三路设计的互补性：

        • $\mathbf{h}_{\mathrm{cur}}$（拓扑嵌入）：2 层 GraphSAGE 在 agent 调用图 $G$ 上传播，编码当前 agent 在全局拓扑中的位置及邻居转移概率。跨工作流共享的结构先验。
        • $\mathbf{h}_{\mathrm{path}}$（前缀摘要）：以当前 agent 嵌入为 query，对工作流前缀做缩放点积注意力，upweight 最具信息量的历史 agent。区分同一 agent 在不同前缀路径下的行为差异。
        • $\mathbf{h}_{\mathrm{txt}}$（语义信号）：复用 LLM prefill 的 post-norm 隐藏状态经 ReLU 投影。区分相同前缀但不同请求内容——零额外推理成本。

        $K$ 步联合预测（single forward pass）而非自回归逐步展开，避免误差累积。

        评分与淘汰机制 #

        Paper Figure 4：对缓存节点 $c$，聚合所有关联活跃工作流的 $K$=3 步预测，各步乘以存活概率 $s^{(k)}$ 和信心衰减 $\gamma^{k-1}$，跨 $m$ 个工作流求和得到 $\mathrm{Score}(c)$。

        评分的物理意义：$\mathrm{Score}(c)$ 等于淘汰 $c$ 在未来 $K$ 步内引起的期望折扣 miss 次数（Lemma G.1 证明）。跨工作流聚合自然保护全局缓存（被多工作流共享的系统 prompt 得分最高）。

        PBKV Serving 循环 #

        stateDiagram-v2 [*] --> WorkflowArrival WorkflowArrival --> PredictorInference: 新请求 / agent 转移 PredictorInference --> ScoreUpdate: K步预测分布 ScoreUpdate --> EvictionCheck: GPU 显存压力?         关键的"确定性护栏"实现点在 EvictionCheck → RetiredEvict 分支：只有所有退役缓存耗尽后才触发概率评分驱动的 ScoreDrivenEvict。WorkflowEnd → RetireTag 是生命周期追踪的核心——监听终止事件并即时标记。

        规划与推理 #

        • 规划风格：PBKV 的"规划"是缓存管理层面的 $K$-step lookahead，非任务层面 planning。预测器产出的是 agent 调用概率分布，而非执行计划。
        • 分解：bottom-up——从当前状态预测未来，而非 top-down 分解任务。
        • 预算：$K$-step horizon（最优 $K$=3）+ 信心衰减 $\gamma$=0.7 限制远期预测的影响。
        • 回退：预测失败时退化至生命周期感知策略（确定性基线），而非退化至 LRU。

        方程物理意义与推导检验 #

        Check 1 — 单步复用价值（Eq 1）

        $$\mathrm{Value}(c) = \sum_{w \in \mathcal{W}^{\mathrm{act}}(c)} A_w(c) \cdot P_w$$

        聚合所有关联活跃工作流下一步调用到 $c$ 所涉 agent 的概率。内积 $A_w(c) \cdot P_w$ 对那些之前访问过 $c$ 的 agent 取其被调用概率求和；跨工作流累加使全局缓存（多 workflow 共享）自然获得更高分。用于保守预取的排序依据（不使用 $K$-step，因为预取是逐步增量的）。

        Check 2 — $K$ 步折扣评分（Eq 2）

        $$\mathrm{Score}(c) = \sum_{k=1}^{K} \gamma^{k-1} \sum_{w \in \mathcal{W}^{\mathrm{act}}(c)} s_w^{(k)} \cdot A_w(c) \cdot P_w^{(k)}$$

        将单步价值扩展到 $K$ 步 horizon。每步贡献被两重折扣抑制：信心衰减 $\gamma^{k-1}$（远期预测不可靠）和存活概率 $s_w^{(k)}$（工作流可能已终止）。物理意义经 Lemma G.1 证明：$\mathrm{Score}(c)$ = 淘汰 $c$ 导致的期望折扣 miss 数。淘汰 Score 最低的节点等价于贪心最小化期望 miss。

        Check 3 — 存活概率

        $$s_w^{(k)} = \prod_{j=1}^{k-1} \bigl(1 - p_{w,\langle\mathrm{END}\rangle}^{(j)}\bigr)$$

        链式条件概率：存活到第 $k$ 步 = 前 $k-1$ 步每步都未终止的联合概率。当 $p_{\langle\mathrm{END}\rangle}$ 较高时，后续步数的贡献自动衰减。$\langle\mathrm{END}\rangle$ 在训练中被当作常规 agent 目标，终止概率可被自然学习。

        Check 4 — Lipschitz 遗憾界（Theorem 5.1 / G.4）

        $$0 \leq \mathcal{R}(B) := \mathcal{L}\!\bigl(\widehat{E}_B\bigr) - \mathcal{L}\!\bigl(E_B^{\star}\bigr) \leq \frac{1}{2(1-\gamma)} \sum_{c \in \widehat{E}_B \triangle E_B^{\star}} \epsilon_c^{\gamma}$$

        遗憾上界 = 对称差集中每个节点的折扣预测误差之和 × 常数 $\frac{1}{2(1-\gamma)}$。该常数独立于活跃工作流数量和缓存大小。预测完美时 $\epsilon_c^{\gamma}=0$，遗憾为零；遗憾随误差线性增长，不存在突变点。这是 algorithms-with-predictions 框架（Lykouris & Vassilvitskii 2021, Mitzenmacher & Vassilvitskii 2022）在 KV-Cache 管理上的首次应用。

        Check 5 — Score = Expected Miss Count（Lemma G.1）

        在淘汰抽象（被淘汰节点 $K$ 步内不恢复）下，$\mathrm{Score}(c) = \mathrm{EMC}(c)$。证明通过分解每步每工作流的 miss 指示随机变量 $\mathbf{1}[w \text{ survives to step } k \wedge \text{invokes some } a \in \mathcal{O}_w(c)]$，取期望后各项恢复 Eq 2 的形式。建立了评分函数与决策论目标的等价性。

        Check 6 — GraphSAGE 传播（Eq 3）

        $$\mathbf{H}^{(\ell)} = \mathrm{ReLU}\!\left(\bigl\mathbf{H}^{(\ell-1)} \| A\mathbf{H}^{(\ell-1)}\bigr})^\top\right)$$

        自身表示 $\mathbf{H}^{(\ell-1)}$ 与邻域聚合 $A\mathbf{H}^{(\ell-1)}$ 拼接后线性变换 + ReLU。行归一化转移矩阵 $A$ 从离线轨迹构建。2 层感受野覆盖 2-hop，对 5-10 节点的 agent 调用图已足以编码全局拓扑。归纳式学习（vs. transductive GCN）允许对训练时未见过的工作流前缀做推理。

        Agent-Specific 验证 #

        • Success-rate model：不适用于传统 agent 成功率。论文的等价量是"预测精度 → 缓存命中率 → 端到端延迟"的链式映射。通过 predictor backbone × 训练数据量 × lookahead $K$ 三维 sweep 给出（§6.3, Appendix C）。GraphSAGE 在所有 horizon 上领先；精度→收益单调递增。
        • 延迟预算/轮：预测推理 1.18ms（P99: 1.68ms），淘汰决策 0.44ms，评分更新 1.53μs，预取决策 0.64ms。总计 <2.3ms vs. decode step 12.34ms（<19%），且全部在 CPU 线程或独立 CUDA stream 上，不在 GPU 关键路径。
        • 失败模式分类：(1) 预测错误导致高价值缓存被淘汰 → re-prefill 代价——Lipschitz 界约束此类损失；(2) 错误预取低价值缓存 → 仅浪费带宽、无负面影响——保守设计保证；(3) 高并发下显存耗尽 → LRU 在并发 84 时 crash 3/10 trials，96 时 10/10 trials；PBKV 保持稳定。论文确认 (1) 为主导失败模式。
        • 可绑定但缺失的形式化指标：agent-level success rate 可定义为"给定 cache hit rate $h$，端到端延迟低于 SLO 阈值的工作流比例"，但论文未建模 SLO 约束。

          预测误差鲁棒性 #

          Paper Figure 12：向预测器注入噪声 $\tilde{P}_w^{(k)} = (1-\lambda) P_w^{(k)} + \lambda U$ 后，PBKV-HE（无预取）、保守预取、激进预取（$\rho$=20%）的命中率变化。

          关键发现：

          • 激进预取仅在 $\lambda \leq 10\%$ 时略优于保守（~2pp），$\lambda=20\%$ 时已被反超，$\lambda=30\%$ 时低于无预取的 PBKV-HE——错误预测使激进预取适得其反。
          • 保守预取在所有噪声水平下一致优于 PBKV-HE——验证"只用空闲资源"设计的稳健性。
          • 激进策略的标准差随 $\rho$ 单调增长，即使无噪声也不稳定。

          工具与环境接口 #

          PBKV 在 serving 层运行，其"工具接口"：

          • 输入事件：agent 完成/启动通知（工作流状态转移）
          • 环境假设：SGLang Radix Tree + HiCache 两级存储；每次 agent 调用是独立 LLM 请求；工作流间私有缓存不共享（随机解码导致前缀数 token 内发散）
          • 副作用：淘汰/预取修改 GPU↔host 缓存分布（可逆——被淘汰到 host 的缓存可被预取回 GPU）
          • 环境确定性：缓存存储状态在请求间持久化；预测器推理确定性（给定输入）；LLM 解码的随机性通过概率建模吸收

          核心技术壁垒——实现视角 #

          分层淘汰的「确定性护栏嵌入概率系统」设计需要两个关键实现：

          1. 工作流生命周期追踪：为每个 Radix Tree 缓存节点维护访问工作流集合 $\mathcal{W}(c)$。服务端监听工作流终止事件，将已终止工作流的 $\mathcal{W}^{\mathrm{act}}(c)$ 条目迁移至退役集。退役节点按"剩余访问工作流数"降序排列，使共享全局前缀最后被淘汰。实现上是 per-node workflow-set + 终止回调——极其简单，但此前无系统实现。
            1. 评分堆维护：活跃缓存节点按 $\mathrm{Score}(c)$ 维护最小堆。取顶 $O(1)$，pop/insert/score-update $O(\log n)$。每次工作流状态变更触发预测器刷新 → 相关节点评分批量更新。

              2. 关键实现细节 #

              1. Prefill 隐藏状态零成本复用：语义信号 $\mathbf{h}_{\mathrm{txt}}$ 取自 LLM prefill 的 post-norm 最后 token 隐藏状态——不需额外推理，只需在 prefill 结束时读取一个向量并投影。最优提取层因模型/workload 而异（Appendix C.3），但 post-norm 作为通用 fallback 在所有实验中表现稳健。
                1. 预取的空间/带宽双约束实现：可用预取空间 $S_a$ = 空闲 GPU 内存 + 退役缓存占用量；可用带宽 $S_{bw} = \text{Bandwidth} \times \text{StepDuration}$；最终预算 $S = \min\{S_a, S_{bw}\}$。仅在 decode-only 批次（>90% 批次属此类）激活。候选按 1-step $\mathrm{Value}(c)$ 降序贪心装载（精确最优是 knapsack NP-hard）。

                  2. LLM 骨干要求 #

                  • 测试模型：Qwen3-14B（单 GPU）、Qwen3-32B（2-way tensor parallelism）
                  • 最低模型规模：未测试更小模型。PBKV 的收益与缓存量和显存压力正相关——小模型缓存压力低，收益空间也更小。
                  • 能力要求：无特殊要求，PBKV 是 serving 层优化，不要求 tool-call 格式或结构化输出。
                  • 骨干敏感性：缓存管理策略与 LLM 架构无关；预测器需按 workload 训练（但仅需 1K 轨迹，~350K 参数）。

                  多 Agent 拓扑 #

                  • 拓扑：由上游 agent 框架（LangChain / AutoGen / CrewAI）定义。PBKV 通过调用图 $G=(V,E)$ 隐式编码拓扑，不需显式拓扑感知。
                  • 协调机制：工作流内 agent 通过共享上下文协调（消息传递 + KV-Cache 前缀继承）。PBKV 利用 Radix Tree 的前缀共享结构捕获这种协调。
                  • 角色特化：不同 agent 有不同 system prompt，编码为 Radix Tree 中的不同前缀分支。PBKV 的跨工作流聚合自然给高频 agent 分支更高评分。
                  • 失败隔离：单个 agent 失败不影响 PBKV——工作流异常终止后触发退役标记，等价于正常终止。

                  Production Readiness #

                  • 沙箱隔离：依赖 SGLang 进程级隔离，PBKV 不引入新隔离边界。
                  • 可观测性：论文未讨论 trace logging。预测分布、评分、淘汰/预取决策可记录但未实现。
                  • 成本控制：保守预取设计即是成本控制——不消耗有价值 GPU 资源。预测器开销 <2.3ms/轮。
                  • Secrets / auth：不适用——PBKV 运行在推理引擎内部，不接触外部 API。
                  • 并发控制：论文测试并发 24-96 工作流。LRU 在 84+ 并发时崩溃；PBKV 在所有测试并发下稳定。

Kimi Linear: An Expressive, Efficient Attention Architecture

LLM   Yu Zhang, Zongyu Lin, Xingcheng Yao, Jiaxi Hu, Fanqing Meng  ·  2025-10  ·  arXiv ↗
kernelalgorithmframework
Summary

Core Contribution

首次提出在公平对比下全面超越Full Attention的混合线性注意力架构——Kimi Linear，核心是Kimi Delta Attention (KDA)，基于channel-wise diagonal gate扩展Gated DeltaNet，并通过DPLR约束变体实现~2x kernel加速。

Summary

Kimi Linear解决了线性注意力长期以来质量不如Full Attention的核心痛点。其核心KDA模块将Gated DeltaNet的scalar forget gate替换为channel-wise diagonal gate Diag(α_t)，使每个特征维度维持独立的遗忘率，实现更精细的有限状态RNN记忆管理。关键公式：S_t = (I - β_t k_t k_t^T) Diag(α_t) S_{t-1} + β_t k_t v_t^T。

通过将transition matrix约束为DPLR的特殊形式(绑定a=b=k)，KDA kernel比通用DPLR实现快~2x，同时保持与经典delta rule的一致性。chunkwise算法将序列分为L/C chunks (C=64)，结合WY表示和UT变换实现高效硬件映射。

在3B activated / 48B total的MoE架构上采用3:1 KDA:MLA混合比例，MLA层使用NoPE（无位置编码）。预训练1.4T tokens后，Kimi Linear在所有评估任务上超越full MLA（MMLU-Pro 51.0 vs 47.2, RULER 84.3 vs 81.3），同时减少75% KV cache、在1M上下文实现6.3x解码加速。5.7T tokens的最终版本RULER@1M达到94.8。

Key Findings

  • KDA的channel-wise gate在合成任务(Palindrome, MQAR, Stack)和真实benchmark上均显著优于GDN的scalar gate，Mamba2完全失败
  • 3:1 KDA:MLA混合比例最优——pure MLA(0:1)最差，过高比例(15:1)validation退化
  • NoPE在MLA层上比RoPE效果更好(RULER 84.3 vs 78.8)，且简化长上下文训练
  • Sigmoid output gate > 无gate > Swish gate（Swish显著更差）
  • Scaling law显示~1.16x计算效率优势
  • RL training中Kimi Linear收益随训练推进不断扩大

Limitations

  • 纯线性注意力在精确复制和细粒度检索上仍有局限，需要hybrid设计
  • Scaling law优势适度(~1.16x)，可能受超参数影响
  • 5.7T模型48B total vs Moonlight 16B total不完全公平
  • Fine-grained decay引入数值精度问题，需secondary chunking

Infrastructure Impact

  • Algorithm: 约束DPLR变体将delta rule与channel-wise gating统一，开辟新设计空间
  • Kernel: 开源KDA kernel和vLLM实现，~2x优于通用DPLR
  • Framework: 75% KV cache削减直接利好推理系统，集成vLLM
  • LLM: 首次证明线性注意力可在同等训练条件下超越full attention
  • Agent: 1M context下6.3x解码加速，直接利好长上下文agent应用

#### 1b. Architecture Details

  • Base: MoE Transformer (基于Moonlight架构)
  • Attention: KDA (3层) + MLA (1层) 交替，inter-layer hybrid
  • Position Encoding: KDA层无显式PE (gates隐式编码), MLA层NoPE
  • Normalization: RMSNorm (pre-norm)
  • Activation: SwiGLU (FFN), Swish (v branch), Sigmoid (gate, β)
  • MoE: 256 routed + 1 shared expert, top-8, first layer dense

2. Scale & Training

  • 3B activated / 48B total (256 routed + 1 shared expert, top-8)
  • MuonClip optimizer, WSD lr schedule, lr=1.1e-3
  • 1.4T tokens (fair comparison), 5.7T tokens (final release)
  • Context: 4k pretrain → 128k extension → 1M (final)
  • SFT: 扩展K2 SFT数据，多阶段(broad → reasoning-intensive)
  • RL: Math/Code/STEM, truncated importance sampling, dynamic KL, PTX loss

3. Quantization & Compression

  • KV cache: 75%的层使用fixed-size state (d_k×d_v = 128×128 per head per KDA layer)
  • 仅MLA层需要传统KV cache，25%的层
  • 论文未讨论weight quantization，但KDA的fixed-size state天然适合量化

4. Performance

  • Short-context: MMLU-Pro 51.0 (+3.8 vs MLA), BBH 72.9 (+1.3)
  • Long-context: RULER 84.3 (+3.0 vs MLA), MRCR 29.6 (+7.0)
  • Efficiency: 1M TPOT 1.84ms vs 11.48ms (6.3x), Prefill 2.9x faster
  • Final model (5.7T): RULER@1M 94.8, MATH500 94.6, GPQA-Diamond 71.7

5. Key Innovations

KDA Channel-wise Gating: 问题——GDN的scalar gate过于粗糙，所有channel共享遗忘率。KDA让每个channel独立decay，实现细粒度记忆管理。影响——合成任务(Palindrome 100% vs GDN <80%)和真实任务全面提升。

DPLR约束变体: 问题——通用DPLR kernel计算量大。通过绑定a=b=k，secondary chunk matrix从4个减到2个，消除3个额外矩阵乘。影响——~2x kernel加速。

NoPE for MLA: 问题——RoPE与MLA的latent compression不兼容，限制长上下文。NoPE让MLA在inference时转换为MQA，且不需要频率调参。影响——RULER 84.3 vs 78.8 (+5.5)。

3:1 Hybrid Ratio: 问题——纯线性无法精确检索。3:1比例平衡了quality(接近7:1)和efficiency(远超1:1)。

7. Deployment Considerations

  • 开源: KDA kernel, vLLM实现, 模型checkpoints (base + instruct)
  • 最小部署: 需支持MoE的推理框架 + KDA kernel
  • 推荐: vLLM with KDA kernel support
  • 量化: KDA层的fixed-size state天然低memory footprint

8. Position in Model Landscape

  • 首个在公平对比下超越full attention的hybrid linear attention方案
  • 为线性注意力的实际部署扫清了quality barrier
  • 后续被Kimi K2.5采用，证明了在更大规模系统中的可行性

Kimi Linear: An Expressive, Efficient Attention Architecture

LLM   Yu Zhang, Zongyu Lin, Xingcheng Yao, Jiaxi Hu, Fanqing Meng  ·  2025-10  ·  arXiv ↗
kernelalgorithmframework
TL;DR
首个在公平对比下全面超越 Full Attention 的混合线性注意力架构。核心 KDA (Kimi Delta Attention) 把 Gated DeltaNet 的 scalar gate 替换为 per-channel diagonal gate Diag(α_t)；通过 DPLR 约束 (a=b=k) 让 chunkwise kernel 比通用 DPLR 快 ~2x；层级混合 3 KDA + 1 MLA (NoPE), 实测 3:1 最优。

模型架构互动图

4 页：(1) 27 层中 KDA / MLA 的交错布局，(2) KDA 块内部（线性注意力 + 通道门 + 短卷积），(3) MLA 块（NoPE 模式），(4) 训练管线 + 与对照对比。

Core Contribution

第一个在公平对比下全面超越 Full Attention 的混合线性注意力架构。核心构件 KDA (Kimi Delta Attention) 把 Gated DeltaNet 的 scalar forget gate 替换成 per-channel diagonal gate \(\text{Diag}(\alpha_t)\)，让每个特征维度有独立的遗忘率；通过把转移矩阵约束为 DPLR 的特例 (a = b = k) 让 chunkwise kernel 比通用 DPLR 快 ~2×。层级混合：每 3 层 KDA 后插 1 层 MLA（NoPE），实测 3:1 比例最优。

Summary

Kimi Linear 解决线性注意力多年的"质量-效率"两难。其核心 KDA 模块的递推：

$$S_t = \bigl(I - \beta_t k_t k_t^\top\bigr) \cdot \text{Diag}(\alpha_t) \cdot S_{t-1} + \beta_t k_t v_t^\top, \qquad o_t = S_t^\top q_t$$

相比 Mamba2 的 scalar gate、GLA 的全 channel gate，KDA 的 diagonal gate 给出"中间地带"——每 channel 一个遗忘率但保持 rank-1 delta 修正，因此可以用约束 DPLR 推导出 chunkwise kernel，在 H100/MI300X 上比通用 DPLR 快 2× 且与 delta rule 数值一致。

层级 hybrid 设计：

  • 设计：每 3 个 KDA 层 + 1 个 MLA 层（3:1 比例）
  • 释出 27 层模型实际布局：20 KDA + 7 MLA（≈ 2.86:1，不是完美 3:1，因为 27 不被整除）
  • KDA 层（1-based config）: 1, 2, 3, 5, 6, 7, 9, 10, 11, 13, 14, 15, 17, 18, 19, 21, 22, 23, 25, 26
  • MLA 层（1-based）: 4, 8, 12, 16, 20, 24, 27

MLA 层用 NoPE——不施加 RoPE。位置/recency 完全靠 KDA 层的 data-dependent gate + short conv 提供。这让 model_max_length 拉到 1 048 576 (1M) 而无 RoPE 外推问题。

MoE backbone：48B total / 3B active；256 routed + 1 shared, top-8；layer 0 dense；vocab 163840；hidden 2304；FFN dense=9216, MoE expert=1024。

训练：Pretrain (fair) 1.4T @ 4K · MuonClip + WSD lr=1.1e-3 · global batch 32M。Pretrain (release) 5.7T。SFT 两段（IF → reasoning）。RL 用 K1.5 算法 + PTX loss + truncated importance sampling + dynamic KL。

推理优势：长上下文 decode 吞吐 +75%；KV cache 减 ~75%（KDA 层无 per-token KV，只有 fixed 128×128 state）。

Key Findings

  • KDA 公式：\(S_t = (I − β_t k_t k_t^\top)\text{Diag}(α_t) S_{t-1} + β_t k_t v_t^\top\) — diagonal gate 替代 scalar gate
  • Hybrid 比例：3:1 最优（消融证）；pure MLA (0:1) 最差，过疏 (15:1) 退化
  • NoPE on MLA > RoPE on MLA（Table 5），KDA 提供隐式 recency 已足够
  • Output gate sigmoid > Swish > none（消融）
  • ShortConv (k=4) 必需（消融），施加在 q/k/v 各路径上 + Swish
  • L2 norm on q,k（每 head 每 channel）— 训练稳定性关键
  • DPLR 约束 (a = b = k) 让 chunkwise 算法跳过 secondary chunk 的部分 matmul → ~2× speedup
  • Released 模型实际布局：27L 中 20 KDA + 7 MLA，不是完美 3:1（27 不整除）
  • Per-head state size 128×128 = 16 384 floats / head，32 heads → ~524K floats / layer (KDA 部分)
  • MLA layers 用 latent KV（512+64 = 576 floats / token），相对标准 MHA 减 14×
  • Pretrain：1.4T fair / 5.7T release，MuonClip + WSD
  • Long context 1M：NoPE + KDA gates，not RoPE/YaRN
  • Decode throughput +75% on long context（论文 Fig. 2）
  • AttnRes 已用（与 Attention Residuals 论文协同）

Limitations

  • 与 Moonlight (5.7T tokens, 16B total / 3B active) 公平比较不严谨（48B vs 16B total）
  • KDA chunkwise kernel 对 chunk size 64 优化，其他尺寸性能未知
  • pure linear attention 仍不可行 — 需要 MLA 层兜底（论文也承认）
  • 1M context 是设计支持，但实测在多少长度仍可用未给完整 needle 测试

Infrastructure Impact

  • Algorithm: KDA recurrence + 3:1 hybrid pattern + NoPE-on-MLA 都是新 design choice
  • Kernel: KDA Triton kernel (fla/ops/kda/)；secondary chunking in log domain；fused_kda_gate
  • Framework: vLLM (KimiDeltaAttention, chunk_kda / fused_recurrent_kda) 和 SGLang 都已支持
  • LLM: 证明线性注意力可在 production scale (48B) 全面超越 full attention
  • Hardware: 长上下文 KV 节省 75% → 单卡可服务更长 context

#### 1b. KDA Block (Linear attention with channel-wise gates)

💡 滚轮缩放 · 拖动平移 · 顶部页签切换 · 下载 .drawio

完整内部流程：

1. RMSNorm(2304)

2. 投影: q_proj, k_proj, v_proj Lin(2304 → 32·128)；β_proj Lin(2304 → 32) → sigmoid；α path 是 low-rank（α_proj_a Lin(d, r), α_proj_b Lin(r, 32)）

3. fused_kda_gate: g = -exp(A_log) · softplus(α + dt_bias) → 这是 channel-wise 遗忘率（log domain）

4. ShortConv1D (k=4) + Swish 在 q/k/v 各路径上独立

5. L2 normalize Q, K per head per channel — 稳定性关键

6. KDA Recurrence：上述 \(S_t\) 公式，状态 \(S \in \mathbb{R}^{128 \times 128}\) per head

7. FusedRMSNormGated — 输出门是 sigmoid，不是 Swish

8. o_proj Lin(32·128 → 2304) + Residual

关键设计点：

设计点数值 / 选择为什么
Diagonal gate \(\text{Diag}(\alpha_t)\)每 channel 独立遗忘率scalar gate 表达力不够，full gate 太贵；diagonal 最佳折中
DPLR 约束 a = b = k转移矩阵特殊形式让 chunkwise 跳过 secondary chunk 的部分计算 → 2× kernel 加速
Per-head state 128×128d_k = d_v = 128head_dim 128 是 H100 flash kernel 友好；状态 16K floats per head
Output gate sigmoid不是 SwishSwish 输出可正可负，破坏 RMSNorm 假设
L2 norm on q, kper head per channel稳定 KDA 数值（state 累积放大噪声）
ShortConv k=4causal Conv1D + Swish给 q/k/v 加局部时序，避免纯 token-wise
α 用 low-rankα_proj_a then α_proj_b直接 Lin(d, n_heads) 太多参数；low-rank 更高效

#### 1c. MLA Block (NoPE)

💡 滚轮缩放 · 拖动平移 · 顶部页签切换 · 下载 .drawio

Kimi Linear 的 MLA 与 K2.5 的 MLA 主要差异：

  • NoPE 模式（mla_use_nope=True）：放弃 RoPE 部分，只用 Q_nope 和 K_nope 计算 attention
  • 没有 q_lora_rank（直接 q_proj）— K2.5 用 1536, Kimi Linear 用 None
  • Latent KV 仍然 512 (kv_lora_rank) + 64 (qk_rope_head_dim)，但 RoPE k 部分实际不参与（NoPE 模式下）

KV cache 对照：

  • KDA layer: NO per-token KV cache — 只有 fixed 128×128 recurrent state + 3-step conv state
  • MLA layer (Kimi Linear): 576 floats / token (latent + nope k)
  • 假设 27 层中 7 层 MLA：长上下文 decode 时 KV cache 比纯 MLA 减 ~75%

#### 2c. RL 细节

Kimi K1.5 算法（不是 PPO/DPO/GRPO）：

  • PTX loss（来自 K2）— 防止 capability 崩塌
  • Truncated importance sampling — 解决 off-policy 偏差
  • Dynamic KL — 根据 KL 偏移自适应调系数
  • Dynamic minibatch size — 训练效率

#### 2e. 核心训练壁垒

最难复现的一件事：KDA chunkwise kernel 的数值正确性 + 速度。

线性注意力的难点不在于训练 loss（容易）而在于：

1. 长序列下 \(S_t\) 状态会数值漂移

2. Chunkwise 算法用 secondary chunk 减少 \(O(T^2)\)，但 secondary chunk 的累积矩阵要在 log domain 处理才稳定

3. KDA 用 DPLR a=b=k 约束跳过部分 matmul，但正确性证明非平凡（论文 §3.2, §6.2）

4. chunk_size=64 是论文实测最佳，其他尺寸需重新调

第二壁垒：1.4T tokens 公平 pretrain 配合 MuonClip + WSD + 1.1e-3 LR。这套配方对 attention 类型敏感——直接换 attention 算法可能让相同配方 diverge。

3. Performance

  • Long context: decode throughput +75% vs MLA-only stack
  • KV memory: ~75% reduction
  • Quality (1.4T fair-cmp): matches or beats MLA baseline on most benchmarks
  • Long-context Table 5: NoPE-MLA + KDA > RoPE-MLA + KDA
  • Ablation Table 1: 3:1 hybrid > 0:1 pure MLA > 15:1 too sparse

4. Open Questions

1. KDA 在 100B+ scale 是否仍能保持质量优势？（目前仅 48B）

2. NoPE on MLA 在超长 context (1M+) 真实任务的 needle test 成绩？

3. 3:1 比例是 Kimi 偏好还是普适，DeepSeek 风格 backbone 也适用？

4. KDA chunkwise kernel 对 AMD MI300X / MI355X 的移植成本？

5. Attention Residuals 是否在 Kimi Linear 中也用了？论文没明说

5. Position in Landscape

  • 首个 production-scale 在 fair-comparison 下超越 full attention 的混合线性方案
  • 把"线性注意力质量不够"的标签摘掉
  • 与 Mamba2 / Mamba in the Llama / Jamba 的差异：chunkwise kernel + diagonal gate + DPLR 约束 是 Kimi 独家
  • 已开源 KDA kernel (fla/ops/kda/)，社区可复用

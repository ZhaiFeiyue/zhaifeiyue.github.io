SPECTRE: Hybrid Ordinary-Parallel Speculative Serving for Resource-Efficient LLM Inference #

§1 TL;DR #

SPECTRE reuses idle tail-model GPUs as remote speculative drafters, switching ordinary/parallel decoding per round via a throughput-derived rollback threshold $r^*$. Reaches 2.28× over AR and 66% over best baseline on Qwen3-235B TP8, with <2% interference from shared draft traffic.

§2 痛点 · 方法 · 结果 #

Q1 痛点 #

Multi-model serving platforms exhibit long-tail demand: a few popular large models saturate GPUs while numerous smaller tail models sit underutilized. Existing speculative decoding approaches either (a) waste this idle capacity by requiring dedicated draft GPUs that serve no native traffic (PEARL, MineDraft), or (b) use pure parallel execution that suffers catastrophic rollback overhead at high batch sizes — PEARL degrades to 0.36× AR throughput at bs=128 on reasoning benchmarks, actively harming performance.

The fundamental tension: parallel speculative decoding hides draft latency by overlapping with target verification, but pre-generated drafts become invalid when verification discovers mismatches. At high batch sizes the rollback fraction climbs and the throughput gain from latency-hiding is overwhelmed by the loss of useful accepted tokens.

Q2 方法 #

Three interlocking mechanisms:

1. Hybrid ordinary-parallel coordination. Derives closed-form throughput for both modes. Ordinary mode throughput (sequential draft → verify):

$$\mathrm{Thr}_{\mathrm{ord}} = \frac{BL}{T_T + (\gamma - 1)T_D}$$

Parallel mode throughput (round latency collapses to $T_T$ since drafting overlaps, but rollback fraction $r$ degrades accepted tokens):

$$\mathrm{Thr}_{\mathrm{par}} = \frac{B(r + (1-r)L)}{T_T}$$

Setting $\mathrm{Thr}_{\mathrm{ord}} > \mathrm{Thr}_{\mathrm{par}}$ and solving for $r$ yields the switching threshold:

$$r^* = \frac{(\gamma - 1)L\, T_D}{(T_T + (\gamma - 1)T_D)(L - 1)}$$

When observed per-round rollback ratio $r_n$ exceeds $r^*$, the system switches to ordinary mode for that round.

2. Speculative priority scheduling. Non-preemptive policy that prioritizes speculative requests on the draft server, with a fairness rule scheduling regular requests every $K$ ($K{=}10$) speculative steps to prevent starvation. Keeps $\gamma T_D < T_T$ under mixed-tenant load.

3. Draft-side context compression. StreamingLLM-style prefix+suffix retention on draft-side prefill:

$$\mathcal{E}(x_{1:S}) = [x_{:\lfloor\frac{p}{2}S\rfloor},\; x_{-\lfloor\frac{p}{2}S\rfloor+1:}]$$

At $p{=}0.1$ (retain 10% of prompt), this keeps draft latency below target verification time even under TP8 asymmetry, at acceptable acceptance-length cost.

核心技术壁垒: The closed-form threshold $r^*$ is the key enabler. It requires recognizing that ordinary and parallel modes have distinct throughput formulas whose crossover is a simple function of four runtime-measurable quantities ($T_D, T_T, \gamma, L$). This transforms a seemingly dynamic scheduling problem into a per-round threshold check on the observed rollback ratio — no RL, no search, no offline profiling needed. The insight that a batch-level rollback ratio alone is sufficient for mode selection (rather than per-request statistics) is what makes the hybrid approach practical for online deployment.

方程物理意义 #

Ordinary throughput (Eq. 1): $\mathrm{Thr}_{\mathrm{ord}} = BL / (T_T + (\gamma{-}1)T_D)$. Numerator counts total accepted tokens across the batch. Denominator is the serial round latency: one target verification plus $\gamma{-}1$ fresh draft steps (the $\gamma$-th token is the bonus from verification). The key constraint is that target sits idle during drafting.

Parallel accepted length (Eq. 3): $L_{\mathrm{par}} = r \cdot 1 + (1{-}r) \cdot L$. A two-case expectation: rollback requests contribute exactly 1 token (the bonus token), non-rollback requests contribute full $L$. Boundary check: $r{=}0 \Rightarrow L_{\mathrm{par}}{=}L$ (no rollback, full acceptance); $r{=}1 \Rightarrow L_{\mathrm{par}}{=}1$ (all rollback, minimal acceptance).

Parallel throughput (Eq. 4): $\mathrm{Thr}_{\mathrm{par}} = B(r + (1{-}r)L) / T_T$. Round latency shrinks to $T_T$ since drafting is fully overlapped, but effective token yield drops via $L_{\mathrm{par}}$. The numerator is $B \cdot L_{\mathrm{par}}$.

Switching threshold (Eq. 50, Appendix C): $r^ = (\gamma{-}1)LT_D / ((T_T + (\gamma{-}1)T_D)(L{-}1))$. Derived from the crossover $\mathrm{Thr}_{\mathrm{ord}} = \mathrm{Thr}_{\mathrm{par}}$ by cross-multiplying and isolating $r$. When $r > r^$, the acceptance loss from rollback outweighs the latency savings from overlapping. Requires $L > 1$ (holds when $\gamma \geq 2$).

Mode switching (Eq. 5): if $r_n \leq r^$ → parallel; if $r_n > r^$ → ordinary. Applied per-round using the observed batch-level rollback ratio.

Context compression (Eq. 17): $\mathcal{E}(x_{1:S}) = [x_{:\lfloor pS/2\rfloor},\, x_{-\lfloor pS/2\rfloor+1:}]$. Retains prefix and suffix, discards middle. At $p{=}0.1$ this removes 90% of draft-side context, trading acceptance quality for latency to maintain $\gamma T_D < T_T$ under TP-asymmetric deployments.

形式化验证清单 #

  1. Ordinary throughput (Eq. 1 → Eq. 41): sequential latency $T_T + (\gamma{-}1)T_D$ per round, $BL$ tokens per round. Dimensionally consistent (tokens/time). ✓
  2. Parallel accepted-length (Eq. 3/43): two-case mixture. Boundary conditions verified ($r{=}0$: full acceptance; $r{=}1$: minimal). ✓
  3. Parallel throughput (Eq. 4/44): substitutes $L_{\mathrm{par}}$ correctly; round latency $T_T$ consistent with full overlap assumption. ✓
  4. Threshold derivation (Eq. 45–50): algebraic manipulation from $\mathrm{Thr}_{\mathrm{ord}} > \mathrm{Thr}_{\mathrm{par}}$ involves cross-multiplication and collecting $r$ terms. Final expression matches Appendix C step-by-step; holds under $L > 1$. ✓
  5. Overlap precondition ($\gamma T_D < T_T$, Eq. 39): required for parallel mode. Violations handled by speculative priority scheduling (reduces $T_D^{\mathrm{mix}}$), context compression (reduces $T_D$), and fallback to conservative mode (Eq. 37–38). ✓
  6. Compression trade-off: Eq. 17 is a deterministic compression rule. No formal bound on acceptance loss is given — the paper validates empirically that $p{=}0.1$ retains sufficient acceptance under TP8 (§4.3). The acceptance-throughput trade-off is demonstrated but not analytically bounded. ✓

    接受长度 #

    Accepted length analysis (Table 3, not shown) reveals SPECTRE trades 1–18% acceptance versus Standalone (2.59–3.61 vs 3.13–3.77) but significantly outperforms PEARL (1.66–2.19) and EAGLE-3 (1.90–2.44). The hybrid strategy preserves near-Standalone acceptance on most rounds by switching to ordinary mode when rollback would be costly, and gains throughput on low-rollback rounds through parallelism. ShareGPT shows the smallest gap (3.11 vs 3.13 on Qwen3-32B) because conversational workloads produce more predictable continuations with lower rollback rates.

    §7 实现 cross-reference #

    代码: SGLang PR #22272，基于 SGLang v0.5.7。

    关键实现细节:

    1. ZMQ 通信层 (Appendix A.3): 自定义多线程 C++ 扩展实现异步消息传递。Target 端作为 ZMQ ROUTER, 每个 draft server 作为 DEALER endpoint。本机使用 IPC transport, 跨机 TCP。包含心跳机制、批量二进制序列化、有界队列和 stale-message 丢弃。这是区别于 gRPC/HTTP 通信的工程选择 — 在 speculative decoding 的关键路径上，ZMQ 的低延迟对 $\gamma T_D < T_T$ 约束至关重要。
      1. Circuit breaker (Appendix A.1, Eq. 24–27): 异步回复通过 $(i, n)$ 标识验证（request-id + round-id 必须同时匹配）。连续 $C_{\max}$ 次未收到有效 draft 回复时，禁用远程推测 $H$ 轮并回退到单 token 自回归。防止 draft server 故障或网络抖动导致 target 持续空等。

        2. 核心技术壁垒（实现层面）: $r^*$ 的在线计算依赖对 $T_D$, $T_T$, $L$ 的运行时估计。论文未详述这些量的估计策略（滑动窗口大小、EMA vs 均值、冷启动处理等），这一工程选择直接影响模式切换的灵敏度与稳定性 — 估计过于灵敏会导致频繁切换（抖动），过于平滑会导致对突发负载变化的响应滞后。

Attention Residuals

llm   Guangyu Chen, Yu Zhang, Jianlin Su, (et al., 37 authors, Kimi Team)  ·  2026-03  ·  arXiv ↗
residual-connectionattention-mechanismdepth-aggregationkimiscaling-law
TL;DR
PreNorm Transformer 的固定权重残差导致 hidden norm $O(L)$ 膨胀、早期层贡献被稀释；本文提出 Attention Residuals (AttnRes)：用 softmax 沿深度加权聚合所有先前层输出，每层学一个伪 query $\mathbf{w}_\ell$。Block 工程版训练 overhead < 4%，已在 Kimi Linear 48B-A3B 落地，GPQA +7.5。

Core Contribution

把 PreNorm Transformer 的"恒定权重为 1 的残差累加"替换成"沿深度做一次 softmax-attention 加权求和"—— 每层学一个伪 query $\mathbf{w}_\ell$ 对所有先前层输出做 softmax 选择性聚合： $\mathbf{h}_\ell = \sum_i \alpha_{i \to \ell} \mathbf{v}_i$。Block-AttnRes 训练 overhead < 4%， 已用于 Kimi Linear 48B-A3B（GPQA +7.5）。

核心三问

Q1: 问题 — PreNorm 残差用固定权重 1 累加每层输出，导致 hidden norm 随深度 $O(L)$ 膨胀、早期层贡献被稀释、 梯度跨深度衰减。

Q2: 方法 — Attention Residuals (AttnRes)：用 softmax-attention 沿深度加权聚合所有先前层输出。 Block 版本只在 block 边界做，块内仍用 +，训练 overhead < 4%。

Q3: 结果 — 16 层小模型 val loss 1.766 → 1.737；scaling law 节省 1.25× compute； Kimi Linear 48B-A3B 生产部署，GPQA-Diamond +7.5。

Summary

PreNorm Transformer 的标准 forward 是 $\mathbf{h}_\ell = \mathbf{h}_{\ell-1} + f_{\ell-1}(\text{Norm}(\mathbf{h}_{\ell-1}))$， 每层输出以 固定权重 1 累加到 hidden state。后果：(1) 隐状态范数随深度 $O(L)$ 增长（PreNorm dilution）， (2) 早期层贡献被后期反复加权稀释，(3) 梯度信号在反传时跨深度逐渐被强势层稀释。

AttnRes 的核心洞察：用 attention 替代固定权重。Full-AttnRes 在每一层都对所有先前层输出做 softmax 加权求和；Block-AttnRes 是工程化版本——只在 block 边界做（每 block 内部仍用普通 +），并且 attention 实际放在 attn 子层之前 和 MLP 子层之前 各一次。

实验：16 层小模型 val loss baseline 1.766 → AttnRes-Full 1.737；DenseFormer 固定标量 1.767（基本无效）； Block-AttnRes 在 scaling law 上节省 ~1.25× compute。已用于 Kimi Linear 48B-A3B (1.4T tokens, Muon+WSD)， GPQA +7.5；训练 overhead < 4%；I/O 5.5d/token/layer (vs 标准 3d)。

Key Findings

  • 核心方程 $\mathbf{h}_\ell = \sum_i \alpha_{i \to \ell} \mathbf{v}_i$，softmax 取代固定权重 1
  • Block-AttnRes 注入点：每个 transformer 层 2 次（attn 之前 + MLP 之前），block 内部仍标准 +
  • 新参数仅 $2L \cdot d$（每注入点一个 Linear(d,1) 伪 query + 一个 RMSNorm），相对 FFN 可忽略
  • 零初始化：pseudo-query 必须 w=0，开始时退化为均匀平均，避免训练初期分布混乱
  • Softmax > Sigmoid（消融），Single head depth-attention > multi-head（消融）
  • DenseFormer 风格固定标量不 work（baseline 1.766 vs DenseFormer 1.767）
  • Hidden norm 行为：bounded periodic vs baseline 单调增长 → 隐式深度归一化
  • 生产验证：Kimi Linear 48B-A3B 用了，GPQA-Diamond +7.5
  • 训练 overhead < 4%（Block 版 + 在线 softmax 合并 + cache）
  • I/O 增加：每层 ~5.5d/token (vs 标准 3d)，但仍在带宽内可承受

Limitations

  • 大规模验证只有 MoE Kimi Linear 一例；标准 dense 大模型 (e.g. 70B+ dense) 未验证
  • Depth-attention 的 $L^2$ 复杂度：层数翻倍开销翻倍（Block 版本缓解到 $O(L)$ blocks）
  • Block size 选择无理论指导，需要实验调（论文用 block_size=6）
  • I/O 5.5d 在 bandwidth-bound 推理下可能拖慢，论文未给精确 latency 数字
  • 对超大词表 / 多模态融合层的影响未研究

Deep Analysis

1. Architecture (with diagram)

1a. Standard vs AttnRes block (side-by-side)

详见下方互动图（diagram arch-comparison）。左侧标准 PreNorm，右侧 AttnRes：每层有两个 block_attn_res 调用（attn 之前和 MLP 之前），block 内部仍走普通 residual +。 Block 边界由 layer_number % (block_size // 2) == 0 决定。

1b. Math flow

AttnRes 的核心是把 PreNorm 的 h = h + f(Norm(h)) 替换为 h = block_attn_res(blocks, partial) + f(Norm(...))。 Block-AttnRes 是工程化版本：每个 transformer 层做 2 次 AttnRes（attn 之前和 MLP 之前）， block 内部仍走标准 +。

从论文官方 README 提取的精确 PyTorch 伪代码：


def block_attn_res(blocks: list[Tensor], partial_block: Tensor,
                   proj: Linear, norm: RMSNorm) -> Tensor:
    """Inter-block attention: attend over block reps + partial sum."""
    V = torch.stack(blocks + [partial_block])              # [N+1, B, T, D]
    K = norm(V)                                            # RMSNorm
    logits = torch.einsum('d, n b t d -> n b t',
                          proj.weight.squeeze(), K)        # w · K
    h = torch.einsum('n b t, n b t d -> b t d',
                     logits.softmax(0), V)                 # softmax over depth
    return h

每次调用 5 个步骤：

  1. V = stack(blocks + partial) 拼出深度方向所有候选向量
  2. K = RMSNorm(V) 对深度向量做归一
  3. logits = w · K 用伪 query 算每个候选的分数
  4. α = softmax(logits, dim=depth) 沿深度归一为概率
  5. h = Σ α_i · v_i 加权求和得到注入到下一个子层的 input

1c. Forward 全图

下面是 transformer 层完整 forward，把两次 block_attn_res 调用、block 边界判定、 partial block 累加全部组合起来。这是论文之所以能跑出来的核心实现：


def forward(self, blocks, hidden_states):
    partial_block = hidden_states
    # AttnRes #1: before attention
    h = block_attn_res(blocks, partial_block,
                       self.attn_res_proj, self.attn_res_norm)

    # if at block boundary, finalize current block, start new
    if self.layer_number % (self.block_size // 2) == 0:
        blocks.append(partial_block)
        partial_block = None

    # standard self-attention sublayer
    attn_out = self.attn(self.attn_norm(h))
    partial_block = (partial_block + attn_out) if partial_block is not None else attn_out

    # AttnRes #2: before MLP
    h = block_attn_res(blocks, partial_block,
                       self.mlp_res_proj, self.mlp_res_norm)

    # standard MLP sublayer
    mlp_out = self.mlp(self.mlp_norm(h))
    partial_block = partial_block + mlp_out

    return blocks, partial_block

2. Training & Empirical Dynamics

2a. Hidden norm and gradient behavior

  • Output norm vs depth: baseline 单调增长（PreNorm dilution）;

AttnRes bounded + periodic（在 block 边界 reset）

  • Gradient norm vs depth: baseline 衰减（信号被深层吃掉）; AttnRes 更均匀 →

早期层得到更强信号

这两个数值现象是 AttnRes "为什么能 work" 的核心证据：它实质上是一种 隐式深度归一化机制。

2b. Scaling law

论文拟合：

$$L \approx 1.891 \cdot C^{-0.057}$$

Block-AttnRes 在相同 $L$ 下需要 $\sim 0.8 C$，即 节省 ~1.25× compute。

2c. Toy validation (16-layer)

见上方 Table 1「Toy validation (16-layer)」。Standard PreNorm 1.766 → DenseFormer 1.767（几乎无效）→ Block AttnRes ~1.740 → Full AttnRes 1.737。证明 input-dependent 选择（而非固定标量）是 AttnRes work 的关键。

3b. 训练技术

3c. 核心壁垒

最难复现的一件事：Block-AttnRes 的 cache-based pipeline communication。Full-AttnRes 实现简单但 $L^2$ 开销高；Block-AttnRes 需要：

  1. 缓存 N-1 个完成的 block 表示
  2. 与 in-progress partial block 在线合并 (online softmax)
  3. 在 pipeline parallelism 下让 cache 跨 stage 通信不阻塞主路径

这是 < 4% overhead 的关键。论文承诺开源但没公开 pipeline 细节。

在线 softmax 两阶段合并（缓存的块表示 + 进行中的 partial block）：

$$h_\ell = \frac{e^{m_1 - m} \cdot o_1 + e^{m_2 - m} \cdot o_2}{e^{m_1 - m} \cdot \ell_1 + e^{m_2 - m} \cdot \ell_2}$$

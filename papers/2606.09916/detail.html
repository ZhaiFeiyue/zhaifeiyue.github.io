IntentKV: Cross-Turn Intent-Aware KV Cache Pruning for Agent Inference

1) TL;DR #

IntentKV addresses agent inference under long multi-turn trajectories by making KV pruning intent-aware across turns and layout-safe for prefix reuse. The key gain is not only better token retention but also a slot-map eviction design that preserves radix/prefix cache identity; this lets an 8k budget remain close to full-cache accuracy while strongly reducing worst-case KV memory/read pressure.

2) Q1 / Q2 / Q3 #

Q1: 痛点是什么？ #

现有 KV 压缩大多是单提示词（single-prompt）视角：用当前 query 的局部注意力给历史 token 打分，然后把保留 token 做 compaction。论文指出这在 agent 场景有两个核心失配：

  1. 跨 turn 意图漂移：后续查询、工具结果、中间决策会让“早期 token”重新变重要，单次打分会过时。
  2. 可复用性破坏：compaction 会改变 surviving token 的 slot/位置身份，破坏后续 turn 对共享前缀的 radix/prefix cache 命中。

    3. Figure 1 给出上述两种失配：一是“prompt-local score stale”，二是“layout relocation breaks reuse”。这不是单纯算法精度问题，而是 retention 与 serving substrate 的耦合问题。

    Q2: 方法核心是什么？ #

    IntentKV 把问题拆成两条正交轴：保留策略（what to keep） 与 布局策略（how to evict without breaking reuse）。

    • 保留策略：维护跨 turn 的 QueryMemory，按会话累计 query 信号；再用一个零初始化 residual head 在冻结 backbone 上做校正。
    • 布局策略：不做 compaction，把被淘汰位置重定向到 dead slot（sentinel），保留 surviving KV 行、RoPE phase 与 slot identity 不变，从而继续可被 prefix/radix cache 复用。

    核心技术壁垒：在不改 attention kernel 的前提下，同时满足“动态跨-turn相关性建模”与“前缀身份稳定性”。这要求 scorer、slot-map、allocator、prefix cache 协同设计；单独改任何一层都无法复现论文中的准确率/效率共赢。

    Figure 2 展示了规则分数、残差分支和最终 top-k 选择的组合路径，并把 dead-slot eviction 放在同一方法图里，强调“打分 + 布局”是联合设计而不是独立技巧叠加。

    Q3: 结果如何？ #

    在 BCP 上，IntentKV 在 8k 预算（高压区）表现出“接近 full-cache 的准确率 + 明显更低系统成本”的组合：

    • Qwen3-8B：True Acc 接近 full-cache（在 16k 下差距仅 0.12；8k 下明显优于启发式基线）。
    • Qwen2.5-14B：8k 下相对最强启发式有 10.36 点 True Acc 优势。
    • 最坏样本上，峰值请求 token 与 Raw KV Reads 出现 1-2 个数量级缓解（Table 3）。

    关键机制分解 #

    1. 跨 turn 记忆更新：$\mathbf{M}_t=e^{-\lambda}\mathbf{M}_{t-1}+\mathrm{Enc}(\mathbf{q}_t)$，并做单位范数约束，避免记忆范数增长导致 softmax 塌缩。
    2. 规则打分 + 残差修正：$s_j=\mathrm{rule}_j+\alpha\cdot\mathrm{MLP}(\phi_j\oplus c_j)$，残差分支 zero-init，确保初始等价于规则打分下界。
    3. 布局保持：只改 slot map，不搬移 surviving KV 行；淘汰位置映射到 sentinel，使被淘汰 token 在注意力上近似硬掩码。
    4. 别名安全释放：回收物理 slot 前检查 prefix 保护、sentinel 与 alias 共享，避免兄弟请求或已重定向位置被误释放。

      5. Figure 3 证明“跨 turn 注意力漂移”是现实现象而非假设：不同 query turn 激活前缀中的不同区域，支撑 QueryMemory 的必要性。

      核心技术壁垒（实现视角） #

      最难复现点不是单个公式，而是服务栈集成的一致性：要在 paged KV + radix prefix cache 的真实推理路径中引入 dead-slot 重定向、alias-aware free、query-memory 更新与 residual 打分，并确保它们在多并发会话下不破坏 correctness。任何一个环节与 serving runtime 的现有内存管理约定冲突，都会导致 prefix 复用失效或 KV 读错位。

      关键实现细节（易漏点） #

      1. sentinel 数值构造：$K[s^\dagger]=-10^4\mathbf{1}, V[s^\dagger]=0$ 的目标是“在现有内核里等价硬掩码”，不是一般意义的空 token；数值尺度不足会在低精度下泄漏注意力。
      2. zero-init residual with gradient path：输出层零初始化但 $\alpha$ 初始为 1，使 day-0 精确回退到 rule scorer，同时梯度可通过 fc2 学到修正；若把 $\alpha=0$ 与零输出同时使用会形成训练停滞。
      3. 训练标签构造：用未来 5 次工具调用参数做 substring 匹配作为正例，且无匹配样本直接丢弃，优先保证标签精度；这是 residual head 能学到“未来可用证据”而非 recency 偏置的关键。

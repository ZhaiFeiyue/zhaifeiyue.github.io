SLA: Beyond Sparsity in Diffusion Transformers via Fine-Tunable Sparse-Linear Attention

Jintao Zhang, Haoxu Wang, Kai Jiang, Shuo Yang, Kaiwen Zheng, Haocheng Xi, Ziteng Wang, Hongzhou Zhu, Min Zhao, Ion Stoica, Joseph E. Gonzalez, Jun Zhu, Jianfei Chen | 2025-09 | https://arxiv.org/abs/2509.24006 Category: kernel | Tags: sparse-attention, linear-attention, diffusion-transformer, gpu-kernel, video-generation Read: 2026-04-16

Core Contribution

SLA (Sparse-Linear Attention) is a trainable hybrid attention method that fuses sparse and linear attention into a single GPU kernel, achieving 95% attention computation reduction and 13.7x kernel speedup for Diffusion Transformers without degrading generation quality.

核心三问

Q1: 这篇论文试图解决什么核心痛点/问题？

Diffusion Transformer (DiT) 模型中，注意力计算是视频生成的主要瓶颈——序列长度从10K到100K，复杂度为 $O(N^2)$。现有的稀疏注意力方法在中等序列长度下难以超过90%的稀疏度而不损失质量，而线性注意力在视频扩散模型上表现极差，严重降低生成质量。两者都无法单独有效解决这个问题。

Q2: 作者提出了什么新的"杀手锏"方法/架构？

作者提出 SLA（Sparse-Linear Attention），一种可训练的混合注意力方法。核心洞察是：注意力权重可以分解为两部分——少量大权重（高秩）和大量小权重（极低秩）。SLA 将注意力权重动态分为三类：critical（用 $O(N^2)$ FlashAttention 精确计算）、marginal（用 $O(N)$ 线性注意力近似）、negligible（直接跳过）。这三种计算融合在单个 GPU kernel 中，支持前向和反向传播，仅需少量微调即可使用。

Q3: 最终效果/结论如何？

在 Wan2.1-1.3B 视频生成模型上，SLA 将注意力计算量减少 95%（从52.75T FLOPs降至2.74T），注意力 kernel 加速 13.7x，端到端视频生成加速 2.2x（注意力延迟从97秒降至11秒）。在所有 VBench 指标上，SLA 的视频质量与全注意力相当甚至更优，远超所有基线方法。

逻辑故事还原

背景 (Context)

现有高效注意力方法分两派：稀疏注意力只计算部分注意力分数，但在中等序列长度（<50K）下通常只能达到40-60%稀疏度，超过90%就会明显损失质量；线性注意力用特征映射将复杂度降到 $O(N)$，但在视频扩散模型上完全失效，生成质量严重下降。两条路线各自都遇到了瓶颈，无法单独实现注意力的大幅加速。

破局 (Insight)

作者的"灵光一现"：注意力权重可以分解为两个矩阵——少量大权重（<10%）具有高秩，而大量剩余权重（>90%）具有极低秩。

👉 打个比方：想象一个班级的考试成绩分布。少数学霸的分数差异很大且难以预测（高秩），而大多数中间学生的分数则非常接近、容易用简单规律概括（低秩）。SLA 就是对"学霸"精确打分（稀疏注意力），对"中间学生"用简单公式估算（线性注意力），对"分数可忽略"的部分直接跳过。

这个发现解释了为什么单独的稀疏或线性注意力都不够好：稀疏注意力丢掉了大量低秩但非零的权重，线性注意力无法处理高秩部分。

拆解 (Deconstruction)

  1. 预测压缩注意力权重：对 $Q$、$K$ 做 mean pooling，计算块级别的注意力分数 $P_c$
  2. 三分类：将 $P_c$ 的每个元素分为 critical（top $k_h$%）、negligible（bottom $k_l$%）、marginal（其余），记录在压缩掩码 $M_c$ 中
  3. 混合计算：critical 块用 FlashAttention 精确计算 $O(N^2)$，marginal 块用线性注意力 $O(N)$ 近似（预计算 $K^\phi \cdot V$ 和 $\text{rowsum}(K^\phi)$），negligible 块直接跳过
  4. 融合输出：最终输出 $O = O^s + \text{Proj}(O^l)$，其中 $\text{Proj}$ 是可学习的线性变换，弥补 softmax 和线性注意力的分布差异
  5. 微调适配：替换原始注意力为 SLA 后，仅需约2000步微调（不到预训练成本的0.1%）

    关键细节与启示

    技术细节补充：

    1. 线性注意力作为可学习补偿：SLA 中的线性注意力并非近似 marginal 权重的输出，而是作为一种可学习的补偿信号，通过 Proj 层和微调来增强稀疏注意力的表现。这解释了为什么简单的 L+S 相加效果很差——需要端到端微调来学习两者的协作。
    2. 三种稀疏度优化策略：针对不同稀疏模式，SLA 提供了不同的高效实现——高稀疏度（>90%）用 lookup table 避免扫描开销；中等稀疏度（~50%）用 Method of Four Russians 将线性注意力累加开销降至 $1/g$；通用情况用 pre-aggregation（预计算全部累加，再减去非零位置的贡献）。

      3. 一句话总结： SLA 发现注意力权重天然具有"少量高秩 + 大量低秩"的结构，通过将稀疏和线性注意力融合为单一 GPU kernel，仅用5%的注意力计算量和2000步微调，就在视频扩散模型上实现了无损的13.7x注意力加速。

      Summary

      Attention computation is the dominant bottleneck in Diffusion Transformer (DiT) models for video generation, with sequence lengths ranging from 10K to 100K causing quadratic scaling. Existing sparse attention methods cap out at ~60-85% sparsity before quality degrades, while linear attention methods completely fail on video diffusion tasks.

      SLA addresses this by exploiting a key observation: attention weights naturally decompose into a small high-rank component (<10% of weights) and a large extremely low-rank component (>90%). SLA classifies attention weight blocks into three categories — critical (computed exactly via FlashAttention at $O(N^2)$), marginal (approximated via linear attention at $O(N)$), and negligible (skipped entirely). These computations are fused into a single efficient GPU kernel supporting both forward and backward passes, with a learnable projection layer bridging the distribution gap between sparse and linear attention outputs.

      Experiments on Wan2.1-1.3B demonstrate that SLA achieves 95% attention sparsity (reducing computation from 52.75T to 2.74T FLOPs) while matching full attention quality across all VBench metrics. The GPU kernel achieves 13.7x speedup in attention and 2.2x end-to-end speedup, with attention latency dropping from 97s to 11s. Fine-tuning requires only 2000 steps (<0.1% of pretraining cost), making SLA highly practical for deployment.

      Key Findings

      • Attention weights in DiT models decompose into high-rank sparse (<10%) and low-rank dense (>90%) components, explaining why neither sparse nor linear attention alone succeeds
      • SLA achieves 95% attention sparsity on Wan2.1-1.3B (30K sequence length) with no quality degradation, far exceeding the 60-85% typical of sparse-only methods
      • The fused GPU kernel delivers 13.7x attention speedup and 2.2x end-to-end video generation speedup on RTX5090
      • Linear attention alone completely fails on video diffusion (VA: 0.042 vs 76.78 for full attention), but works as a learnable compensation within SLA's hybrid framework
      • Only 2000 fine-tuning steps needed — less than 0.1% of pretraining cost
      • Softmax outperforms ELU+1 and Hedgehog as the feature map $\phi$ in the linear attention component

      Limitations

      • Evaluated only on relatively small models (Wan2.1-1.3B, LightningDiT-1.0B); scalability to larger models (14B+) is unknown
      • Only tested on RTX5090 (Blackwell); performance on other GPU architectures (A100, H100, AMD MI300) not reported
      • The three-category classification uses fixed percentile thresholds (kh=5%, kl=10%); adaptive or learned thresholds might improve results
      • Requires fine-tuning (2000 steps) — not a training-free drop-in replacement
      • The "stable rank" decomposition analysis is empirical; theoretical guarantees on when this decomposition holds are lacking
      • Linear attention as "learnable compensation" is not well understood theoretically — the paper acknowledges it doesn't approximate the marginal weights but rather serves as an adaptive correction

      Infrastructure Impact

      • Algorithm: Provides a new paradigm for combining sparse and low-rank attention that could generalize beyond diffusion to autoregressive LLMs with long contexts
      • Kernel: Introduces a novel fused kernel pattern (sparse + linear attention in one pass) with three optimization strategies (lookup table, pre-aggregation, Method of Four Russians) applicable to other sparse kernel designs
      • Framework: Enables practical deployment of DiT video generation models by reducing attention from the bottleneck to near-negligible cost; framework integration needs custom kernel compilation
      • LLM: The sparse-linear decomposition insight may apply to long-context LLM attention, though causal masking adds complexity not addressed here
      • Agent: 2.2x end-to-end video generation speedup could enable more responsive AI video generation in interactive applications

      Deep Analysis (kernel)

      1. Target Operation

      • Computational primitive: Attention (softmax attention with hybrid sparse + linear attention computation)
      • Input/output shapes: $Q, K, V \in \mathbb{R}^{N \times d}$ where $N \approx 30\text{K}$ (Wan2.1-1.3B), $d$ = head dimension. Also operates on $Q^\phi = \phi(Q)$, $K^\phi = \phi(K)$ for the linear attention component.
      • Data types: The paper does not explicitly specify precision; given it targets RTX5090 and uses FlashAttention2, likely BF16 or FP16.
      • Theoretical arithmetic intensity: Full attention is $O(N^2 d)$ FLOPs with $O(Nd)$ bytes → arithmetic intensity = $O(N)$, making it heavily compute-bound for large $N$. SLA reduces effective FLOPs by ~20x (from 52.75T to 2.74T) while maintaining similar memory access patterns for the sparse component, shifting the balance toward memory-bound for the linear attention component.
      • Roofline position: The sparse attention component (5% of blocks) remains compute-bound. The linear attention component ($O(Nd^2)$) is compute-bound when $d$ is large. The overall kernel is likely compute-bound but with significantly better utilization due to skipping 95% of the quadratic computation.

      2. Hardware Model

      • Target GPU: NVIDIA RTX5090 (Blackwell consumer GPU)
      • Hardware units: Tensor Cores for the GEMM-like operations in both sparse attention ($Q \cdot K^\top$, $P \cdot V$) and linear attention ($(K^\phi)^\top \cdot V$, $Q^\phi \cdot H$). SFU potentially for softmax/exp operations.
      • Memory hierarchy:
      • Registers: Store running max $m_{ij}$, running sum $l_{ij}$, and partial output $O^s$ per query block (standard FlashAttention register usage)
      • Shared memory: $Q$ blocks ($b_q \times d$) and $K$/$V$ blocks ($b_{kv} \times d$) loaded into shared memory. For linear attention, $h_j$ and $z_j$ pre-computed values accumulated
      • L2 cache: Lookup tables for sparse mask access, pre-aggregated $H_i$, $Z_i$ values
      • HBM: $Q$, $K$, $V$, $Q^\phi$, $K^\phi$, $M_c$ (compressed mask), output $O^s$, $O^l$
      • Peak throughput: On RTX5090, FlashAttention2 achieves the baseline throughput. SLA's forward pass achieves 13.7x of FlashAttention2's FLOPS (reported as effective FLOPS = full_attention_FLOPs / actual_time), meaning SLA processes the equivalent computation in 1/13.7 the time.

      Additional optimization techniques specific to SLA:

      TechniqueWhen AppliedDetails
      Lookup tableSparsity > 90%Pre-process nonzero positions of $M_c$ into a lookup table. Avoids scanning entire rows/columns to read mask values, reducing memory traffic.
      Pre-aggregationSparsity > 90%Precompute $\sum_j h_j$ and $\sum_j z_j$ (full sums), then subtract contributions where $M_c[i,j] \neq 0$. Replaces ~90% additions with ~10% subtractions.
      Method of Four RussiansSparsity ~50%Group $h_j$, $z_j$ into segments of $g$ consecutive blocks. Precompute all $2^g$ possible subset sums. Look up required subset in $O(1)$ instead of $O(g)$. Theoretical $1/g$ computation reduction.

      4. Performance Analysis

      • % of theoretical peak: The paper reports SLA achieves 13.7x speedup over FlashAttention2 in forward attention kernel. Since FlashAttention2 is already highly optimized (typically 50-70% of Tensor Core peak), this means SLA is not "faster per FLOP" but rather eliminates 95% of FLOPs while maintaining high hardware utilization for the remaining 5%.
      • Performance vs vendor library: Forward: 13.7x vs FlashAttention2, 1.93x vs VSA@95%, 3.36x vs VMoBa@95%. Backward: 6.8x vs FlashAttention2.
      • Problem size scaling: Demonstrated at $N = 30\text{K}$ (Wan2.1-1.3B). The paper notes that achieving high sparsity is easier at longer sequences (100K-300K), so SLA's advantage of working well at moderate 30K is notable. However, sub-30K performance is not reported.
      • Latency breakdown: Attention drops from 97s to 11s (8.8x). End-to-end drops from 185s to 88s (2.2x). The remaining 77s is non-attention computation that SLA does not address.
      • Backward pass gap: Backward achieves only 6.8x (vs 13.7x forward). This is typical — backward passes have more memory traffic (need to reload $Q$, $K$, $V$, plus output gradients) and the sparse component's backward involves recomputation of $P_{ij}$.

      5. Numerical Considerations

      • Accuracy vs baseline: On all 7 VBench metrics, SLA@95% matches or exceeds Full Attention quality. The paper reports <3% $L_1$ error when skipping the bottom 45% of weights, suggesting the approach is numerically sound.
      • Numerical stability: The sparse component uses OnlineSoftmax (Milakov & Gimelshein, 2018), which computes softmax in a numerically stable block-wise manner using running max rescaling (Lines 10-11 of Algorithm 1). This avoids overflow/underflow in the exponential.
      • Linear attention normalization: Output $O^l = Q^\phi \cdot H_i / (Q^\phi \cdot Z_i)$ uses element-wise division for normalization. The paper uses softmax as $\phi$ (which produces non-negative values), avoiding division-by-zero issues.
      • Distribution mismatch: The Proj layer between $O^l$ and the final output helps bridge the distributional gap between softmax-based sparse attention and the linear attention approximation. This is critical for training stability.

      8a. Core Kernel Source Analysis

      Source code is available at https://github.com/thu-ml/SLA but detailed kernel source was not analyzed during this reading. The paper provides two complete pseudocode algorithms (Algorithm 1: Forward, Algorithm 2: Backward).

      8b. Pseudocode Annotation — Algorithm 1 (Forward Pass)

      
// [Line 2-3] Block partitioning — GRID-LEVEL setup
// Q, K, V divided into T_m and T_n blocks of size b_q × d and b_kv × d
// Each thread block handles one Q block (rows) against all K/V blocks
// GPU mapping: gridDim.x = T_m, each block processes one query chunk
Divide Q,Qϕ into T_m blocks; Divide K,V,Kϕ into T_n blocks

// [Line 4] LINEAR ATTENTION PRECOMPUTATION — GLOBAL MEMORY → REGISTERS/SMEM
// h_j = Kϕ_j^T · V_j is a d×d matrix; z_j = rowsum(Kϕ_j^T) is d×1
// These are computed ONCE and reused across all query blocks
// This is the KEY optimization: O(T_n · d²) precomputation avoids
// O(T_m · T_n · d²) repeated computation in the main loop
// Memory: h_j stays in HBM, loaded per-row into shared memory as needed
h = {h_j} = {(Kϕ_j)^T · V_j}; z = {z_j} = {rowsum((Kϕ_j)^T)}

// [Line 5] MASK PREDICTION — lightweight O(T_m · T_n · d) computation
// Mean pooling reduces N×d → (N/b_q)×d, making mask prediction cheap
// Softmax on the compressed scores gives block-level attention distribution
// Memory: P_c is tiny (T_m × T_n), fits in shared memory
P_c = Softmax(pool(Q)·pool(K)^T/√d); Initialize M_c = 0

// [Line 6] THREE-WAY CLASSIFICATION — TopK + BottomK on each row
// Top k_h% → critical (1), Bottom k_l% → negligible (-1), rest → marginal (0)
// GPU: This is a sort/select operation on small T_n-length vectors per row
M_c[i,j] = 1 if in TopK(P_c[i,:], k_h); M_c[i,j] = -1 if in BottomK(P_c[i,:], k_l)

// [Line 7] OUTER LOOP — one iteration per query block
// GPU: Each thread block handles one i; all blocks run in parallel
for i = 1 to T_m:

  // [Line 8] INNER LOOP — iterate over all K/V blocks
  // GPU: Sequential within each thread block (like FlashAttention)
  for j = 1 to T_n:

    // [Line 9-11] SPARSE PATH (COMPUTE-BOUND) — FlashAttention with OnlineSoftmax
    // Only entered for top k_h% = 5% of blocks
    // S_ij = Q_i · K_j^T/√d is a b_q × b_kv GEMM → Tensor Core
    // OnlineSoftmax: track running max m_ij and running sum l_ij
    // Output accumulation with rescaling: O^s += exp(m_old - m_new) · O^s_old + P·V
    // Memory: Q_i in shared mem (persistent), K_j loaded from HBM, V_j loaded from HBM
    // This is identical to FlashAttention2's inner loop
    if M_c[i,j] == 1:
      S_ij = Q_i · K_j^T/√d; P_ij = OnlineSoftmax(S_ij)
      O^s_ij = rescale(O^s_{i,j-1}) + P_ij · V_j

    // [Line 12-13] LINEAR PATH (MEMORY-BOUND) — Single matrix addition
    // Only entered for marginal blocks (neither top nor bottom)
    // H_i += h_j is just accumulating pre-computed d×d matrices
    // Z_i += z_j is accumulating d×1 vectors
    // This is EXTREMELY cheap: one d×d addition per block
    // With pre-aggregation optimization: only ~10% subtractions needed
    // Memory: h_j loaded from HBM/L2, H_i accumulated in registers/shared mem
    else if M_c[i,j] == 0:
      H_i += h_j; Z_i += z_j

    // [Implicit: M_c[i,j] == -1] SKIP — no computation at all

  // [Line 16] FINALIZATION — normalize and compute linear output
  // O^s normalized by 1/l (standard FlashAttention finalization)
  // O^l = Qϕ_i · H_i / (Qϕ_i · Z_i) — two d×d and d×1 GEMMs
  // L_i = log-sum-exp values saved for backward pass
  O^s_i = diag(l_i)^{-1} · O^s_{i,Tn}
  O^l_i = Qϕ_i · H_i / (Qϕ_i · Z_i)
  L_i = m_{i,Tn} + log(l_{i,Tn})

      Key insight: The inner loop (Lines 8-14) is the core of the fused kernel. The branch on $M_c[i,j]$ determines whether to execute the expensive GEMM-based sparse attention or the cheap matrix-addition-based linear attention accumulation. Since only 5% of iterations take the sparse path, the average cost per iteration is dramatically lower than FlashAttention's constant-cost iteration.

      8c. Data Layout & Memory Access Patterns

      • Input tensor layout: Standard row-major ($N \times d$) layout for $Q$, $K$, $V$, assumed contiguous. Block partitioning is along the $N$ (token) dimension.
      • Memory coalescing: Within each block, threads process rows of the $b_q \times b_{kv}$ score matrix. $K$/$V$ loads from HBM are coalesced when threads in a warp access consecutive elements along the $d$ dimension.
      • Shared memory usage: Each thread block needs:
      • $Q_i$: $b_q \times d = 64 \times d$ (persistent in shared memory)
      • $K_j$, $V_j$: $b_{kv} \times d = 64 \times d$ each (loaded per iteration)
      • $H_i$: $d \times d$ (accumulated, can be large — $128 \times 128 = 16\text{K}$ elements in FP16 = 32KB)
      • $Z_i$: $d \times 1$ (small)
      • The $H_i$ accumulator is the main additional shared memory cost vs. standard FlashAttention. For $d = 128$, $H_i$ requires 32KB in FP16, which is significant on GPUs with 48-96KB shared memory per SM.
      • Register allocation: The kernel maintains both FlashAttention's running state ($m$, $l$, $O^s$ partial) AND linear attention's accumulator state ($H_i$, $Z_i$). This roughly doubles the register pressure vs. FlashAttention alone, potentially reducing occupancy.
      • Bank conflict analysis: Not discussed in the paper. The $d \times d$ matrix $H_i$ accumulation pattern may cause bank conflicts if $d$ is a power of 2 and accessed column-wise. Swizzling may be needed.

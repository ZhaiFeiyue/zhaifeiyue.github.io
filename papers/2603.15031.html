Attention Residuals

LLM   Guangyu Chen, Yu Zhang, Jianlin Su, Weixin Xu, Siyuan Pan  ·  2026-03  ·  arXiv ↗
algorithm
TL;DR
把 PreNorm 的恒定权重残差累加替换为'softmax-attention 沿深度加权求和'：每层学一个伪 query w_ℓ 对所有先前层输出 v_i 做 softmax 选择性聚合 h_ℓ = Σ α_{i→ℓ} v_i。Block-AttnRes 训练 overhead < 4%, 已用于 Kimi Linear 48B-A3B (GPQA +7.5)。

3 页：(1) 标准 PreNorm block 与 AttnRes block 并排对比；(2) 数学流程 + 块合并；(3) 训练动态、消融、生产部署。

Core Contribution

把 PreNorm Transformer 的 "恒定权重为 1 的残差累加" 替换成 "沿深度做一次 softmax-attention 加权求和"。每层都学一个伪 query \(\mathbf{w}_\ell \in \mathbb{R}^d\)，对所有先前层输出 \(\mathbf{v}_i\) 做 softmax 选择性聚合：

$$\mathbf{h}_\ell = \sum_{i=0}^{\ell-1} \alpha_{i \to \ell} \cdot \mathbf{v}_i, \quad \alpha_{i \to \ell} = \frac{\exp(\mathbf{w}_\ell^\top \, \text{RMSNorm}(\mathbf{k}_i))}{\sum_j \exp(\mathbf{w}_\ell^\top \, \text{RMSNorm}(\mathbf{k}_j))}$$

直觉是 "时间-深度对偶 (Time-Depth Duality)"：标准残差 = 沿深度的 RNN（固定单元权重为 1），AttnRes = 用 attention 替代深度 RNN，让深层有 input-dependent 的方式选择从哪些早期层吸取信息。Block-AttnRes 把替换降到 block 边界（每 block_size//2 层做一次），训练 overhead < 4%。

Summary

PreNorm Transformer 的标准 forward 是 \(\mathbf{h}_\ell = \mathbf{h}_{\ell-1} + f_{\ell-1}(\text{Norm}(\mathbf{h}_{\ell-1}))\)，每层输出以 固定权重 1 累加到 hidden state。后果：(1) 隐状态范数随深度 \(O(L)\) 增长（PreNorm dilution），(2) 早期层贡献被后期反复加权稀释，(3) 梯度信号在反传时跨深度逐渐被强势层稀释。

AttnRes 的核心洞察：用 attention 替代固定权重。Full-AttnRes 在每一层都对所有先前层输出做 softmax 加权求和；Block-AttnRes 是工程化版本——只在 block 边界做（每 block 内部仍用普通 +），并且 attention 实际放在 attn 子层之前 和 MLP 子层之前 各一次。

数学上每个注入点引入 一个学习参数 \(\mathbf{w}_\ell\)（实现为 Linear(d, 1)），加 一个 RMSNorm，pseudo-query 零初始化保证开始时退化为均匀平均。Block-AttnRes 还需要 两阶段在线 softmax 合并（缓存的块表示 + 进行中的 partial block）：

$$h_\ell = \frac{e^{m_1 - m} \cdot o_1 + e^{m_2 - m} \cdot o_2}{e^{m_1 - m} \cdot \ell_1 + e^{m_2 - m} \cdot \ell_2}$$

实验上：

  • 16 层小模型：val loss baseline 1.766 → AttnRes-Full 1.737 (-0.029)，DenseFormer (固定标量) 1.767（基本无效）
  • Scaling law：Block-AttnRes 在相同 loss 下 节省 ~1.25× compute
  • 已用于 Kimi-Linear (48B-A3B, 1.4T tokens, Muon + WSD)，GPQA +7.5
  • 训练 overhead < 4%（Block 版）；I/O 5.5d / token / layer (vs 标准 3d)

观察到的训练动态：output norm 在 baseline 单调增长，AttnRes 呈 bounded periodic pattern（每个 block 边界 reset）；梯度 norm 在 baseline 沿深度衰减，AttnRes 更均匀。

Key Findings

  • 核心方程：\(\mathbf{h}_\ell = \sum_i \alpha_{i \to \ell} \mathbf{v}_i\)，softmax 取代固定权重 1
  • Block-AttnRes 注入点：每个 transformer 层 两次（attn 之前 + MLP 之前），block 内部仍标准 +
  • Block 划分：每 block_size // 2 层一个 block 边界（每个 transformer 层 = 2 sublayers）
  • 新参数：每注入点 一个 Linear(d, 1) 伪 query + 一个 RMSNorm；总 ≈ 2L · d，相对 FFN 可忽略
  • 零初始化：pseudo-query 必须 zero-init，开始时退化为均匀平均，避免训练初期分布混乱
  • Softmax > Sigmoid（消融）；单 head depth-attention > multi-head（消融）
  • DenseFormer 风格固定标量不 work（Table 4 baseline 1.766 vs DenseFormer 1.767）
  • 生产验证：Kimi Linear 48B-A3B 用了，GPQA +7.5
  • 训练 overhead < 4%（Block 版本 + 在线 softmax 合并 + cache）
  • I/O 增加：每层 ~5.5d / token (vs 标准 3d)，但仍在带宽内可承受
  • Hidden norm 行为：bounded periodic vs baseline 单调增长 → 暗示 AttnRes 具有 stabilizing 效果

Limitations

  • 大规模验证只有 MoE Kimi Linear 一例；标准 dense 大模型 (e.g. 70B+ dense) 未验证
  • Depth-attention 的 \(L^2\) 复杂度：层数翻倍，AttnRes 开销也翻倍（虽然 Block 版本缓解到 \(O(L)\) blocks）
  • Block size 选择无理论指导，需要实验调（论文用 block_size=6 for Kimi Linear）
  • I/O 5.5d 在 bandwidth-bound 推理下可能拖慢，论文未给精确 latency 数字
  • 对超大词表 / 多模态融合层的影响未研究

Infrastructure Impact

  • Algorithm: 替代 PreNorm 残差的新机制，drop-in 可用
  • Kernel: 需要 fused 在线 softmax merge kernel（已开源）
  • Framework: 推理需要缓存 block 输出（轻微 KV-like 缓存）
  • LLM: 已被 Kimi Linear / K2.5 采用；可能成为下一代 LLM 的标配
  • Hardware: I/O 增加 ~1.8×，对 memory-bandwidth bound 推理略不利但训练影响小

1b. Math flow

💡 滚轮缩放 · 拖动平移 · 顶部页签切换 · 下载 .drawio

精确流程（从官方 README 的 PyTorch 伪代码提取）：


def block_attn_res(blocks: list[Tensor], partial_block: Tensor,
                   proj: Linear, norm: RMSNorm) -> Tensor:
    """Inter-block attention: attend over block reps + partial sum."""
    V = torch.stack(blocks + [partial_block])              # [N+1, B, T, D]
    K = norm(V)                                            # RMSNorm
    logits = torch.einsum('d, n b t d -> n b t',
                          proj.weight.squeeze(), K)        # w · K
    h = torch.einsum('n b t, n b t d -> b t d',
                     logits.softmax(0), V)                 # softmax over depth
    return h

每次调用：

1. V = stack(blocks + partial) 拼出深度方向所有候选向量

2. K = RMSNorm(V) 对深度向量做归一

3. logits = w · K 用伪 query 算每个候选的分数

4. α = softmax(logits, dim=depth) 沿深度归一为概率

5. h = Σ α_i · v_i 加权求和得到注入到下一个子层的 input

1c. Forward 全图


def forward(self, blocks, hidden_states):
    partial_block = hidden_states
    # AttnRes #1: before attention
    h = block_attn_res(blocks, partial_block,
                       self.attn_res_proj, self.attn_res_norm)

    # if at block boundary, finalize current block, start new
    if self.layer_number % (self.block_size // 2) == 0:
        blocks.append(partial_block)
        partial_block = None

    # standard self-attention sublayer
    attn_out = self.attn(self.attn_norm(h))
    partial_block = (partial_block + attn_out) if partial_block is not None else attn_out

    # AttnRes #2: before MLP
    h = block_attn_res(blocks, partial_block,
                       self.mlp_res_proj, self.mlp_res_norm)

    # standard MLP sublayer
    mlp_out = self.mlp(self.mlp_norm(h))
    partial_block = partial_block + mlp_out

    return blocks, partial_block

2. Training & Empirical Dynamics

💡 滚轮缩放 · 拖动平移 · 顶部页签切换 · 下载 .drawio

2a. Hidden norm and gradient behavior

  • Output norm vs depth: baseline 单调增长 (PreNorm dilution); AttnRes bounded + periodic (在 block 边界 reset)
  • Gradient norm vs depth: baseline 衰减（信号被深层吃掉）; AttnRes 更均匀 → 早期层得到更强信号

这两个数值现象是 AttnRes "为什么能 work" 的核心证据：它实质上是一种 隐式深度归一化机制。

2b. Scaling law

论文拟合：

$$\text{baseline}: \quad L \approx 1.891 \cdot C^{-0.057}$$

Block-AttnRes 在相同 \(L\) 下需要 \(\sim 0.8 C\)，即 节省 ~1.25× compute。

3c. 核心壁垒

最难复现的一件事：Block-AttnRes 的 cache-based pipeline communication。Full-AttnRes 实现简单但 \(L^2\) 开销高；Block-AttnRes 需要 (1) 缓存 N-1 个完成的 block 表示，(2) 与 in-progress partial block 在线合并 (online softmax)，(3) 在 pipeline parallelism 下让 cache 跨 stage 通信不阻塞主路径——这是 < 4% overhead 的关键。论文承诺开源但没公开 pipeline 细节。

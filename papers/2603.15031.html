Attention Residuals

LLM   Guangyu Chen, Yu Zhang, Jianlin Su, Weixin Xu, Siyuan Pan  ·  2026-03  ·  arXiv ↗
algorithm
Summary

Core Contribution

提出Attention Residuals (AttnRes)——用softmax attention替代固定权重残差连接，让每层通过input-dependent learned weights选择性聚合前层表示，解决PreNorm的hidden-state膨胀和层贡献稀释问题。

Summary

现代LLM的PreNorm残差连接以固定unit weight累积所有层输出：h_l = h_{l-1} + f_{l-1}(h_{l-1})，导致hidden state随深度O(L)增长，逐渐稀释每层贡献。论文提出核心洞察——"Time-Depth Duality"：残差连接在深度维度上传播信息如同RNN在时间维度上传播，正如Transformer用attention替代了temporal recurrence，AttnRes用attention替代depth-wise recurrence。

具体公式：h_l = Σ α_{i→l} · v_i，其中α通过softmax attention计算：α_{i→l} = exp(w_l^T RMSNorm(k_i)) / Z，w_l是每层learnable pseudo-query (唯一新参数)，k_i = v_i = f_i(h_i)。Block AttnRes将L层分成N个block，在block级别做attention，memory从O(Ld)降至O(Nd)。配合cache-based pipeline communication和two-phase computation，实训练overhead < 4%。

在Kimi Linear (48B/3B)上训练1.4T tokens，AttnRes使output magnitude呈bounded periodic pattern（而非monotonic growth），gradient分布更均匀。下游任务全面提升：GPQA-Diamond +7.5, Math +3.6, HumanEval +3.1。Scaling law确认1.25x compute-equivalent advantage。

Key Findings

  • Content-dependent聚合关键：DenseFormer (fixed scalar weights)无提升(1.767 vs baseline 1.766)，AttnRes显著提升(1.737)
  • Softmax > Sigmoid (1.737 vs 1.741)——competitive normalization强制sharper selection
  • Multihead depth attention有害 (1.752 > single-head 1.746)——最优depth-wise mixture对所有channel一致
  • Block AttnRes (N≈8)几乎恢复Full AttnRes全部收益 (528M scale差距仅0.001)
  • AttnRes偏好更深更窄架构：最优d_model/L_b从60→45
  • Multi-step reasoning任务收益最大：GPQA +7.5, Math +3.6

Limitations

  • 新增L²级别attention计算（虽然L<1000，compute可接受）
  • Block大小是超参数，需要调优
  • Pipeline parallelism下仍有<4% overhead
  • 仅在MoE架构上验证

Infrastructure Impact

  • Algorithm: Time-depth duality形式化，attention替代residual是新方向
  • Kernel: N/A — 使用标准矩阵乘法
  • Framework: 需支持cross-layer communication，PP需cache-based策略
  • LLM: 标准residual是性能瓶颈之一，Block AttnRes是drop-in替换
  • Agent: 间接受益于模型质量提升

Deep Analysis (llm)

1. Architecture

#### 1a. Architecture Diagram


Layer 0 (Token Embedding): h_1
    │
    ├── f_1: Attention + MoE FFN
    │   output: f_1(h_1) → stored as v_1
    │
    ├── AttnRes: h_2 = Σ α_{i→2} · v_i    ← softmax over {h_1, f_1(h_1)}
    │
    ├── f_2: Attention + MoE FFN
    │   output: f_2(h_2) → stored as v_2
    │
    ├── AttnRes: h_3 = Σ α_{i→3} · v_i    ← softmax over {h_1, v_1, v_2}
    │
    ...
    
Block AttnRes (N=8 blocks, S=6 layers per block):
    
    Block 0: layers 1-6
    │  b_0 = Σ f_j(h_j) for j in block 0
    │
    Block 1: layers 7-12
    │  h_7 = AttnRes(q_7, [h_1, b_0])              ← inter-block attention
    │  h_8 = AttnRes(q_8, [h_1, b_0, f_7(h_7)])    ← + intra-block partial sum
    │  ...
    │  b_1 = Σ f_j(h_j) for j in block 1
    │
    ...

#### 1b. Key Innovation: Pseudo-Query

每层唯一新参数是一个d维learnable vector w_l (pseudo-query)。初始化为零——确保训练开始时所有attention weights均匀，等价于equal-weight average。这使AttnRes在训练初期不干扰标准残差行为，逐渐学习选择性。

#### 1c. Two-Phase Computation

Phase 1 (Parallel): 一个block内所有S个query batched成单次MatMul，对cached block representations做inter-block attention。返回outputs + softmax statistics (max, log-sum-exp)。

Phase 2 (Sequential): 逐层计算intra-block attention (基于evolving partial sum)，通过online softmax与Phase 1 merge：

h_l = (exp(m1-m)·o1 + exp(m2-m)·o2) / (exp(m1-m)·ℓ1 + exp(m2-m)·ℓ2)

2. Scale & Training

  • 48B/3B MoE, 27 Transformer blocks (54 layers), 9 blocks + embedding = 10 sources
  • Block size S=6, N=9 blocks
  • Muon optimizer, WSD lr schedule
  • Two stages: 1T WSD pretrain + 400B mid-training on high-quality data
  • Context: 4K → 32K (no YaRN needed due to NoPE)

3. Core Technical Barrier

核心壁垒是"zero-initialization of pseudo-queries"。这个看似trivial的初始化选择实际上是方案能work的根本原因：

  • 零初始化使初始attention weights均匀 → AttnRes退化为equal-weight average
  • 训练从"安全"起点出发，逐渐学习选择性
  • 如果随机初始化，早期训练会因为random selective aggregation导致instability
  • 这类似于LoRA的zero-initialization trick——确保新模块初始时不改变原模型行为

4. Challenge Assumptions

  • 假设1: "所有层的output应该被选择性聚合" — 是否某些层的output天然就应该被all后续层使用？Figure 8的attention heatmap显示token embedding确实被广泛关注。
  • 假设2: "Block-level粒度足够" — Block内部的层无法被单独选择，可能损失信息。但实验显示S=2-8性能接近。
  • 假设3: "Depth-wise attention不需要multihead" — 论文显示multihead反而有害。这是否意味着depth-wise mixing的最优策略对所有feature channel一致？

5. Design Binding Critique

  • 绑定PreNorm: AttnRes假设PreNorm架构。PostNorm不存在hidden-state growth问题。
  • 绑定Learnable query: 使用fixed learnable vector而非input-dependent query (后者+0.006但需d×d projection)。这限制了per-token的depth selection能力。
  • 绑定Block partitioning: Blocks是等大的固定分组。自适应block大小（如attention层 vs FFN层不同block策略）未被探索。

6. Ecosystem Influence

  • AttnRes已被集成到Kimi Linear中，并在K2.5的post-training中使用
  • 代码开源在 https://github.com/MoonshotAI/Attention-Residuals
  • 与DenseFormer、Highway Networks形成理论统一框架（Figure 9的structured matrix view）
  • 可能影响后续模型的residual connection设计

7. Position in Landscape

  • 首个将attention机制应用于depth-wise aggregation的方案
  • 解决了PreNorm的根本问题——fixed unit weight accumulation
  • Block AttnRes实现<4%训练overhead，是practical的drop-in replacement
  • 对"deeper is better"提供了新的理论依据——AttnRes让深层模型更能利用depth

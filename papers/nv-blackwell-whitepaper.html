Core Contribution

双Die统一GPU突破光刻极限(NV-HBI 10TB/s)，FP4 Tensor Core + 第二代Transformer Engine，NVLink5(1.8TB/s) + NVL72机架级GPU域，推理能效比H100提升25x。

Summary

Blackwell是NVIDIA继Hopper之后的重大架构飞跃，采用TSMC 4NP工艺，创造性地将两个reticle-limited die通过10TB/s NV-HBI(NVIDIA High Bandwidth Interface)连接为统一的逻辑GPU，总计208B晶体管——是Hopper的2.6倍。

核心计算创新包括：(1)第五代Tensor Core支持FP4精度和微缩放(micro-tensor scaling)格式，FP4峰值估计约10 PFLOPS；(2)第二代Transformer Engine支持FP4/FP8/FP16三级动态精度选择。互联方面：(3)NVLink 5每GPU 1.8TB/s(14x PCIe Gen5)；(4)NVL72系统将36个Grace Blackwell Superchip(72 GPU + 36 Grace CPU)组成机架级NVLink域，提供130TB/s全局带宽和13.5TB总显存。

其他创新：解压缩引擎(800GB/s硬件解压)加速数据分析；机密计算TEE扩展到GPU支持NVLink加密。192GB HBM3e提供8TB/s带宽。GB200 Superchip通过NVLink-C2C(900GB/s)连接2个Blackwell GPU和1个Grace CPU。

Key Findings

  • 双Die设计: 两个reticle-limited die通过NV-HBI 10TB/s互联为统一GPU
  • 208B晶体管, TSMC 4NP，2.6x Hopper
  • 第五代Tensor Core: FP4 + 微缩放格式支持
  • 第二代Transformer Engine: FP4/FP8/FP16动态精度
  • NVLink 5: 18 links, 1.8TB/s per GPU (14x PCIe Gen5)
  • NVL72: 72-GPU机架级NVLink域, 130TB/s全局带宽
  • 解压缩引擎: 800GB/s硬件解压
  • 192GB HBM3e, 8.0TB/s
  • GB200 Superchip: 2 Blackwell GPU + 1 Grace CPU (NVLink-C2C 900GB/s)
  • B200 vs H200: 1.56x混合精度吞吐, 42%更好能效, 58%降低内存延迟

Limitations

  • 双die增加封装复杂度和成本
  • NVL72需要全液冷机架基础设施
  • 功耗高(1000W per GPU, NVL72整柜120kW+)
  • 技术简报未公开完整SM/TC规格

Infrastructure Impact

  • Algorithm: FP4训练/推理开辟新精度研究方向
  • Kernel: 双die需要考虑die间通信和数据布局优化
  • Framework: NVL72需要新的大规模并行策略和内存管理
  • LLM: NVL72的13.5TB显存可服务超大模型(万亿参数)
  • Agent: 推理能效比H100提升25x，大幅降低每token成本
  • Cluster: NVL72改变了集群设计范式——从节点级到机架级

3. NVL72 — 机架级GPU域

NVL72是Blackwell最具革命性的系统级创新：36个Grace Blackwell Superchip(每个含2 Blackwell GPU + 1 Grace CPU)通过NVLink Switch连接成72-GPU NVLink域，提供130TB/s的all-to-all带宽。

这本质上创造了一个"virtual giant GPU"抽象——72个GPU共享13.5TB显存，对于万亿参数模型的训练和推理具有变革意义。系统采用全液冷设计，整柜功耗约120kW+。

4. Ops:Byte Analysis


FP8 Ops:Byte = ~5,000 TOPS / 8.0 TB/s ≈ 625 Ops/Byte
FP4 Ops:Byte = ~10,000 TOPS / 8.0 TB/s ≈ 1,250 Ops/Byte

FP4的极高Ops:Byte(1250)意味着几乎所有workload在FP4下都是memory-bound。HBM带宽和缓存层次成为FP4性能的瓶颈。

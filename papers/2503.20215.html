Qwen2.5-Omni Technical Report

multimodalomni-modelspeech-generationthinker-talkerstreaming
Summary

Core Contribution

首次提出 Thinker-Talker 架构，构建端到端多模态模型 Qwen2.5-Omni，能同时感知 text/image/audio/video 并以流式方式同步生成 text 和自然语音。引入 TMRoPE 对齐音视频时间戳，通过 sliding-window DiT 实现流式语音合成。

Summary

Qwen2.5-Omni 是一个端到端多模态模型，能够接受 text、image、audio、video 四种模态输入，并同时生成 text 和自然语音输出。为实现流式多模态输入处理，audio 和 visual encoder 均采用 block-wise 处理方式。为同步 video 和 audio 的时间戳，采用 interleaved 排列并提出 TMRoPE（Time-aligned Multimodal RoPE）位置编码。

核心架构创新是 Thinker-Talker 框架：Thinker 作为 LLM 负责文本生成，Talker 是一个双轨自回归模型，直接利用 Thinker 的 hidden representation 来产生 audio tokens。两者端到端训练和推理。为实现流式 audio token 解码，引入 sliding-window DiT 限制感受野以降低首包延迟。

实验表明 Qwen2.5-Omni 与同等规模的 Qwen2.5-VL 相当，超越 Qwen2-Audio，在 Omni-Bench 等多模态 benchmark 上达到 SOTA。端到端语音指令跟随能力与文本输入相当（MMLU、GSM8K 上验证），流式 Talker 在鲁棒性和自然性上超越多数现有方案。

Key Findings

  • Thinker-Talker 架构首次实现 text 和 speech 的同步并发生成，避免两种模态间的干扰
  • TMRoPE 将 RoPE 分解为 temporal/height/width 三个维度，通过绝对时间编码同步 audio 与 video
  • Block-wise 处理使 audio 和 vision encoder 支持流式输入
  • Sliding-window DiT 限制 vocoder 感受野，降低首包延迟
  • 端到端语音指令跟随性能与纯文本输入相当（MMLU/GSM8K 差距极小）
  • 7B 参数规模下实现 text+image+audio+video 全模态感知与生成

Limitations

  • 模型规模仅 7B，相对于 GPT-4o 等闭源模型参数量有限
  • 语音生成基于 sliding-window DiT，计算开销仍较大
  • 音视频 interleaved 排列依赖 2 秒 chunk，限制了时间分辨率灵活性
  • 未公开详细的预训练数据组成和规模
  • 语音理解语种支持有限

Infrastructure Impact

  • Algorithm: Thinker-Talker 架构为多模态模型的 text-speech 同步生成开创了新范式
  • Kernel: Sliding-window DiT 和 block-wise attention 需要特定的 kernel 支持
  • Framework: 需要支持流式推理的 serving 框架，Thinker 和 Talker 间的异步 prefill 机制
  • LLM: TMRoPE 为多模态位置编码提供了通用方案
  • Agent: 端到端语音对话能力直接支持 voice agent 场景
Details

Deep Analysis (llm)

1. Architecture

  • Base: Dense Transformer，Thinker-Talker 双组件架构
  • Thinker: 标准 LLM，基于 Qwen2.5 系列，负责文本生成
  • Talker: 双轨自回归模型（Dual-track），接收 Thinker 的 hidden representation，生成 audio tokens
  • Audio Encoder: 基于 Whisper，block-wise 处理支持流式
  • Vision Encoder: 标准 vision encoder，block-wise temporal 处理
  • Position Encoding: TMRoPE — 将 RoPE 分解为 temporal(时间) + height(高度) + width(宽度) 三维
  • Speech Decoder: Sliding-window DiT vocoder，限制感受野以支持流式输出
  • Vocab: Qwen 标准 tokenizer + audio discrete tokens

2. Scale & Training

  • Parameters: ~7B (total)
  • Training: 端到端训练 Thinker + Talker
  • Data: Text, image-text, audio-text, video-text 多模态数据
  • Training stages: 详细信息未在公开 abstract 中披露

3. Quantization & Compression

N/A — 论文 abstract 未涉及量化相关内容，作为第一代 Omni 模型主要关注架构验证

4. Performance

  • Multimodal: SOTA on Omni-Bench
  • Vision: 与同规模 Qwen2.5-VL 相当
  • Audio: 超越 Qwen2-Audio
  • Speech instruction following: 端到端语音输入在 MMLU、GSM8K 上与文本输入表现相当
  • Speech generation: 流式 Talker 在鲁棒性和自然性上超越多数现有流式和非流式方案

7. Deployment Considerations

  • Min hardware: 单 A100/H100 即可（7B 模型）
  • License: Apache 2.0（开源）
  • Streaming: 需要支持流式输出的推理框架
  • Integration: 可替代 cascaded ASR→LLM→TTS pipeline

8. Position in Model Landscape

  • 定位: 首个开源的 omni-modal 端到端模型（text+image+audio+video → text+speech）
  • 里程碑: 证明了在 7B 规模下实现全模态感知与生成的可行性
  • 后续: 直接催生了 Qwen3-Omni，后者在架构、规模、性能上全面升级

SageAttention3: Microscaling FP4 Attention for Inference and An Exploration of 8-Bit Training

Jintao Zhang, Jia Wei, Pengle Zhang, Xiaoming Xu, Haofeng Huang, Haoxu Wang, Kai Jiang, Jianfei Chen, Jun Zhu | 2025-05 | https://arxiv.org/abs/2505.11594 Category: kernel | Tags: llm, algorithm, framework Read: 2026-04-16

Core Contribution

首次利用 Blackwell GPU 的 FP4 microscaling Tensor Core 实现 FP4 attention 加速推理（RTX5090 上达到 1038 TOPS，5x 快于 FlashAttention），并首次将低比特 attention 扩展到训练任务（INT8 SageBwd 在微调上无损）。

核心三问

Q1: 这篇论文试图解决什么核心痛点/问题？

Attention 的二次时间复杂度是生成模型的性能瓶颈。现有低比特 attention（FlashAttention3、SageAttention2）仅关注推理加速，没有人探索将低比特 attention 用于训练。同时，Blackwell GPU 新增的 FP4 Tensor Core 尚未被利用来加速 attention 计算。

Q2: 作者提出了什么新的"杀手锏"方法/架构？

两大贡献：(1) SageAttention3 — 使用 NVFP4 microscaling 量化（1×16 block，E4M3 scale factor）加速 $Q \cdot K^\top$ 和 $P \cdot V$ 两个矩阵乘法，并提出 two-level quantization 解决 softmax 输出 P 在 FP4 量化时 scale factor 动态范围过窄的问题；(2) SageBwd — 首次将 INT8 per-block 量化扩展到 attention 的反向传播，关键洞察是保持 $dO \cdot V^\top$ 在 FP16 以避免梯度误差累积。

Q3: 最终效果/结论如何？

SageAttention3 在 RTX5090 上达到 1038 TOPS（5x 快于 FlashAttention2，11x 快于 xformers），端到端加速 HunyuanVideo 视频生成 3x（489s→164s）。SageBwd 在微调任务上与 BF16 无差异（Qwen2.5、Llama3.2），但预训练收敛速度较慢。

逻辑故事还原

背景 (Context):

想象你在一个巨大的图书馆里找一本书。Attention 机制就像逐一对比每一对书的相关性——当书的数量翻倍，工作量翻四倍。FlashAttention 通过聪明的分块策略减少了内存搬运，但计算量本身没变。SageAttention2 用 INT8 量化把计算速度翻了一倍，但 FP4 能提供 8x 的理论加速——问题是 FP4 只有 15 个可表示的值，直接量化会严重损失精度。另外，所有低比特 attention 都只管推理，训练时的反向传播从未被量化加速。

破局 (Insight):

👉 核心洞察：softmax 输出 P 的值在 $[0,1]$ 范围内，FP4 microscaling 的 scale factor 必须用 E4M3 FP8 表示，但 $[0, 0.167]$ 的窄范围只能利用 E4M3 的 35 个值（总共 127 个可用值的 28%）。通过先做 per-token 归一化将范围扩展到 $[0, 448 \times 6]$，再做 microscaling FP4 量化，可用值从 280 个增加到 1016 个——精度提升 3.6 倍。这就像先把所有小数放大成整数再量化，充分利用了数字格式的表示能力。

拆解 (Deconstruction):

  1. FP4 Microscaling 量化 Q、K、V: 将 Q、K、V 按 1×16 block 量化为 FP4 + E4M3 scale factor，使用 SageAttention2 的 smooth Q/K 技巧减少离群值
  2. Two-level Quantization for P: 先 per-token 归一化 $\tilde{P}$ 到 $[0, 448 \times 6]$，再做标准 FP4 microscaling 量化，两个 scale 相乘还原
  3. 硬件级优化: K permutation（避免 accumulator layout 不匹配的 thread shuffle）、reuse shuffle（融合量化和 softmax 的 max reduction，节省 50% shuffle 操作，~10% kernel 加速）、producer warp epilogue（在寄存器压力下实现 MatMul 和 global store 的流水线重叠）
  4. SageBwd 训练: 7 个矩阵乘法中 6 个用 INT8 per-block 量化，保持 $dO \cdot V^\top$ 在 FP16（因为 $dP \to dS \to dQ/dK$ 的误差链会沿序列长度累积）
  5. 互补性: SageBwd 微调 + SageAttention3 推理的组合比 BF16+FP4 更准（INT8 和 FP4 的数据分布更接近）

    Summary

    SageAttention3 是清华大学朱军团队推出的第三代量化 attention 加速方案，有两大核心贡献。

    第一，FP4 Attention 推理加速：利用 NVIDIA Blackwell GPU 的 FP4 Tensor Core（NVFP4，E2M1 数据类型，1×16 block，E4M3 scale factor），将 attention 中的两个矩阵乘法（$Q \cdot K^\top$ 和 $P \cdot V$）量化到 FP4。核心技术创新是 two-level quantization——softmax 输出 P 的值域 $[0,1]$ 导致 FP4 scale factor 只能利用 E4M3 格式的 28% 可表示值（35/127），通过先 per-token 归一化到 $[0, 448 \times 6]$ 再做 microscaling 量化，将可用表示值从 280 提升到 1016。硬件实现方面，K permutation 避免了 accumulator layout 的 thread shuffle，reuse shuffle 融合了量化和 softmax 的 max reduction（~10% kernel 加速），producer warp epilogue 在寄存器压力下实现了计算和存储的流水线重叠。在 RTX5090 上达到 1038 TOPS，5x 快于 FlashAttention2，端到端加速视频生成 2.4-3x。

    第二，8-bit Trainable Attention：首次将低比特量化 attention 扩展到训练的前向和反向传播。关键发现是反向传播中 $dO \cdot V^\top$ 的量化误差会通过 $dP \to dS \to dQ/dK$ 的依赖链沿序列长度累积，因此必须保持 $dO \cdot V^\top$ 在 FP16，其余 6 个矩阵乘法用 INT8 per-block 量化。选择 INT8 而非 FP8 的原因：(1) INT8 梯度精度更高（CosSim: 0.9987 vs 0.9880 for dQ）；(2) INT8 硬件支持更广（A100, MI250, Ascend 910B）。实验表明微调任务上无损但预训练收敛较慢。

    Key Findings

    • FP4 microscaling attention 在 RTX5090 上达到 1038 TOPS，比 FlashAttention2 快 5x，比 xformers 快 11x
    • Two-level quantization 将 softmax 输出 P 的 FP4 量化 CosSim 从 93.32% 提升到 99.52%
    • NVFP4（E2M1+E4M3, block 1×16）显著优于 MXFP4（E2M1+E8M0, block 1×32）
    • 8-bit trainable attention (SageBwd) 在微调任务上与 BF16 完全一致，但预训练收敛较慢
    • 反向传播中 $dO \cdot V^\top$ 是精度最敏感的矩阵乘法，必须保持 FP16
    • INT8 SageBwd 优于 FP8 SageBwd（更高精度 + 更广硬件兼容）
    • SageBwd 微调 + SageAttention3 推理的组合产生互补效果（INT8 和 FP4 数据分布更接近）
    • 端到端加速：HunyuanVideo 3x、CogVideoX 2.4x（推理），Llama 1.15x（训练）

    Limitations

    • FP4 attention 仅支持 Blackwell GPU（RTX5090/B200/B300），不向下兼容
    • SageBwd 在预训练任务上收敛速度明显慢于 BF16，目前不适用于预训练
    • SageBwd 的实际加速（1.1-1.15x）远低于理论上限，受限于 Triton kernel 实现
    • FlashAttention3 仅支持 Hopper GPU，无法在 Blackwell 上直接对比
    • 论文未探讨 FP4 attention 在 LLM 长序列 prefill 场景的效果
    • 层间量化误差累积问题需要手动选择敏感层保持 FP16

    Infrastructure Impact

    • Algorithm: 证明低比特 attention 可用于微调训练，为 quantization-aware training 提供新思路；预训练的收敛问题值得进一步研究
    • Kernel: 首个 FP4 attention kernel，为 Blackwell GPU 上的 attention 计算提供了新的性能基准；two-level quantization 是通用的 FP4 softmax 量化技巧
    • Framework: 即插即用的 PyTorch 接口，可直接替换 FlashAttention；CUTLASS + CUDA 实现（推理），Triton 实现（训练）
    • LLM: 大幅加速视频/图像生成模型的推理；微调训练也能加速；对 LLM prefill 的影响待验证
    • Agent: 5x attention 加速可显著降低实时生成模型的延迟，使交互式视频生成更可行

    Deep Analysis (kernel)

    1. Target Operation

    计算原语: Attention（包含两个 GEMM：$S = Q \cdot K^\top$ 和 $O = P \cdot V$，以及 online softmax）

    输入/输出形状和数据类型:

    • 输入：$Q, K, V \in \mathbb{R}^{N \times D}$，FP16/BF16
    • 中间：$S, P \in \mathbb{R}^{N \times N}$（tiled，不完整存储）
    • 输出：$O \in \mathbb{R}^{N \times D}$，FP16/BF16
    • FP4 推理：Q, K, V 量化为 NVFP4（E2M1）+ E4M3 scale；P 经 two-level quant 后 FP4；累加器 FP32
    • INT8 训练：Q, K, V, P, dS 量化为 INT8 per-block；$dO \cdot V^\top$ 保持 FP16

    理论算术强度: 标准 attention 的 FLOPs = $4 \cdot N^2 \cdot D$（两个 $N \times D \times D \times N$ 矩阵乘法），memory = $O(N \cdot D)$（Q/K/V IO），所以 arithmetic intensity $\propto N$。长序列时 compute-bound，短序列时 memory-bound。

    Roofline 位置: FP4 的 theoretical peak = 1676 TOPS (RTX5090)，实测 1038 TOPS ≈ 62% utilization。对于 $N \geq 4K$ 的场景，kernel 处于 compute-bound 区域。

    2. Hardware Model

    目标 GPU: NVIDIA Blackwell (RTX5090, B200, B300) — FP4 attention 推理；NVIDIA Ampere/Ada (RTX4090) — INT8 SageBwd 训练

    硬件单元使用:

    • FP4 Tensor Core (Blackwell)：核心 GEMM 加速，NVFP4 格式（1×16 block，E4M3 scale）
    • INT8 Tensor Core (Ampere/Ada)：SageBwd 的 6 个 GEMM
    • CUDA 核心：softmax、量化/反量化、scale 运算
    • SFU：exp() 用于 softmax

    内存层级利用:

    • Registers：FlashAttention tiling 的 O 累加器、softmax 状态（m, l）
    • Shared memory (SMEM)：Q, K, V 的 tiles，FP4 quantized tiles + scale factors
    • L2 cache：隐式受益于 FlashAttention 的 tiling
    • HBM：输入 Q/K/V 和输出 O 的全局读写

    3. Optimization Techniques

    如 Algorithm 1 (FP4 Attention) 和 Figure 2 所示：

    TechniqueApplied?Details
    Tiling / blocking✅FlashAttention-style tiling，Bq × D 和 Bkv × D blocks
    Software pipelining✅Producer warp epilogue：producer warps 的 ping-pong 调度，一个 load 下一 tile 的同时另一个 store 结果
    Warp specialization✅Producer warps (load/store) + Consumer warps (FP4MM)，非传统的 consumer-centric 设计
    Vectorized memory access✅FP4 格式天然支持 128-bit 向量化（32 个 FP4 = 16 bytes）
    SMEM optimization✅K permutation 避免 accumulator→operand layout 的 thread shuffle；量化相关数据在 SMEM 中传递
    Register pressure management✅因寄存器压力放弃了 consumer-side ping-pong，改用 producer warp epilogue
    Kernel fusion✅Reuse shuffle：将 FP4 microscaling 的 max-16 reduction 与 online softmax 的 rowmax 融合，减少 50% shuffle 操作，约 10% kernel 加速
    Asynchronous operations✅CUTLASS 3.x 的 async copy 用于 global→shared memory 数据搬运

    4. Performance Analysis

    达到理论峰值的百分比:

    • RTX5090: 1038 / 1676 = 61.9% FP4 peak utilization
    • 对比：FlashAttention2 在 RTX5090 上约 214 TOPS / 209.5 TOPS FP16 peak ≈ ~100%

    如 Figure 4 所示，在不同序列长度下性能表现：

    vs vendor library / SOTA:

    • vs FlashAttention2：5x faster (RTX5090)
    • vs xformers：11x faster (RTX5090)
    • vs SageAttention2：1.6x faster (RTX5090)
    • SageBwd vs FlashAttention2 (fwd+bwd)：1.67x (RTX4090)

    性能随问题规模的变化: 在 $N \geq 4K$ 时 kernel 处于 compute-bound 区域，TOPS 稳定在 ~1000。$N < 2K$ 时 launch overhead 和 memory-bound 导致效率下降。

    延迟 vs 吞吐量: 对于视频生成（长序列、batch=1），延迟改善显著（3x）。对于 LLM decode（短序列、大 batch），FP4 attention 的优势可能不明显。

    5. Numerical Considerations

    精度 vs baseline:

    • FP4 SageAttention3 CosSim = 99.551%（vs FP16 = 100%）— 仅损失 0.45%
    • INT8 SageBwd dQ CosSim = 99.87%，dK = 99.93%，dV = 99.95%

    数值稳定性技巧:

    1. Smooth-K: $K = K - \text{mean}(K)$，减少 K 的离群值
    2. Smooth-Q: 每 block $Q_i$ 减去均值 $\bar{q}_i$，量化后通过 GEMV 补偿偏置项
    3. Two-level quantization: 解决 P 的窄动态范围问题
    4. Online softmax: 数值稳定的 softmax 实现（减最大值）
    5. 选择性 FP16 保持: $dO \cdot V^\top$ 在反向传播中保持 FP16 防止梯度误差累积
    6. 敏感层 FP16 fallback: 可将误差增长最大的 3 层保持 FP16，显著降低层间累积误差

      7. 数据类型选择: 选择 NVFP4 而非 MXFP4 — NVFP4 的 1×16 block + E4M3 scale 比 MXFP4 的 1×32 block + E8M0 scale 精度高得多（CosSim 99.52% vs 98.37%），因为更小的 block 更好地约束离群值影响，E4M3 scale 有更细粒度的动态范围。

      8c. Data Layout & Memory Access Patterns

      • 输入 tensor layout: Q, K, V 按 [batch, seq_len, num_heads, head_dim] 连续存储（head_dim 维度连续）
      • K permutation: FP4 GEMM 的 FP32 accumulator layout 与 operand A 的 register layout 不匹配（Figure 19-21），通过 permute P tile 的列来适配 accumulator layout，对应 permute K 的列（与量化 kernel 融合）
      • V transpose: V 需要变为 seq_len 维度连续（row-major for FP4 MMA），与量化 kernel 融合
      • Memory coalescing: FP4 格式（4 bits/element）天然支持高带宽加载，32 个 FP4 值 = 16 bytes = 一个 128-bit load
      • Bank conflict: K permutation 的设计目的之一就是避免 shared memory 中的 bank conflict
      • Register allocation: 寄存器压力大是本 kernel 的关键约束——两个 FP4 GEMM 的累加器、online softmax 状态、two-level 量化的中间变量都需要 registers，导致放弃传统的 consumer ping-pong 调度

      9. Infrastructure Impact

      LayerImpact
      Algorithm证明 FP4 量化可用于 attention 推理（无损）、INT8 可用于微调训练。Two-level quantization 是可推广的 FP4 softmax 量化范式。预训练的收敛问题指示需要更精细的量化策略或混合精度方案。
      Framework即插即用 PyTorch API，可直接替换 FlashAttention。推理框架（vLLM, SGLang）可直接集成 SageAttention3 获得 Blackwell 上的 5x attention 加速。训练框架可集成 SageBwd 加速微调。
      LLM5x attention kernel 加速对视频/图像生成模型影响巨大（attention 占比高）。对 LLM prefill 的加速取决于序列长度和 attention 占比。Decode 阶段因 batch 大、序列短，FP4 优势可能不明显。
      Agent3x 端到端视频生成加速使交互式 AI 创作更可行（HunyuanVideo 489s→164s）。对 LLM agent 的实时推理，需要结合 KV cache 量化等技术。

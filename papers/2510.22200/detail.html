LongCat-Video Technical Report

Meituan LongCat Team, Xunliang Cai, Qilong Huang, Zhuoliang Kang, Hongyu Li, Shijun Liang, Liya Ma, Siyu Ren, Xiaoming Wei, Rixu Xie, Tong Zhang | 2025-10 | https://arxiv.org/abs/2510.22200 Category: llm | Tags: algorithm, kernel, framework, cluster Read: 2026-04-16

Core Contribution

A unified 13.6B video generation model that natively supports long video generation through video-continuation pretraining, with multi-reward GRPO post-training and efficient coarse-to-fine inference achieving 12x speedup.

Summary

LongCat-Video is a 13.6B parameter Diffusion Transformer (DiT) model from Meituan that unifies Text-to-Video (T2V), Image-to-Video (I2V), and Video-Continuation (VC) tasks within a single framework by treating all tasks as conditional video continuation with varying numbers of conditioning frames (0 for T2V, 1 for I2V, multiple for VC). The model employs a standard DiT architecture with single-stream transformer blocks (48 layers, hidden size 4096, 32 attention heads), 3D self-attention + cross-attention, SwiGLU FFN, 3D RoPE, and RMSNorm QKNorm. It uses WAN2.1 VAE (4×16×16 compression) and umT5 for bilingual text encoding.

The paper introduces several technical innovations: (1) Block Causal Attention that isolates condition tokens from noisy tokens, enabling KV caching of condition frames across all sampling steps for efficient long video generation; (2) an improved GRPO training method for Flow Matching that interprets GRPO as stochastic noise search ($dR/dv_\theta \approx -\frac{3}{2} \cdot \hat{A} \cdot \varepsilon$), fixes SDE timesteps for precise credit assignment, introduces policy/KL loss reweighting to address gradient vanishing, and uses max group standard deviation across all groups for training stability; (3) multi-reward training with HPSv3 (visual quality), VideoAlign-based motion quality (trained on grayscale), and text-video alignment rewards that naturally prevent reward hacking through mutual constraints; (4) a coarse-to-fine inference pipeline generating 480p@15fps first then refining to 720p@30fps via a LoRA expert using flow matching with $t_{\text{thresh}}=0.5$ and only 5 refinement steps; (5) 3D Block Sparse Attention (open-sourced, Triton implementation with ring attention support) that retains $<$10% computation at near-lossless quality.

Comprehensive evaluations show LongCat-Video achieves a VBench 2.0 total score of 62.11% (3rd overall, 1st among open-source), leads all models in Commonsense (70.94%), and is competitive with commercial models (Veo3, PixVerse-V5) in internal T2V benchmarks. The combined optimizations yield 12.3x inference speedup on a single H800 GPU. Code and model weights are open-source.

核心三问

Q1: 这篇论文试图解决什么核心痛点/问题？

视频生成模型面临三大核心痛点：(1) 长视频生成中，随时间积累的误差导致颜色漂移和质量下降；(2) 高分辨率高帧率视频的推理效率极低（attention复杂度随token数二次增长，720p@30fps在H800上需1429秒）；(3) GRPO应用于视频生成时收敛慢、单一reward容易导致reward hacking。

Q2: 作者提出了什么新的"杀手锏"方法/架构？

LongCat-Video的杀手锏是三位一体的创新：(1) 通过将所有视频任务统一为video continuation并原生预训练，配合Block Causal Attention+KV Cache实现分钟级长视频生成；(2) 采用coarse-to-fine策略（480p→720p）+ 3D Block Sparse Attention（93.75% sparsity）实现>12x推理加速；(3) 为Flow Matching设计改进的多reward GRPO（固定SDE时间步 + loss重加权 + max group std + 多reward加权组合）。

Q3: 最终效果/结论如何？

在VBench 2.0上总分62.11%（开源模型第一），Commonsense维度70.94%领先所有模型（含Veo3的69.48%）。在内部T2V评测中Overall Quality与PixVerse-V5持平，明显优于Wan2.2开源模型。推理加速12.3x（1429s→116.5s），720p@30fps视频仅需约2分钟。

逻辑故事还原

背景 (Context):

视频生成正朝着world model方向发展，但现有方法面临几个关键瓶颈：长视频生成时误差随时间累积导致退化，高分辨率推理代价极高（attention的二次复杂度使720p视频生成需要数十分钟），GRPO虽在LLM和图像生成中成功但在视频生成中收敛慢且reward优化复杂。大多数现有模型无法在单一框架中同时支持T2V、I2V和视频续写。

破局 (Insight):

作者的核心洞察是：所有视频生成任务本质上都是"视频续写"——区别仅在于给了多少条件帧。通过原生预训练而非后期微调来获取长视频能力，模型天然学会了帧间的时间一致性。

👉 类比：就像一个厨师如果从学徒阶段就练习"接力做菜"（前一道菜的剩余食材作为下一道菜的起点），而不是只学做单道菜再硬加续集，他做出来的连续宴席自然更连贯、风味更统一。

拆解 (Deconstruction):

  1. 统一架构：基于DiT框架，条件帧设timestep=0注入无损信息，noisy帧正常采样。Block Causal Attention确保条件tokens不受noisy tokens影响，KV可跨采样步缓存
  2. 渐进式预训练：256p图像→256p视频→256p多任务→480p→720p，逐步提升分辨率和任务复杂度，共约678k iterations
  3. GRPO后训练：将GRPO解释为随机噪声搜索，固定单个SDE时间步解决credit assignment模糊，引入loss重加权消除梯度消失，max group std增强鲁棒性，多reward（VQ+MQ+TA）加权组合防止hacking
  4. 高效推理：CFG+CM蒸馏（50步→16步）→C2F（480p@15fps生成 + LoRA refinement expert升频到720p@30fps，仅5步）→BSA（93.75% sparsity，attention计算量 $<10\%$），三者叠加实现12.3x加速

    Figure 5: Unified Transformer & Block Causal Attention

    What it shows: Left side shows the unified transformer architecture supporting T2V (0 condition frames), I2V (1 frame), and VC (multiple frames) through a single model with different condition frame counts. Right side shows Block Causal Attention where condition tokens only attend to themselves while noisy tokens attend to both condition and noisy tokens.

    Why it matters: This is the architectural foundation of the entire paper — it explains how a single model handles three distinct video generation tasks and why KV caching of condition tokens is possible (their computation is independent of noisy tokens).

    Detailed description: The left diagram shows input sequences partitioned into condition frames (green, timestep=0) and noisy frames (blue, timestep sampled from [0,1]). For T2V there are no condition frames, for I2V there is one, for VC there are multiple. The right diagram shows the attention mask: condition tokens form a self-contained block (Eq.1: $X_{\text{cond}} = \text{Attention}(Q_{\text{cond}}, K_{\text{cond}}, V_{\text{cond}})$), while noisy tokens attend to the full concatenated KV (Eq.2: $X_{\text{noisy}} = \text{Attention}(Q_{\text{noisy}}, [K_{\text{cond}}, K_{\text{noisy}}], [V_{\text{cond}}, V_{\text{noisy}}])$). Cross-attention only applies to noisy tokens.

    Figure 12: 3D Block Sparse Attention

    What it shows: The three-step process of 3D block sparse attention: (a) partition latents into 3D blocks of size $t \times h \times w$, compute similarity between query block and all key blocks using average pooling; (b) select top-$r$ key blocks with highest scores; (c) compute standard attention only within selected blocks.

    Why it matters: This mechanism reduces attention computation to $<$10% of dense attention while maintaining near-lossless quality, enabling practical high-resolution video generation. It is open-sourced with both forward and backward implementations in Triton.

    Detailed description: The figure shows a 3D video latent ($T \times H \times W$) partitioned into non-overlapping blocks. For each query block $q_i$, average-pooled representations of all key blocks are computed, and a similarity score matrix is calculated. The top-$r$ key blocks (with highest similarity) are selected, and standard attention is computed only between $q_i$ and the selected key blocks. This exploits the high redundancy in video latents — nearby spatial-temporal regions have similar features, so most attention mass is concentrated in a few key blocks.

    Figure 11: Coarse-to-Fine Generation Pipeline

    What it shows: The two-stage generation pipeline for T2V, I2V, and VC tasks. Green arrows indicate low-resolution generation (480p@15fps), orange arrows indicate the refinement phase (720p@30fps). I2V and VC include additional condition frame handling.

    Why it matters: This design not only achieves >10x inference speedup but actually improves quality over native 720p generation by correcting local distortions and enhancing texture details.

    Detailed description: For T2V: base model generates 480p@15fps video → trilinear upsample to 720p@30fps → add noise at level $t_{\text{thresh}}=0.5$ → LoRA refinement expert denoises in 5 steps. For I2V/VC: condition frames are downsampled for stage 1, but original high-res condition frames are concatenated with upsampled output for stage 2 refinement. The refinement uses flow matching between the upsampled distribution and the high-res target distribution.

    Figure 13: Training Process Overview

    What it shows: The complete three-component training pipeline: (1) base model training (progressive pre-training + SFT), (2) RLHF training via GRPO with LoRA, (3) acceleration training (distillation LoRA + refinement expert LoRA + BSA).

    Why it matters: Shows how LoRA is used as a composable mechanism to stack multiple enhancements (RLHF, distillation, refinement) independently on top of the base model.

    Detailed description: The base model undergoes progressive pre-training from low to high resolution across multiple stages (images → videos → multi-task), followed by SFT on curated high-quality data. Three LoRA modules are then trained independently: RLHF LoRA (multi-reward GRPO), distillation LoRA (CFG + CM distillation for step reduction), and refinement expert LoRA (with BSA for coarse-to-fine generation). This modular design enables flexible combination and future extensions.

    Why it matters: Visual proof that the proposed GRPO techniques meaningfully improve generation quality, not just benchmark numbers.

    关键细节与启示

    技术细节补充:

    1. GRPO的随机噪声搜索视角: 论文证明GRPO for Flow Matching本质上用相对优势 $\hat{A}$ 和SDE噪声 $\varepsilon$ 来近似奖励对速度场的梯度 $dR/dv_\theta \approx -\frac{3}{2} \cdot \hat{A}_t \cdot \varepsilon$（Eq.5），这一理论洞察直接指导了固定SDE时间步（解决credit assignment模糊）和loss重加权（消除梯度消失）的设计
    2. Motion Quality reward用灰度视频训练: 为避免MQ reward model对特定颜色的偏好，使用灰度视频训练和推理，这一trick使验证集loss的上升延迟，说明减少了过拟合，提高了泛化能力

      3. 一句话总结: 美团LongCat-Video通过统一视频续写框架+原生长视频预训练+改进的多reward GRPO+coarse-to-fine推理策略，在13.6B参数下实现了与商业SOTA comparable的视频生成质量（VBench 2.0开源第一，Commonsense维度全场最佳），特别在长视频生成和物理常识方面表现突出，且推理效率提升12倍。

      Key Findings

      • 12.3x inference speedup (1429.5s → 116.5s for 720p) through LCM distillation + C2F + BSA on a single H800 GPU
      • Coarse-to-fine not only accelerates but improves quality — better texture details and local distortion correction vs. native 720p
      • Multi-reward GRPO prevents reward hacking that single-reward (HPSv3-only) training exhibits (static video tendency)
      • Block Sparse Attention achieves near-lossless quality at 93.75% sparsity (only retaining ~6.25% of attention computation)
      • GRPO improvements from T2V generalize to I2V and VC tasks without task-specific rewards
      • 70.94% on VBench 2.0 Commonsense (best across all models including Veo3), validating physical understanding capability
      • Training MFU of 33-38% with DeepSpeed-Zero2, Context Parallelism, Ring Attention, and Activation Checkpointing

      Limitations

      • I2V overall quality (3.17 MOS) lags behind Seedance 1.0 (3.35), Hailuo-02 (3.27), and Wan2.2 (3.26)
      • Image-alignment (4.04) is lower than competitors (4.12-4.18), suggesting room for improvement in preserving reference image attributes
      • RLHF training only uses T2V tasks — no task-specific rewards for I2V or VC (acknowledged as future work)
      • No audio generation capability
      • Training infrastructure achieves only 33-38% MFU, suggesting optimization room
      • Human Fidelity score (80.20%) is the lowest among tested models, indicating weaker human generation quality

      • Kernel: Open-source 3D Block Sparse Attention in Triton with forward/backward implementations and ring attention support for context parallelism; directly reusable as a modular component in other video/image generation models
      • Framework: C2F inference pipeline with LoRA refinement expert, Block Causal Attention with KV caching, and step distillation collectively provide a replicable recipe for efficient video serving; patterns applicable to other generation serving systems
      • LLM: The multi-reward GRPO method with weighted advantage aggregation provides a template for multi-objective RLHF in LLMs; the "GRPO as stochastic noise search" theoretical insight may generalize beyond flow matching

      Deep Analysis (llm)

      1. Architecture

      Base architecture: Dense Diffusion Transformer (DiT) with single-stream transformer blocks. Not a pure LLM but follows the Transformer-based generative model paradigm.

      Key architectural details:

      • Attention: 3D self-attention (MHA with 32 heads) + cross-attention for text conditioning. Block Causal Attention separates condition and noisy tokens. The model uses full attention (not GQA/MQA) — this is typical for DiT models which have shorter sequences than autoregressive LLMs
      • Position encoding: 3D RoPE across temporal, height, and width dimensions, enabling the model to handle varying video resolutions and aspect ratios
      • Normalization: RMSNorm applied as QKNorm within both self-attention and cross-attention for training stability. AdaLN-Zero for timestep modulation (each block has a dedicated modulation MLP with embedding size 512)
      • Activation: SwiGLU in FFN (FFN hidden size 16384 = 4× hidden size 4096)
      • Text encoder: umT5 (multilingual, supporting English and Chinese)
      • VAE: WAN2.1 VAE with 4×8×8 compression + 1×2×2 patchify = 4×16×16 total compression ratio
      • Model size: 48 layers, hidden size 4096, 32 attention heads, 13.6B parameters total

      Figure 5: Unified Transformer & Block Causal Attention

      解读: 这张图是整个架构的核心。左侧展示了统一输入表示——条件帧（green, $t=0$）和噪声帧（blue, $t \in [0,1]$）沿时间轴拼接。右侧的Block Causal Attention是关键设计：条件tokens自成一组只做self-attention（不受噪声tokens影响），而噪声tokens attend to全部tokens。这意味着条件部分的KV可以在所有采样步骤中缓存复用，对长视频生成（条件帧多）的效率提升至关重要。

      Comparison to competitors:

      • vs. Wan2.1: LongCat-Video uses single-stream DiT (Wan2.1 also uses DiT but different architecture scale)
      • vs. HunyuanVideo: Similar DiT backbone but HunyuanVideo uses dual-stream architecture
      • The key differentiator is the unified video-continuation formulation and Block Causal Attention for KV caching

      2. Scale & Training

      Parameter count: 13.6B total parameters (dense model, no MoE)

      Training data: Large-scale curated video dataset with comprehensive annotation pipeline:

      • Data preprocessing: deduplication (ID + MD5), scene detection (PySceneDetect + TransNetV2), black border cropping
      • Data annotation: aesthetic score, blur score, text coverage, watermark detection, optical flow for motion dynamics
      • Caption system: fine-tuned LLaVA-Video for basic captions + Tarsier2 data for temporal dynamics + Qwen2.5VL for cinematography/visual style + caption augmentation (Chinese/English translation, summarization)
      • Data distribution: text embedding clustering for category analysis and rebalancing

      Training recipe (multi-stage):

      1. Progressive pretraining: 5 stages from T2I@256p (285k iters) → T2V@256p → multi-task@256p → @480p → @480p+720p, total ~678k iterations, AdamW with decreasing LR (1e-4 → 2e-5)
      2. SFT: 7.5k iterations on curated high-quality data, LR 1e-5, including specialized datasets for camera motion and visual style
      3. RLHF (GRPO): 500 iterations only, LR 1e-4, group size 4, 64 prompts per step, T2V only
      4. Distillation: CFG distillation (2k iters) + CM distillation (3k iters), enabling 16-step inference
      5. Refinement expert: Full attention (500 iters) → BSA (500 iters, 93.75% sparsity), $t_{\text{thresh}}=0.5$

        6. Training stability techniques:

        • RMSNorm as QKNorm
        • Logit-normal-like loss weighting (more stable than logit-normal timestep sampling)
        • Adaptive timestep shift based on noise token volume
        • Per-frame independent noise perturbation for VC condition frames (reduces color drift)

        Training infrastructure: DeepSpeed-Zero2, Context Parallelism, Ring Attention, Activation Checkpointing, bucket-based mixed-resolution training, VAE cache to eliminate computation bubbles. MFU 33-38%.

        3. Quantization & Compression

        N/A — The paper does not discuss weight or activation quantization for the model. The "compression" in this paper refers to:

        • VAE compression: 4×16×16 pixel-to-latent compression
        • Step reduction: CFG + CM distillation reduces sampling from 50 to 16 steps
        • Spatial-temporal compression: C2F generates at 480p@15fps then refines to 720p@30fps
        • Attention sparsity: BSA retains only ~6.25% of attention computation (93.75% sparsity)

        These are all inference-time efficiency optimizations rather than model weight compression. Given the 13.6B parameter model size (~27GB in FP16), quantization to INT8 or INT4 would be valuable future work for deployment on consumer GPUs, but the paper does not explore this.

        5. Key Innovations

        Innovation 1: Unified Video-Continuation Framework

        • Problem: Separate models or adapters for T2V/I2V/VC increase complexity and prevent knowledge sharing
        • Mechanism: All tasks are formulated as conditional video continuation. T2V has 0 condition frames, I2V has 1, VC has multiple. Condition frames get timestep=0 (clean), noisy frames get sampled timesteps
        • Impact: Single model handles all tasks; shared pretraining improves each task; Block Causal Attention enables KV caching for condition tokens
        • Generality: Applicable to any conditional video generation model

        Innovation 2: GRPO for Flow Matching with Theoretical Foundation

        • Problem: GRPO for video generation suffers from slow convergence, credit assignment ambiguity, and gradient vanishing
        • Mechanism: (a) Theoretical insight that GRPO approximates $dR/dv_\theta$ via stochastic noise search; (b) Fix SDE timestep to single critical timestep for precise credit assignment; (c) Policy/KL loss reweighting $\lambda = \sqrt{t/(\Delta t(1-t))}$ to normalize gradient magnitude; (d) Max group std across all groups for stable advantage estimation
        • Impact: Ablation in Figure 7 shows each technique contributes; combined they enable successful GRPO training for video generation in only 500 iterations
        • Generality: The theoretical framework and techniques are applicable to any flow matching model, not just video generation

        Innovation 3: Multi-Reward Training with Anti-Hacking

        • Problem: Single reward optimization leads to reward hacking (e.g., HPSv3-only training produces static videos)
        • Mechanism: Three reward models (HPSv3 for VQ, VideoAlign for MQ trained on grayscale, VideoAlign for TA) with weighted relative advantage aggregation: $\hat{A}_{\text{total}} = \sum w_k \cdot \hat{A}_{k,t}$
        • Impact: Figure 9 demonstrates that HPSv3-only training hacks motion (videos become static) while multi-reward maintains balanced improvement across all dimensions. Figure 8 shows all reward curves improve together
        • Generality: Multi-reward GRPO with weighted advantages is directly applicable to any RL-based training

        Innovation 4: Coarse-to-Fine with Flow Matching Refinement

        • Problem: Native high-resolution generation is prohibitively expensive
        • Mechanism: Stage 1 generates 480p@15fps, stage 2 uses flow matching between upsampled distribution and target HR distribution with $t_{\text{thresh}}=0.5$ (only 5 refinement steps). LoRA training for efficient adaptation
        • Impact: 10.6x speedup AND improved quality (Figure 10 shows better textures than native 720p)
        • Generality: Applicable to any latent diffusion video model

        Innovation 5: 3D Block Sparse Attention (Open-Source)

        • Problem: Quadratic attention complexity is the primary bottleneck for high-resolution video generation
        • Mechanism: Partition video latents into 3D blocks ($t \times h \times w = 4 \times 4 \times 4$), compute block-level similarity scores via average pooling, select top-$r$ key blocks, compute attention only within selected blocks
        • Impact: 93.75% sparsity with near-lossless quality; ring attention variant for context parallelism; open-sourced Triton implementation
        • Generality: Open-sourced as modular component; users can customize block selection patterns (top-$r$, CDF-based, 2D+1D)

        Figure 12: 3D Block Sparse Attention

        解读: 这张图详细展示了BSA的三步流程。(a) 将3D视频latent分割为 $t \times h \times w$ 的非重叠block，对query block和所有key blocks做average pooling计算block级相似度分数；(b) 选择得分最高的top-$r$ 个key blocks；(c) 只在选中的key blocks范围内计算标准attention。这利用了视频latent的高冗余性——空间-时间上相近的区域特征相似，attention权重集中在少数key blocks上。93.75% sparsity意味着只保留1/16的key blocks（$r = N_k/16$）。

        7. Deployment Considerations

        • Minimum hardware: Single H800 (80GB) GPU for inference at 720p@30fps with all optimizations (116.5s per video). FP16 model weights ~27GB + KV cache. Without C2F/BSA, native 720p requires more memory
        • Recommended parallelism: Single GPU for inference (demonstrated). Training uses context parallelism + ring attention across multiple GPUs. No TP/PP details given
        • Open weights: Yes, code and model weights publicly available (GitHub: meituan-longcat/LongCat-Video). License not specified in paper
        • LoRA compatibility: Excellent — the paper's entire post-training strategy is based on LoRA (RLHF LoRA dim 128, alpha 64, applied to all linear layers in self-attention, cross-attention, and FFN). Multiple LoRA modules can be stacked
        • Fine-tuning friendly: Progressive training recipe is well-documented; LoRA-based adaptation demonstrated for multiple downstream tasks (refinement, distillation, RLHF)

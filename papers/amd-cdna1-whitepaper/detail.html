AMD CDNA Architecture White Paper

AMD | 2020-11 | https://www.amd.com/content/dam/amd/en/documents/instinct-business-docs/white-papers/amd-cdna-white-paper.pdf Category: hardware | Tags: gpu-architecture, matrix-core, hpc, mi100, cdna Read: 2026-04-16

Core Contribution

AMD首个专用计算GPU架构CDNA，引入Matrix Core Engine和MFMA指令族，将GPU从图形通用架构分化为计算专用架构。

Summary

AMD CDNA架构是AMD首次将GPU架构分化为计算专用(CDNA)和图形专用(RDNA)的产物。CDNA移除了所有图形固定功能单元（光栅化、曲面细分、图形缓存、混合、显示引擎），将释放的面积和功耗投入额外计算单元。

核心创新是Matrix Core Engine，引入MFMA (Matrix Fused Multiply-Add) 指令族，支持FP32/FP16/BF16/INT8四种输入数据类型的矩阵运算，所有MFMA输出均为FP32/INT32以避免累加溢出。MI100在7nm工艺下实现120个CU，首次突破10 TFLOPS FP64大关(11.5 TFLOPS)，FP16矩阵性能比MI50提升7x达184.6 TFLOPS。

配备32GB HBM2(1.23TB/s)、8MB L2 Cache(32 slices, 3TB/s+聚合带宽)。3条Infinity Fabric链路(23GT/s)实现4-GPU全互联，较前代环形拓扑提升27-44%多GPU性能。ROCm 4.0提供完整Exascale软件栈。

Key Findings

  • 首次将GPU分化为CDNA(计算)和RDNA(图形)两个独立架构
  • Matrix Core Engine引入MFMA指令族，FP16矩阵性能比MI50提升7x
  • 120 CU, 11.5 TFLOPS FP64 (首次突破10TF), 46.1 TFLOPS FP32 Matrix, 184.6 TFLOPS FP16 Matrix
  • 32GB HBM2, 1.23TB/s, 8MB L2
  • 3条IF链路实现4-GPU全互联(前代仅环形)

Limitations

  • 无FP8支持
  • 无结构化稀疏
  • 显存仅32GB
  • BF16 Matrix性能仅FP16的一半

Infrastructure Impact

  • Algorithm: BF16支持降低ML训练精度门槛
  • Kernel: MFMA指令族需要全新kernel开发
  • Framework: ROCm 4.0 + HIP提供CUDA移植路径

1. Architecture Overview

  • 产品: AMD Instinct MI100 / CDNA
  • 工艺: TSMC 7nm
  • 封装: Monolithic die
  • 目标市场: 数据中心HPC与AI训练

2. Generational Delta (vs GCN/MI50)

新增: Matrix Core Engine, MFMA指令族, BF16支持, 第3条IF链路

移除: 所有图形固定功能(光栅化、曲面细分、图形缓存、混合、显示引擎)

增强: CU数量120(vs 60)，内存带宽1.23TB/s(vs 1.0TB/s)，L2扩大至8MB

3. Compute Unit Micro-Architecture Details

Wavefront执行模型

  • Wavefront size = 64 work-items
  • 每CU含4个SIMD单元，每SIMD = 16-wide ALU
  • 64-wide wavefront在16-wide SIMD上需要4个cycle执行: \( C_{\text{wave}} = \frac{64}{16} = 4\ \text{cycles} \) — 这是AMD GPU的基本执行特征
  • SIMD位宽: \( B_{\text{SIMD}} = 16 \times 32 = 512\ \text{bits} \)，每CU: \( B_{\text{CU}} = 512 \times 4 = 2048\ \text{bits} \)
  • Stream Processors: \( N_{\text{SP}} = 64\ \text{SP/CU} \times 120\ \text{CU} = 7{,}680 \)

MMA Shape: AMD vs NVIDIA

  • AMD Matrix Core (CDNA1): EEDP shape — 典型MMA shape为 16×16×4 (FP16) 或更大
  • NVIDIA Tensor Core (V100): FEDP shape — 4×4×4 (FP16)
  • AMD选择更大shape意味着：(1)每条MMA指令做更多计算，(2)需要更多register暂存操作数，(3)适合大tile GEMM但小矩阵效率低

Peak Performance Derivation (从第一性原理推导)

\[ f_{\text{boost}} = 1.502\ \text{GHz} \]

\[ \text{FP16 Matrix} = 1024 \frac{\text{FLOPS}}{\text{clk} \cdot \text{CU}} \times 120\ \text{CU} \times 1.502\ \text{GHz} = 184.6\ \text{TFLOPS} \]

\[ \text{FP32 Matrix} = 256 \frac{\text{FLOPS}}{\text{clk} \cdot \text{CU}} \times 120\ \text{CU} \times 1.502\ \text{GHz} = 46.1\ \text{TFLOPS} \]

\[ \text{FP32 Vector} = 128 \frac{\text{FLOPS}}{\text{clk} \cdot \text{CU}} \times 120\ \text{CU} \times 1.502\ \text{GHz} = 23.1\ \text{TFLOPS} \]

\( 128 = 64 \times 2 \) (FMA: fused multiply-add)

\[ \text{FP64 Vector} = 64 \frac{\text{FLOPS}}{\text{clk} \cdot \text{CU}} \times 120\ \text{CU} \times 1.502\ \text{GHz} = 11.5\ \text{TFLOPS} \]

\[ \text{BW}_{\text{HBM}} = 1.2\ \text{GHz} \times \frac{4096\ \text{bits}}{8} \times 2 = 1.23\ \text{TB/s} \]

LDS Bank Structure

\[ C_{\text{LDS}} = 32 \times 512 \times \frac{32}{8} = 65{,}536\ \text{B} = 64\ \text{KB/CU} \]

32个bank每bank读/写32-bit → 每cycle最多 \( 32 \times 4\ \text{B} = 128\ \text{B} \) throughput。Bank conflict会降低实际带宽。

ACE限制

4个ACE (Asynchronous Compute Engine)，每ACE维护1个command stream → 最多4个concurrent stream。这限制了multi-stream overlap的程度。

3. Compute Unit Deep Dive

  • 每CU: 1个Matrix Core Engine, 4个Vector单元(16-wide), Scalar单元
  • VRF: 128KB/CU, SRF: 3.2KB/CU
  • Wavefront: 64 work-items, 10 wave counters
  • MFMA支持: FP32/FP16/BF16/INT8输入, FP32/INT32输出

  • HBM2: 32GB, 2.4GT/s, 1.23TB/s

  • Infinity Fabric: 3x links, 23GT/s, 16-bit wide
  • PCIe Gen4: x16, 32GB/s/direction

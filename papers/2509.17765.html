Qwen3-Omni Technical Report

LLM   Jin Xu, Zhifang Guo, Hangrui Hu, Yunfei Chu, Xiong Wang et al.  ·  2025-09  ·  arXiv ↗
multimodalomni-modelMoEspeech-generationthinker-talkerstreaming
Summary

Core Contribution

首次实现跨所有模态 (text/image/audio/video) 均无性能退化的单一多模态模型。采用 Thinker-Talker MoE 架构 (30B-A3B)，引入从头训练的 AuT 音频编码器和 multi-codebook 语音生成方案，在 36 个 audio/audio-visual benchmark 上取得 32 个开源 SOTA 和 22 个总体 SOTA，超越 Gemini-2.5-Pro 等闭源模型，端到端首包延迟仅 234ms。

Summary

Qwen3-Omni 是 Qwen2.5-Omni 的全面升级版，核心突破在于首次证明联合多模态训练可以不牺牲任何单模态性能。模型在 text、vision 上与同规模单模态 Qwen3 模型持平，在 audio 上大幅超越所有竞品。

架构层面，Qwen3-Omni 在 Qwen2.5-Omni 的 Thinker-Talker 架构上做了五大升级：(1) Thinker 和 Talker 均升级为 MoE（30B 总参, 3B 激活），大幅提升高并发推理能力；(2) 用从头训练的 AuT (Audio Transformer, 0.6B 参数, 20M 小时监督音频) 替代 Whisper encoder，获得更强的通用音频表示；(3) 语音生成采用 multi-codebook 表示，增加编码容量以忠实建模多样化语音特征；(4) Talker 从单轨改为多轨 codec 建模，通过 MTP module 自回归预测多个 codebook 层，Code2Wav 从 block-wise DiT 替换为轻量 causal ConvNet；(5) 输入输出 audio code rate 降至 12.5 Hz。

训练分三阶段：S1 锁定 LLM 训练 encoder alignment；S2 解锁全参数训练 ~2T tokens（text 0.57T, audio 0.77T, image 0.82T, video 0.05T, video-audio 0.05T）；S3 长序列训练（32768 tokens）。Post-training 包括 SFT → Strong-to-Weak Distillation → GSPO (RL) 三阶段。

关键实验结果：text 性能超越 GPT-4o 和 Qwen3-235B (Non-Thinking)；audio 在 36 个 benchmark 中 32 个开源 SOTA、22 个总体 SOTA；vision 与 Qwen2.5-VL-72B 相当；首包延迟 234ms（audio）/547ms（video）。

Key Findings

  • No modality degradation: 首次证明联合多模态预训练可实现所有模态无退化——Omni base 模型在 text、vision 上均与同规模单模态 base 模型持平或更优
  • Audio 能力突破: 36 个 audio/audio-visual benchmark 中取得 32 个开源 SOTA、22 个总体 SOTA，超越 Gemini-2.5-Pro 和 GPT-4o
  • Text 能力超预期: 30B-A3B 规模下 text 性能超越 GPT-4o 和 Qwen3-235B Non-Thinking（在 GPQA、AIME25、ZebraLogic 等指标上）
  • MoE 架构优势: MoE 设计使 Thinker/Talker 在高并发下 prefill 延迟基本不增长，decode TPS 保持稳定
  • AuT encoder: 从头训练的 AuT (20M 小时音频) 大幅超越 Whisper 在通用音频表示上的能力
  • Multi-codebook + ConvNet: 多 codebook 表示 + 轻量 ConvNet 替代 DiT，实现单帧即可流式语音合成，首包延迟 234ms
  • 长音频支持: 支持最长 40 分钟音频理解和交互
  • Thinking 模式: Thinking 变体在 AIME25 等推理任务上显著提升
  • Audio 对 vision 的互利: 实验表明加入 audio 数据可以改善 vision 性能（MMMU、OCR 任务）

Figure 2: Architecture Overview (described)

Detailed description: Thinker 是一个 30B-A3B 的 MoE Transformer，接收 text tokens + AuT 编码的 audio features + SigLIP2 编码的 vision features（通过 TM-RoPE 对齐时间戳），生成 text tokens 和 high-level representations。Talker 是 3B-A0.3B 的 MoE Transformer，接收 Thinker 的 multimodal features（不再使用 text representation，仅使用 audio/visual features），自回归生成 multi-codebook 的 zeroth codebook token。MTP module (80M, dense) 预测当前帧的 residual codebook tokens。Code2Wav (200M, ConvNet) 将 codec tokens 流式转换为波形。

Figure 3: AuT Architecture (described)

What it shows: Audio Transformer (AuT) 编码器——attention-encoder-decoder 自回归模型。

Detailed description: AuT 从头训练在 20M 小时监督音频上。Filter bank features 通过 Conv2D 8× 下采样，得到 12.5 Hz token rate。训练数据包含 80% 中英伪标注 ASR、10% 其他语言 ASR、10% 音频理解数据。使用 flash attention + dynamic window sizes (1-8秒) 平衡实时 prefill caching 效率和离线任务性能。

Limitations

  • Vision 长视频理解能力不足（受限于位置外推能力和 context length）
  • Thinking 模式在 ASR/music 等感知任务上反而不如 Instruct（引入幻觉）
  • 仅支持 10 种语言的语音生成，远少于 119 种文本语言
  • 需要独立 GPU 资源运行 30B MoE 模型
  • Audio 理解最长 40 分钟，更长音频仍受限
  • 实验主要在 Qwen 系列内部对比，缺少与 GPT-4o 的全面音频对比

Infrastructure Impact

  • Algorithm: Strong-to-Weak Distillation + GSPO (RL) 的 post-training pipeline 可泛化到其他多模态模型
  • Kernel: AuT 的 dynamic window attention、MoE routing、MTP module 均需高效 kernel 支持
  • Framework: vLLM 已支持 Qwen3-Omni 推理；Thinker/Talker 异步 prefill 需 serving 框架配合
  • LLM: 证明多模态联合训练不牺牲单模态性能，重塑了 "多模态必然退化" 的认知
  • Agent: 119 语言文本 + 19 语言语音理解 + 10 语言语音生成 → 全球化 voice agent 方案
Details

Deep Analysis (llm)

1. Architecture

  • Base: MoE Transformer (Thinker-Talker 双组件)
  • Thinker: 30B total / 3B active MoE Transformer
  • 基于 Qwen3 architecture
  • 输入: text tokens + AuT audio features (12.5 Hz) + SigLIP2 vision features
  • 位置编码: TM-RoPE (temporal 24 angles / height 20 / width 20, interleaved)
  • 输出: text tokens + high-level multimodal representations (传给 Talker)
  • Talker: 3B total / 0.3B active MoE Transformer
  • 关键变化: 不再消费 Thinker 的 text representation，仅使用 audio/visual multimodal features
  • 原因: (i) text tokens 和 embeddings 信息等价; (ii) multimodal conditioning 对 speech translation 等场景必要
  • 好处: Thinker/Talker 可使用独立 system prompt，外部模块可介入 Thinker text output
  • Audio Encoder (AuT): 0.6B params
  • Attention-encoder-decoder, 从头训练 on 20M hours supervised audio
  • Conv2D 8× 下采样 → 12.5 Hz token rate
  • Flash attention + dynamic window (1-8s)
  • 训练数据: 80% 中英 ASR + 10% 多语 ASR + 10% audio understanding
  • Vision Encoder: SigLIP2-So400M (543M params), from Qwen3-VL
  • MTP Module: 80M dense Transformer, 预测 residual codebook tokens
  • Code2Wav: 200M causal ConvNet, 替代 Qwen2.5-Omni 的 block-wise DiT
  • Tokenizer: Qwen tokenizer (BPE, 151,643 tokens)

2. Scale & Training

  • Total params: ~32B (Thinker 30B + Talker 3B + AuT 0.6B + Vision 0.5B + MTP 80M + Code2Wav 200M)
  • Active params: ~4.5B (Thinker 3B + Talker 0.3B + other modules)
  • Pretraining data (S2): ~2T tokens total
  • Text: 0.57T
  • Audio: 0.77T
  • Image: 0.82T
  • Video: 0.05T
  • Video-audio: 0.05T
  • Pretraining stages:
  • S1 (Encoder Alignment): 锁定 LLM，分别训练 audio/vision encoder → adapter → encoder
  • S2 (General): 全参数训练，2T tokens
  • S3 (Long Context): 最大 token length 8192 → 32768，增加长音频/视频比例
  • Post-training:
  • SFT → Off-policy Distillation → On-policy Distillation (teacher: Qwen3-32B/235B) → GSPO (RL with rule-based + model-based reward)

3. Quantization & Compression

N/A — 论文未讨论量化。但 MoE 架构天然减少 KV cache IO（仅激活参数的 KV cache 需读写），有利于推理效率。

Serving 相关估算:

  • KV cache: 按 Qwen3 MoE 架构估算，active 3B params 的 KV cache 开销远小于 30B dense
  • MoE 优势: 减少了长序列处理时的 IO consumption（Table 2 验证 TPS 在高并发下保持稳定）
  • 首包延迟: 234ms (audio) / 547ms (video) @ 1 concurrency
  • Generation RTF: 0.47 @ 1 concurrency, 0.66 @ 6 concurrency（始终 < 1）

4. Performance

Text→Text (non-reasoning):

  • MMLU-Redux 89.3 (vs GPT-4o 86.8, vs Qwen3-235B 91.3)
  • GPQA 70.4 (超越 GPT-4o 69.7 和 Qwen3-235B 66.9)
  • AIME25 61.3 (大幅超越 Qwen3-30B 24.7)
  • ZebraLogic 90.0 (大幅超越 GPT-4o 76.1)

Audio→Text:

  • ASR: Librispeech clean 1.22 WER (vs Seed-ASR 1.58, vs GPT-4o 1.39)
  • VoiceBench overall 88.8 (Thinking) (vs GPT-4o 86.8, vs Gemini-2.5-Pro 89.6)
  • MMAU 77.5 (超越 Gemini-2.5-Pro 77.4 和 GPT-4o 62.5)

Vision→Text:

  • MMMU-Pro 57.0 (vs GPT-4o 51.9, vs Qwen2.5-VL-72B 51.1)
  • MathVista 75.9 (vs GPT-4o 63.8, vs Qwen2.5-VL-72B 74.8)
  • Video-MME: 70.5 (略低于 Qwen2.5-VL-72B 73.3, 长视频是弱点)

AudioVisual→Text:

  • WorldSense 54.0 (大幅超越 Qwen2.5-Omni 45.4 和 Gemini-2.5-Flash 50.9)
  • DailyOmni 75.8 (Thinking) (超越 Gemini-2.5-Flash 72.7)

Speech Generation:

  • Seed TTS test-en 1.39 WER (vs CosyVoice3 1.45, vs Qwen2.5-Omni 2.33)
  • 10 语言 multilingual generation，中英法等语言超越 MiniMax/ElevenLabs

7. Deployment Considerations

  • Min hardware: 单 A100-80G 可服务 (MoE active ~4.5B)
  • Parallelism: TP=1 即可 for MoE 30B; 高并发下可用 vLLM batch scheduling
  • License: Apache 2.0, 完全开源
  • Streaming: 全链路 streaming 支持, RTF < 1
  • Integration: 可替代 ASR→LLM→TTS cascaded pipeline, 降低 E2E 延迟和系统复杂度

8. Position in Model Landscape

  • vs Qwen2.5-Omni: 全面升级 — MoE(7B→30B-A3B), AuT替Whisper, ConvNet替DiT, multi-codebook, 首包234ms, 40min音频
  • vs GPT-4o: Audio 任务大幅超越, text 多项持平或更优, 完全开源
  • vs Gemini-2.5-Pro: Audio SOTA 22 项超越, vision 略弱于 Gemini, text 可比
  • 里程碑意义: 首次证明 "全模态无退化" 是可实现的目标，终结了 "多模态必然牺牲单模态" 的认知
  • Next step: 多说话人 ASR, video OCR, audio-visual proactive learning, agent/function calling 增强

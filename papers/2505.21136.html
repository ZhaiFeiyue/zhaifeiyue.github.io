SageAttention2++: A More Efficient Implementation of SageAttention2

Kernel   Jintao Zhang, Xiaoming Xu, Jia Wei, Haofeng Huang, Pengle Zhang et al.  ·  2025-05  ·  arXiv ↗
attentionFP8quantizationtensor-coreinference-acceleration
Summary

Core Contribution

通过使用 FP16 累加器的 FP8 矩阵乘法指令（mma.f16.f8.f8.f16），将 SageAttention2 的 PV 矩阵乘法加速 2 倍，实现比 FlashAttention2 快 3.9 倍的 attention 加速，且精度无损。

Summary

核心三问

Q1: 这篇论文试图解决什么核心痛点/问题？

Attention 的时间复杂度随序列长度二次增长。SageAttention2 已经用 FP8 量化加速了 attention 中的矩阵乘法，但其使用的 mma.f32.f8.f8.f32 指令（FP32 累加器）仅比 FP16 快 2 倍。存在一个 4 倍加速的 FP16 累加器指令未被利用。

Q2: 作者提出了什么新的"杀手锏"方法/架构？

使用更快的 mma.f16.f8.f8.f16 指令（FP8 输入 + FP16 累加器）替代原来的 FP32 累加器指令。核心挑战是 FP16 的表示范围有限（±65504），直接使用会导致累加溢出。解决方案是缩窄 P 和 V 的 FP8 量化范围（$P_r = 224$, $V_r = 4.5$），使得 32 次乘加结果不超过 FP16 范围，并用 Delayed FP32 Buffering（两次 MMA 结果在 FP16 中累加后再转 FP32）减少类型转换开销。

Q3: 最终效果/结论如何？

SageAttention2++(4+8) 在 RTX4090 上相对 FlashAttention2 实现 3.9 倍加速，SageAttention2++(8+8) 实现 3.0 倍加速。在 Llama3.1、CogvideoX、HunyuanVideo、Wan、Flux、StableDiffusion3.5 等多种模型上端到端指标几乎无损。

逻辑故事还原

背景 (Context):

SageAttention2 已经实现了对 FlashAttention2 的显著加速，核心思路是将 Q/K 量化为 INT4/INT8、将 P/V 量化为 FP8 来利用 Tensor Core 的低精度计算能力。但在 PV 矩阵乘法中，它使用 FP32 累加器的 MMA 指令 (mma.f32.f8.f8.f32)，该指令仅比 FP16 快 2 倍。

破局 (Insight):

NVIDIA Ada 架构（RTX4090/5090）上存在一条更快的指令 mma.f16.f8.f8.f16，它使用 FP16 累加器做 FP8 乘法，速度是 FP16 的 4 倍（相比 FP32 累加器的 2 倍）。

👉 类比：这就像发现同一条高速公路有个限速更高的快车道。之前因为担心"翻车"（数值溢出）而走慢车道（FP32 累加器），现在通过"给货物减重"（缩窄量化范围），就能安全地走快车道（FP16 累加器）了。

拆解 (Deconstruction):

1. 识别瓶颈：PV Matmul 使用 mma.f32.f8.f8.f32，是 2x 加速；而 mma.f16.f8.f8.f16 能达到 4x 加速

2. 解决溢出问题：FP16 累加器范围为 $\pm 65504$。mma.m16n8k32 指令累加 32 个 $p \times v$ 乘积，因此需要 $|32 \times pv| \leq 65504$。将 $P$ 的量化范围从 448 缩窄到 $P_r = 224$，$V$ 从 448 缩窄到 $V_r = 4.5$，满足 $P_r \times V_r \leq 2047$

3. Delayed FP32 Buffering：连续两次 MMA 的 FP16 结果先累加再转 FP32，减半类型转换开销。约束变为 $P_r \times V_r \leq 1023.5$

4. 最优参数选择：通过实验验证 ($P_r = 224$, $V_r = 4.5$) 能保持与 SageAttention2 相同的精度（cosine similarity 99.97%）

关键细节与启示

技术细节补充:

1. 核心约束 Pr × Vr ≤ 2047 来自 MMA 指令的 $K = 32$ 累加维度和 FP16 的 65504 上限（$65504 / 32 = 2047$）。Delayed Buffering 将约束收紧为 $\leq 1023.5$。

2. SageAttention2++ 沿用了 SageAttention2 的 Q/K smoothing 和 INT4/INT8 量化，仅修改 PV Matmul 的指令选择和量化范围，是一个非常 targeted 的优化。

一句话总结: SageAttention2++ 通过使用 FP16 累加器的 FP8 MMA 指令并缩窄量化范围来避免溢出，在不损失精度的情况下将 attention 加速从 2x 提升到 3.9x（相对 FlashAttention2）。

Key Findings

  • FP8 Matmul with FP16 accumulator (mma.f16.f8.f8.f16) is 4x faster than FP16, vs 2x for FP32 accumulator — a 2x speedup opportunity for the $PV$ matmul in attention
  • Narrowing FP8 quantization ranges ($P_r = 224$, $V_r = 4.5$ vs original 448) maintains identical attention accuracy (cosine similarity 99.97%)
  • Delayed FP32 Buffering (accumulating 2 MMA results in FP16 before FP32 conversion) halves type conversion overhead
  • SageAttention2++(4+8) achieves 3.9x speedup over FlashAttention2 on RTX4090 at 32K sequence length
  • End-to-end metrics on Llama3.1, CogvideoX, HunyuanVideo, Wan, Flux, SD3.5 show negligible degradation
  • Works on both RTX4090 (Ada) and RTX5090 (Blackwell consumer) GPUs

Limitations

  • Only evaluated on NVIDIA Ada/Blackwell consumer GPUs (RTX4090, RTX5090); no data for data center GPUs (H100, H200, B100) or AMD
  • FP16 accumulator instruction availability is architecture-dependent — not available on Hopper (which has FlashAttention3 instead)
  • Paper is very short (5 pages + appendix) with limited ablation studies
  • No analysis of when the narrowed quantization range might cause problems (e.g., extreme outlier distributions)
  • Does not address the $QK$ matmul acceleration — only optimizes $PV$ matmul

Infrastructure Impact

  • Algorithm: Enables faster attention computation that can be plugged into any transformer-based model without algorithmic changes
  • Kernel: Demonstrates practical use of FP16-accumulated FP8 MMA instructions; technique generalizable to other FP8 GEMM workloads where output range can be bounded
  • Framework: Drop-in replacement for FlashAttention2/SageAttention2 in inference pipelines; reduces attention latency by ~3.9x
  • LLM: Directly accelerates prefill phase of LLM inference; particularly impactful for long-context applications
  • Agent: Reduced attention latency enables faster real-time inference for interactive agent systems
Details

Deep Analysis (kernel)

1. Target Operation

Computational primitive: Attention mechanism, specifically the PV matrix multiplication (P×V) in the second half of attention computation.

Input/output shapes and data types:

  • Input $P$ (post-softmax): shape [batch, heads, seq_len_q, seq_len_kv], quantized to FP8 E4M3 with range narrowed to $P_r = 224$
  • Input $V$: shape [batch, heads, seq_len_kv, head_dim], quantized to FP8 E4M3 with per-channel granularity, range narrowed to $V_r = 4.5$
  • $Q$, $K$: INT4 or INT8 for the $QK$ matmul (unchanged from SageAttention2)
  • Output $O$: FP32 (accumulated), eventually FP16/BF16

Theoretical arithmetic intensity: For attention with seq_len $N$ and head_dim $d$:

  • $QK^\top$: $O(N^2 d)$ FLOPs, $O(Nd)$ bytes → arithmetic intensity = $O(N)$
  • $PV$: $O(N^2 d)$ FLOPs, $O(N^2 + Nd)$ bytes → for large $N$, compute-bound
  • With FP8 quantization, the byte count per element halves → further improves arithmetic intensity

Roofline position: At long sequences (8K+), attention is compute-bound on the Matmul operations. Using FP8 inputs with 4x instruction throughput shifts the bottleneck further toward memory bandwidth for the quantize/dequantize overhead.

2. Hardware Model

Target GPU architecture: NVIDIA Ada Lovelace (RTX 4090 — SM89) and Blackwell consumer (RTX 5090).

Hardware units used:

  • Tensor Cores: FP8 MMA with FP16 accumulator (mma.f16.f8.f8.f16). On Ada, the 4th generation Tensor Cores support FP8 with both FP16 and FP32 accumulators. The FP16 accumulator variant runs at 2x the throughput of FP32 accumulator.
  • CUDA cores: For quantization (scaling factors computation), softmax, dequantization
  • Special function units: Not directly used

Memory hierarchy exploitation:

  • Registers: FP16 accumulator results stored in registers (fewer bits = less register pressure than FP32 accumulators)
  • Shared memory/LDS: Tiled Q, K, P, V blocks following FlashAttention's tiling scheme
  • L2 cache: Not explicitly discussed
  • HBM: Source tensors Q, K, V, output O

Theoretical peak throughput:

  • RTX 4090: FP8 with FP32 accumulator = 661 TFLOPS, FP8 with FP16 accumulator = ~1322 TFLOPS (2x)
  • The paper achieves up to 640 TOPS at 32K seq_len on RTX4090 (headdim=128, non-causal), which includes the entire attention pipeline not just PV Matmul

Kernel fusionYes (inherited)Full attention (QK^T → softmax → PV) fused in single kernel

Novel technique — Delayed FP32 Buffering: Accumulate 2 consecutive mma.m16n8k32 results in FP16 before converting to FP32. This halves the number of FP16→FP32 conversion PTX instructions, reducing overhead. The tradeoff is a tighter range constraint ($P_r \times V_r \leq 1023.5$ instead of 2047).

Novel technique — Quantization range narrowing: Instead of using the full E4M3 range ($\pm 448$), use custom scale factors to restrict $P$ to $\pm 224$ and $V$ to $\pm 4.5$. This ensures 32 accumulated products fit in FP16 range.

4. Performance Analysis

% of theoretical peak:

  • RTX4090, headdim=128, non-causal, 32K: 640 TOPS achieved. If we estimate FP8+FP16acc peak at ~1322 TFLOPS, but the full attention includes QK^T (INT4/INT8), softmax, quantization overhead — 640 TOPS for the full pipeline is excellent.
  • The speedup ratios are more informative: 3.9x over FlashAttention2 for (4+8) variant, 3.0x for (8+8) variant.

Performance vs baselines:

  • FlashAttention2: 3.9x speedup (4+8 variant), 3.0x (8+8 variant)
  • SageAttention1: ~1.1-1.3x speedup over Sage1
  • SageAttention2: ~1.1-1.5x speedup over corresponding Sage2 variants

Scaling with problem size: Performance improves significantly from 1K to 8K sequence length, then plateaus from 16K-32K. This is expected as longer sequences amortize kernel launch overhead and achieve better Tensor Core utilization. At 1K, the speedup over FlashAttention2 is smaller (~2x for 4+8) because the kernel doesn't fully saturate.

5. Numerical Considerations

Accuracy vs baseline:

  • Cosine similarity: 99.97% (same as SageAttention2, measured across all attention layers of CogvideoX)
  • L1 distance: 0.01862 (essentially identical to SageAttention2)
  • End-to-end metrics on 6 diverse models show negligible degradation

Numerical stability tricks:

  • Scale factor adjustment: $\delta_P = |\max(\tilde{P})| / P_r$ and $\delta_V = |\max(V)| / V_r$ where $P_r = 224$, $V_r = 4.5$
  • Per-block granularity for $P$: Each tile block has its own scale factor
  • Per-channel granularity for $V$: Each column of $V$ has a separate scale factor (inherited from SageAttention2)
  • Delayed FP32 Buffering: Limits FP16 accumulation to 2 MMA results before promoting to FP32, preventing long-chain FP16 accumulation errors

Edge cases: Not discussed. The narrow quantization range ($V_r = 4.5$) could be problematic for $V$ values with extreme dynamic range, but the per-channel scaling helps mitigate this.

8. Key Source Code Walkthrough

#### 8b. Pseudocode Annotation

The paper doesn't provide explicit pseudocode, but the algorithm can be reconstructed:


# SageAttention2++ PV Matmul (per tile block)
# Grid: blocks over Q tiles × head × batch
# Block: warps cover the tile computation

1. Load Q_tile, K_tile from HBM → shared memory
2. Compute QK^T via INT4/INT8 Matmul (unchanged from SageAttention2)
3. Apply online softmax → P_tile (FP32)

4. QUANTIZE P_tile to FP8:
   δP = max(|P_tile|) / 224          # [Register] narrowed range (was /448)
   P_hat = round(P_tile / δP)        # [Register → Shared] FP8 E4M3

5. Load V_tile from HBM → shared memory
6. QUANTIZE V_tile to FP8:
   δV = colmax(|V_tile|) / 4.5       # [Register] per-channel, narrowed (was /448)
   V_hat = round(V_tile / δV)        # [Shared] FP8 E4M3

7. COMPUTE PV via mma.f16.f8.f8.f16:
   for k_chunk in range(0, K, 32):
     # [Tensor Core] MMA: FP8 × FP8 → FP16 accumulator
     acc_fp16 += mma.f16.f8.f8.f16(P_hat_chunk, V_hat_chunk)  # 4x speed vs FP16
     if k_chunk % 2 == 1:           # Delayed FP32 Buffering
       acc_fp32 += float32(acc_fp16) # [Register] promote every 2 MMAs
       acc_fp16 = 0

8. DEQUANTIZE: O_tile = acc_fp32 * δP * δV  # [Register]
9. Online softmax rescaling + write O_tile to HBM

Key performance insight: Step 7 is where the 2x speedup happens. By using mma.f16.f8.f8.f16 instead of mma.f32.f8.f8.f32, the Tensor Core throughput doubles. The Delayed FP32 Buffering in Step 7's if-branch reduces FP16→FP32 conversion overhead by 50%.

#### 8c. Data Layout & Memory Access Patterns

  • Input tensors: Standard [batch, heads, seq_len, head_dim] layout (row-major for Q/P, column-major granularity for V scaling)
  • Memory coalescing: FP8 quantized tensors enable twice the elements per cache line, improving effective bandwidth
  • Register allocation: FP16 accumulator registers use half the bits of FP32, freeing registers for other uses (potential occupancy improvement)
  • Bank conflicts: Inherited from SageAttention2's shared memory layout design

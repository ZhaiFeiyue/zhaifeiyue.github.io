What It Does

дёҖеҸҘиҜқжҖ»з»“: дёҖдёӘиҪ»йҮҸдҪҶеҠҹиғҪејәеӨ§зҡ„еӨҡ Agent е·ҘдҪңжөҒзј–жҺ’жЎҶжһ¶пјҢи®©ејҖеҸ‘иҖ…з”ЁеЈ°жҳҺејҸж–№ејҸе®ҡд№ү AI Agent зҡ„жҢҮд»ӨгҖҒе·Ҙе…·гҖҒжҠӨж Ҹе’ҢеҚҸдҪңе…ізі»пјҢз„¶еҗҺиҮӘеҠЁжү§иЎҢгҖӮ

зұ»жҜ”: еҰӮжһңжҠҠ LLM жҜ”дҪңдёҖдёӘиҒӘжҳҺдҪҶиў«еҠЁзҡ„еӨ§и„‘пјҢAgents SDK е°ұжҳҜз»ҷиҝҷдёӘеӨ§и„‘иЈ…дёҠдәҶ"жүӢи„ҡ"(Tools)гҖҒ"еҗҢдәӢ"(Handoffs)гҖҒ"е®үе…Ёз»і"(Guardrails)гҖҒе’Ң"жүӢжңҜе®Ө"(Sandbox)гҖӮдҪ еҸӘйңҖиҰҒжҸҸиҝ°жҜҸдёӘ Agent иҜҘе№Ід»Җд№ҲпјҢжЎҶжһ¶иҙҹиҙЈзј–жҺ’жү§иЎҢеҫӘзҺҜгҖҒй”ҷиҜҜйҮҚиҜ•гҖҒдёӯж–ӯжҒўеӨҚе’ҢиҝҪиёӘи®°еҪ•гҖӮ

зӣ®ж Үз”ЁжҲ·: йңҖиҰҒжһ„е»ә multi-agent зі»з»ҹзҡ„ ML е·ҘзЁӢеёҲе’Ңе№іеҸ°ејҖеҸ‘иҖ…гҖӮ

з”ҹжҖҒе…ізі»:

  • жӣҝд»ЈдәҶзӨҫеҢәж—©жңҹзҡ„ Agent жЎҶжһ¶(LangChain AgentгҖҒAutoGPT зӯү)зҡ„ж ёеҝғи°ғеәҰйғЁеҲҶ
  • ж·ұеәҰз»‘е®ҡ OpenAI Responses API / Chat Completions API
  • йҖҡиҝҮ LiteLLM / any-llm жү©еұ•ж”ҜжҢҒ 100+ жЁЎеһӢ
  • еҺҹз”ҹйӣҶжҲҗ MCP (Model Context Protocol) еҚҸи®®

Core Concepts & Data Flow

1. Agent вҖ” жҷәиғҪдҪ“е®ҡд№ү


@dataclass
class Agent(AgentBase, Generic[TContext]):
    name: str                           # Agent еҗҚз§°
    instructions: str | Callable        # зі»з»ҹжҸҗзӨәиҜҚ (йқҷжҖҒ / еҠЁжҖҒ)
    model: str | Model | None           # LLM жЁЎеһӢ (й»ҳи®Ө gpt-4.1)
    tools: list[Tool]                   # еҸҜз”Ёе·Ҙе…·
    handoffs: list[Agent | Handoff]     # еҸҜе§”жүҳзҡ„еӯҗ Agent
    input_guardrails: list[InputGuardrail]   # иҫ“е…ҘжҠӨж Ҹ
    output_guardrails: list[OutputGuardrail]  # иҫ“еҮәжҠӨж Ҹ
    output_type: type | None            # з»“жһ„еҢ–иҫ“еҮәзұ»еһӢ
    hooks: AgentHooks | None            # з”ҹе‘Ҫе‘Ёжңҹеӣһи°ғ
    tool_use_behavior: ...              # е·Ҙе…·и°ғз”ЁеҗҺиЎҢдёә

е…ій”®и®ҫи®Ў: Agent жҳҜдёҖдёӘ @dataclassпјҢдёҚжҳҜ Pydantic ModelгҖӮиҝҷж„Ҹе‘ізқҖ:

  • иҪ»йҮҸгҖҒеҸҜеәҸеҲ—еҢ–
  • ж”ҜжҢҒ clone() еӨҚеҲ¶
  • ж”ҜжҢҒ as_tool() е°ҶиҮӘе·ұеҸҳдёәеҸҰдёҖдёӘ Agent зҡ„е·Ҙе…·

2. Runner вҖ” жү§иЎҢеј•ж“Һ

  в”Ӯ     в””в”Җ еҰӮжһң tripwire и§ҰеҸ‘ вҶ’ жҠӣејӮеёёеҒңжӯў

Key Design Decisions

1. Provider-Agnostic дҪҶ OpenAI-First

  • еҶ…йғЁж•°жҚ®ж јејҸз»ҹдёҖдёә OpenAI Responses API ж јејҸ
  • е…¶д»– LLM йҖҡиҝҮ chatcmpl_converter.py иҪ¬жҚў Chat Completions ж јејҸ
  • LiteLLM / any-llm дҪңдёә extensions ж”ҜжҢҒ 100+ жЁЎеһӢ

2. Dataclass over Pydantic for Agent

  • Agent жҳҜ @dataclass иҖҢйқһ Pydantic BaseModel
  • еҘҪеӨ„: жӣҙиҪ»йҮҸгҖҒж”ҜжҢҒ dataclasses.replace() жө…жӢ·иҙқ
  • д»Јд»·: зұ»еһӢйӘҢиҜҒйңҖиҰҒжүӢеҠЁе®һзҺ° (__post_init__)

3. зҠ¶жҖҒеӨ–зҪ® + еҸҜдёӯж–ӯжҒўеӨҚ

  • RunState еҸҜеәҸеҲ—еҢ–дёә JSON вҶ’ е®һзҺ° human-in-the-loop
  • е®Ўжү№зі»з»ҹ: е·Ҙе…·и°ғз”ЁеҸҜжҡӮеҒңзӯүеҫ…дәәе·Ҙе®Ўжү№
  • жҒўеӨҚж—¶е°Ҷ RunState дј е…Ҙ Runner.run() з»§з»ӯжү§иЎҢ

4. е№¶еҸ‘жЁЎеһӢ: asyncio дёәдё»

  • жүҖжңүж ёеҝғи·Ҝеҫ„дҪҝз”Ё async/await
  • е·Ҙе…·жү§иЎҢе№¶еҸ‘: asyncio.gather() е№¶иЎҢжү§иЎҢеӨҡдёӘе·Ҙе…·и°ғз”Ё
  • run_sync() йҖҡиҝҮ asyncio.run() жЎҘжҺҘеҗҢжӯҘи°ғз”Ё

5. MCP дҪңдёәдёҖзӯүе…¬ж°‘

  • еҺҹз”ҹж”ҜжҢҒ MCP Server еЈ°жҳҺејҸжҢӮиҪҪ
  • жҜҸж¬Ў Agent иҝҗиЎҢж—¶иҮӘеҠЁеҸ‘зҺ°е№¶жіЁеҶҢ MCP е·Ҙе…·
  • ж”ҜжҢҒ hosted MCP (OpenAI жңҚеҠЎз«Ҝ) е’Ң local MCP

Verdict & Recommendations

еә”иҜҘдҪҝз”Ёзҡ„еңәжҷҜ:

  • еҰӮжһңдҪ зҡ„дё»иҰҒ LLM жҳҜ OpenAI вҶ’ иҝҷжҳҜжңҖдҪійҖүжӢ©
  • йңҖиҰҒ multi-agent handoff + tool calling + guardrails зҡ„з»„еҗҲ
  • йңҖиҰҒ human-in-the-loop дёӯж–ӯжҒўеӨҚ
  • йңҖиҰҒ MCP йӣҶжҲҗ

дёҚйҖӮеҗҲзҡ„еңәжҷҜ:

  • зәҜ open-source LLM еңәжҷҜ (дёҺ OpenAI API ж јејҸиҖҰеҗҲиҫғж·ұ)
  • йңҖиҰҒжһҒз®ҖжЎҶжһ¶ (иҝҷдёӘ SDK еҠҹиғҪе·Із»ҸзӣёеҪ“дё°еҜҢ)
  • йңҖиҰҒиҮӘе®ҡд№үеә•еұӮи°ғеәҰйҖ»иҫ‘ (Runner зҡ„ turn loop дёҚеӨӘе®№жҳ“жү©еұ•)

еҖјеҫ—еӯҰд№ зҡ„и®ҫи®Ў:

1. RunState еҸҜеәҸеҲ—еҢ– + дёӯж–ӯжҒўеӨҚ вҖ” з”ҹдә§зә§ human-in-the-loop ж–№жЎҲ

2. Guardrails дёҺ Agent жү§иЎҢе№¶иЎҢ вҖ” дёҚеўһеҠ йўқеӨ–е»¶иҝҹ

3. Agent-as-Tool жЁЎејҸ вҖ” дјҳйӣ…ең°е®һзҺ° Agent еөҢеҘ—и°ғз”Ё

4. Tracing еҶ…е»әиҖҢйқһжҸ’д»¶ вҖ” дҝқиҜҒдәҶеҸҜи§ӮжөӢжҖ§

5. Sandbox Capabilities жҸ’д»¶еҢ– вҖ” еҸҜжү©еұ•зҡ„жІҷз®ұиғҪеҠӣзі»з»ҹ

Massive Values in Self-Attention Modules are the Key to Contextual Knowledge Understanding

Mingyu Jin, Kai Mei, Wujiang Xu, Mingjie Sun, Ruixiang Tang, Mengnan Du, Zirui Liu, Yongfeng Zhang | 2025-02 | https://arxiv.org/abs/2502.01563 Category: llm | Tags: quantization, attention-mechanism, RoPE, interpretability, contextual-knowledge Read: 2026-04-16

Core Contribution

首次系统性揭示了 LLM 中 self-attention 的 Q 和 K 矩阵中"集中式大值"(concentrated massive values) 的成因 (RoPE) 和功能角色 (上下文知识理解的关键)，并证明保护这些大值的量化方法能更好地保留模型的上下文理解能力。

Summary

大语言模型在上下文知识理解方面取得了显著成功，但对其内部机制的理解仍然有限。本文首次系统性地揭示了一个反直觉现象：在使用 RoPE 位置编码的现代 Transformer LLM (Llama, Gemma, Qwen 等) 中，attention 的 Q 和 K 表征在特定维度区域出现了集中分布的异常大值 (massive values)，而 V 中不存在这种模式。更关键的是，这些大值跨越不同注意力头时，集中在相似的维度位置——主要是 RoPE 的低频区域。

通过在 10 个数据集上对 Gemma2-9B、Llama3-8B、Qwen2.5-7B 进行系统实验，作者证明这些 massive values 对上下文知识理解 (contextual knowledge, CK) 至关重要，但对参数知识检索 (parametric knowledge, PK) 影响很小。当 massive values 被破坏时，GSM8K 准确率从 76.9% 暴跌至 4.0%，Passkey Retrieval 从 100% 跌至 0%，而参数知识任务 (Cities, Sports 等) 仍保持 65-88% 的性能。

量化实验进一步验证了这一发现：AWQ 和 SmoothQuant (保护 massive values) 在上下文任务上表现良好，而 GPTQ (不保护) 则出现明显退化。因果分析表明，massive values 的集中是 RoPE 频率分布机制导致的，并从第一层开始出现。

Key Findings

  • Q 和 K 中 massive values 集中在 RoPE 的低频维度区域，跨注意力头位置一致，V 中无此模式
  • Massive values 对上下文知识理解至关重要：破坏后 GSM8K 从 76.9% → 4.0%，Passkey Retrieval 100% → 0%
  • 参数知识检索任务受 massive value 破坏影响有限 (仅降 15-20%)，说明知识存储与上下文处理有分离的机制
  • 保护 massive values 的量化方法 (AWQ, SmoothQuant) 保留上下文理解能力优于不保护的方法 (GPTQ)
  • RoPE 是 concentrated massive values 的根本成因：无 RoPE 模型 (GPT-2, OPT, Jamba) 不出现此现象
  • 知识冲突实验: 破坏 massive values 后模型忽略误导性上下文，回退到参数知识，准确率反而提升

Limitations

  • 仅在 7B-9B 规模模型上实验，未验证在更大模型 (70B+) 或更小模型上的一致性
  • Disruption 实验方法 (替换为 mean/zero) 较为粗暴，可能无法精确量化 massive values 的信息承载量
  • 未深入分析 massive values 在 training 过程中如何逐渐形成（仅观察到"从第一层开始"）
  • 未探讨 GQA/MQA 等 Q/K 共享机制如何与 massive values 交互
  • 知识冲突实验仅在简单事实问答上验证，复杂推理场景下的行为未知
  • 未提供如何在训练中主动利用此发现改善模型设计的具体方案

Infrastructure Impact

  • Algorithm: 启发基于 massive value 保护的新型训练策略和量化感知训练方法设计，例如在 loss function 中加入对低频 Q/K 维度的正则化
  • Kernel: 量化 kernel 设计时需要对 Q/K 的低频维度采用更高精度或特殊处理（per-channel scaling），FlashAttention 等 kernel 也可考虑对 massive value 维度做特殊优化
  • Framework: 推理框架在 KV cache 量化时需要特别保护 K 的 massive value 维度，AWQ/SmoothQuant 类方法应优先于 GPTQ 用于上下文理解密集场景
  • LLM: 直接影响 LLM 架构设计：RoPE 的低频维度承载语义信息，模型设计可考虑增加这些维度的容量或精度
  • Agent: Agent 场景高度依赖上下文理解 (tool calling, memory retrieval)，使用 massive-value-aware 量化对 agent 性能至关重要
  • Cluster: N/A

Deep Analysis (llm)

1. Architecture

本文并非提出新架构，而是分析现有 RoPE-based Transformer 架构的内部机制。

涉及的模型架构：

  • Attention variant: 分析了 MHA (Llama2/3, Gemma, Qwen) 和 GQA 模型中 Q/K/V 的表征特性
  • Position encoding: 核心发现——RoPE 是 concentrated massive values 的成因。RoPE 将 head dimension 分为 $d/2$ 对，每对以不同频率旋转 ($\theta_j = 10000^{-2j/d}$)。低频维度 (大 $j$) 旋转慢，主要编码语义；高频维度 (小 $j$) 旋转快，编码精细位置信息
  • 对比架构: GPT-2 (学习式绝对位置编码)、OPT (学习式)、Jamba (Mamba+Transformer 混合，无 RoPE) 均不出现此模式
  • 跨模态验证: LLaVA-1.5 (使用 RoPE) 和 Qwen2-VL (使用 M-RoPE: 分解为 1D/2D/3D 位置) 也观察到同样模式

Figure 7: Gemma2-9B vs OPT-350M Q/K/V

!Figure 7: Gemma vs OPT

解读: 上排 OPT (无 RoPE) 的 K/Q/V 热力图均呈均匀分散分布。下排 Gemma2-9B (有 RoPE) 的 K 和 Q 在低频区域（底部）出现明亮条带，V 无此模式。这张图是 RoPE 因果关系的最直接视觉证据。

2. Scale & Training

  • 参数规模: 实验覆盖 Gemma2-9B, Llama3-8B, Qwen2.5-7B 三个 7-9B 规模模型
  • 训练数据: 未涉及训练数据分析（本文为分析性工作，非训练新模型）
  • 关键训练观察: Massive values 从第一层就出现，并非在 training 后期才形成。作者指出 "massive value is gradually formed through training rather than being solely caused by the addition of RoPE (since RoPE itself is not trainable)"——即 RoPE 提供结构性诱因，训练过程逐步放大
  • Scaling law 观察: 本文未做 scaling law 分析，但 Table 3 显示所有测试的 RoPE 模型（从 GPT-NeoX 到 Llama3-8B）均有此现象，暗示其与模型规模无关

3. Quantization & Compression

这是本文的关键应用领域：

量化方法对比:

  • AWQ (W4A16, activation-aware weight quantization): 通过 per-channel scaling 保护"重要"权重通道，有效保留 massive values。在所有 benchmark 上性能接近原始模型
  • SmoothQuant (W8A8): 使用平滑因子 $S$ 在权重和激活之间重新分配 outlier，通过数学等价变换保护 massive values。性能同样良好
  • GPTQ (W4A16, 基于 Hessian 的二阶量化): 不特别处理 massive values。在参数知识任务上正常，但上下文理解任务（GSM8K, AQUA）降至约 75% 归一化准确率

Figure 5: Quantization Method Impact

!Figure 5: Quantization Comparison

解读: 该图清晰地展示了三种量化方法的差异模式。AWQ 和 SmoothQuant 的雷达图几乎完全覆盖了 vanilla baseline 区域。GPTQ 在参数知识任务（右侧）维持良好，但在上下文任务（左侧 GSM8K/AQUA 区域）出现明显内缩。这验证了 massive values → 上下文理解的因果链。

KV cache 量化启示:

  • K 矩阵中包含 concentrated massive values，量化时需要特殊处理
  • 低频维度对精度更敏感，可考虑混合精度 KV cache (低频高精度，高频低精度)
  • 这与 KIVI 等 KV cache 量化方法的经验观察一致

Memory footprint 推算 (Llama3-8B):

  • Head dim = 128, 分为 64 个 RoPE 维度对
  • 低频 ~10-15% 维度承载主要 massive values
  • 量化这些维度需要至少 FP8，而其他维度可安全降至 INT4

4. Performance

Disruption 实验 (Table 1):

  • 上下文任务破坏: GSM8K -72.9pp, AQUA -43.8pp, Passkey -100pp, IMDB -84.4pp (Llama3-8B)
  • 参数任务破坏: Cities -11.2pp, Sports -20.5pp, Celebrity -22.0pp (温和下降)
  • 非 massive value 破坏: 所有任务 ±1% (几乎无影响)

PPL 和 Diversity 指标:

  • Massive value disrupted IMDB: PPL ~80 (vanilla ~10), 2-gram diversity ~0.1 (vanilla ~0.9)
  • 非 massive value disrupted: 与 vanilla 无显著差异

知识冲突实验 (如 Figure 4 所示):

  • 注入矛盾信息后，vanilla 模型准确率 ~50%（被误导至随机猜测）
  • Massive value disrupted 模型准确率 >50%（忽略误导上下文，回退参数知识）

5. Key Innovations

创新 1: Massive Values 的功能角色分离

  • 问题: LLM 中 outlier features 的功能一直不清楚
  • 机制: 通过 disruption 实验将 massive values 功能精确绑定到"上下文知识理解"而非"参数知识检索"
  • 影响: 跨 3 个模型、10 个 benchmark 一致验证
  • 通用性: 适用于所有 RoPE-based 模型

创新 2: RoPE → Massive Values 的因果链

  • 问题: 为什么 Q/K 有 massive values 但 V 没有
  • 机制: RoPE 仅作用于 Q/K，其频率衰减机制 ($\theta_j = 10000^{-2j/d}$) 导致低频维度积累大值
  • 影响: 在 10+ 个模型家族中完美的 RoPE ↔ massive values 对应关系
  • 通用性: 推广到多模态 RoPE 变体 (M-RoPE)

创新 3: 知识冲突反转实验

  • 问题: 如何证明 massive values 专门处理上下文而非一般性计算
  • 机制: 注入矛盾上下文后，破坏 massive values 的模型反而表现更好（回退到正确的参数知识）
  • 影响: 提供了优雅的反证法验证

7. Deployment Considerations

  • 量化推荐: 对于上下文理解密集的部署场景 (RAG, agent)，优先使用 AWQ 或 SmoothQuant 而非 GPTQ
  • KV cache 压缩: 不建议对 K cache 的低频维度 (~15% of head dim) 进行激进量化
  • 开源可用: 代码在 https://github.com/MingyuJ666/Rope_with_LLM
  • LoRA/QLoRA: 本文暗示 fine-tuning 时 Q/K 的低频维度可能比高频维度更需要被更新

8. Position in Model Landscape

本文是 LLM mechanistic interpretability 领域的重要贡献，填补了以下空白：

  • 将 outlier/massive value 现象从"量化干扰因素"重新定义为"功能性信号"
  • 建立了 RoPE → massive values → contextual understanding 的完整因果链
  • 连接了 position encoding 设计、注意力机制内部工作方式、和实际应用性能

与 "Round and Round We Go" (Barbero et al., ICLR 2025) 互相印证，后者发现 RoPE 低频编码语义、高频编码位置。

本文为后续研究开辟了方向：

  1. 如何在训练中主动塑造 massive value 分布以提升特定能力
  2. 非 RoPE 位置编码 (ALiBi, NoPE) 是否有等价的功能性信号
  3. 可否设计 "massive-value-aware" 的注意力机制变体

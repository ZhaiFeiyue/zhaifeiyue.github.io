Seedance 2.0: Advancing Video Generation for World Complexity

ByteDance Seed Team | 2026-04 | https://arxiv.org/abs/2604.14148 Category: llm | Tags: video-generation, audio-video-joint, MMDiT, diffusion-transformer, multimodal, evaluation-report Read: 2026-04-18

Core Contribution

Seedance 2.0 是字节 ByteDance Seed 团队的"原生 (native) 多模态音视频联合生成基模"，在统一架构下首次把 text / image / video / audio 四模态输入 与 同步音频+视频输出 集成在一个模型中，并在 SeedVideoBench 2.0、Arena.AI 双榜上以 Elo 1450 (T2V) / 1449 (I2V) 的成绩超越 Sora 2 Pro / Veo 3.1 / Kling 3.0 / Wan 2.6。

⚠️ 重要前置：这篇 26 页的 arXiv 文档本质上是一份"模型发布 + 评测报告" (model card style)，并非传统 technical report。全文 没有任何架构 / 训练 / 数据 / 算力披露——section 1 是 capabilities 罗列，section 2 是大量评测表 (T2V 30+ 类、I2V 复杂分类、R2V 任务矩阵、Arena.AI 截图)，section 3 是 280+ 作者署名。架构和训练细节需从前作 Seedance 1.5 pro [arXiv:2512.13507] 倒推。

Summary

问题动机。视频生成在 2025-2026 进入"原生音视频联合生成"赛道：Wan 2.5 / Kling 2.6 / Sora 2 / Veo 3.1 都已支持音频输出，但大多采用"视频先生成，音频后配"或"两个独立模型拼接"，导致 (1) 唇同步漂移 (2) 音轨缺乏与视觉节奏的耦合 (3) 多模态参考输入支持有限 (大多只支持文本或单图)。Seedance 团队的目标是把 4 输入模态、AV 联合生成、参考/编辑/续写 6 类任务，全部装入一个统一架构。

方法。Seedance 2.0 沿用 Seedance 1.5 pro 的 dual-branch MMDiT 架构 (Esser et al. SD3-style 多模态 Diffusion Transformer)，其中视频和音频各有一支 DiT branch，通过 cross-modal joint module 在每个去噪 step 做深度交互，文本以 cross-attention / AdaLN 注入。后训练用 SFT + RLHF，奖励模型采用 RewardDance / DanceGRPO 系列 (字节自家的 GRPO on visual generation)。推理侧通过多阶段蒸馏 + 量化 + 并行实现 ≥10× 加速 (Seedance 2.0 Fast 变体)。所有 4-15s、480p/720p、双声道 binaural 音频、3 视频 + 9 图 + 3 音频参考的能力，都装在这一套架构里。

结果。在 SeedVideoBench 2.0 上：

  • T2V：6 维度全部第一，Motion Quality 3.75 / 5 (runner-up Kling 3.0 仅 3.10)，Audio Quality 3.67 (其它都 < 2.9)；30 类 fine-grained 中 29 类排第一
  • I2V：6 维度全部第一，Motion Quality satisfaction 43.88% 是 Kling 3.0 的 3.6×；Audio Prompt Following satisfaction 63.52% 是次优的 1.7×、Kling 2.6 的 11×
  • R2V：22 个 multi-modal 任务支持 20 个 (其它最多 13/22)；7 个任务 (VFX/Creative reference + Continuation/Extension) 是字节独家
  • Arena.AI 公开榜：T2V Elo 1450 (领先 veo-3.1-audio-1080p 79 分)、I2V Elo 1449 (领先 grok-imagine-video 29 分)，且都是 720p 输出击败 1080p 对手

Key Findings

  • 唯一在所有评测维度全面领先的模型：T2V/I2V/R2V 三大任务、视觉+音频+多模态任务依从 6 个核心维度，没有一个被竞品反超。
  • 音频维度是最大护城河：T2V audio 三项 (quality / sync / prompt-following) 满意度 (≥ 4 分率) 全部 > 51%，Veo 3.1 / Kling 3.0 / Sora 2 Pro 全部 < 35%。
  • 多模态任务覆盖独家：VFX / Creative reference (3 种) + Continuation / Extension (4 种) 共 7 个任务没有任何竞品支持——这构成了真实的"产品级"差异化。
  • Motion Quality 子类 30 选 29 第一：Editing Rhythm 4.21、Multi-Entity Feature Match 4.43、Special Camera Shots 3.92——均超过 4.0，反映了对人体运动、物理一致性、镜头语言的系统性提升。
  • 720p 击败 1080p：Arena.AI Elo 第一是在 720p 输出实现的，说明运动+视觉一致性带来的感知质量提升超过分辨率。
  • 诚实承认的不足：multi-subject 一致性、文本还原精度、复杂编辑、video extension (Veo 3.1 在 task following 上 2.78 vs Seedance 2.0 仅 1.93) 仍有差距。

Limitations

  • 架构 / 训练完全不披露：本论文不提供任何模型规模、训练 token 数、GPU-hours、损失函数、并行策略——属于"产品发布稿"性质，需引用前作 1.5 pro 才能反推。
  • video extension 仍输 Veo 3.1：唯一一个 Seedance 2.0 不领先的子任务 (1.93 vs 2.78 task following)，说明在"既要兼容任意上传视频又要扩展"上还没找到最佳方案。
  • multi-subject 一致性 / 文本还原精度 论文自承仍有提升空间。
  • 音频 Chinese Opera prompt-following 仅 2.50 (T2V)，是 Seedance 2.0 audio 维度最弱项。
  • 未与 HunyuanVideo 1.5 / Wan-Animate / Step-Video 等开源 SOTA 对比——只对标商业闭源模型 (Veo / Sora / Kling)，对学术界友好度低。
  • 没有效率/成本数据：4-15s 视频的生成时延、GPU 成本、Fast 变体相对完整版的质量损失曲线均未给出。

Infrastructure Impact

  • Algorithm: 沿用 GRPO 在视觉生成上的扩展 (DanceGRPO, Flow-GRPO)，validation: video-audio 联合 RLHF 是可行的新方向；多阶段蒸馏 (Hyper-SD / RayFlow / MeanFlow) 是必备 toolkit。
  • Kernel: 长视频 + 双声道音频联合去噪意味着每步要同时 forward 两个 DiT branch + cross-attention，对 attention kernel 的"非对称序列长度跨模态"是新挑战 (类似 SLA paper 的方向)。
  • Framework: 4-15s × 720p × 双声道音频 = 大状态推理；如何在 MMDiT joint denoising 中做 KV cache、何时切换 fast variant、如何调度多模态参考输入，都是 inference framework 层面的新问题。
  • LLM: 对 LLM-as-foundation 思路是个反例——视频/音频联合生成当前最佳路径是 dedicated MMDiT，而非"扩 LLM 上下文吃所有模态"。
  • Agent: 视频生成 agent (例如自动剧本→分镜→视频) 的能力天花板上移；R2V 22 任务支持丰富了 agent tool-use 选项。
  • Hardware: 没有具体硬件信息，但 1.5 pro 提到 "10× inference acceleration via quantization + parallelism"，说明字节内部有 dedicated 推理优化栈 (可能与 Seedream 系列共用)。

Phase 2 重申: 时代定位 + 约束推导 + 核心壁垒

时代定位

视频生成在 2025-2026 年完成两次范式跃迁：

  1. 第一跃迁 (2024 年底): 从短视频片段 → 长视频 + 高分辨率 (Sora, Kling 1.x)
  2. 第二跃迁 (2025 中-2026 初): 从纯视频 → 原生音视频联合生成 (Wan 2.5 / Kling 2.6 / Sora 2 / Veo 3 / Seedance 1.5)

    3. Seedance 2.0 处在第二跃迁完成后的"产品化深水区"——同代模型 (Wan 2.6 / Kling 3.0 / Sora 2 Pro / Veo 3.1) 都已经具备 AV 联合能力，竞争已经从"能否生成"转向"multi-modal task coverage 广度 + 细粒度评测维度的均衡度"。Seedance 2.0 的策略不是单点突破，而是把 22 个 R2V 任务、30+ Motion 子类、17 个 Audio 子类全部刷到第一——这是典型的"工程深水区"打法。

    约束推导（"为何不可 X？"）

    设计选择备选方案为何不选
    统一架构 dual-branch MMDiT视频模型 + 音频模型独立训练后拼接 (Wan 2.5 / Sora 2 走的部分路径)拼接路线唇同步漂移、音轨与视觉节奏割裂；MMDiT joint denoising 让 AV 在每一步都能互相约束
    MMDiT (跨模态共享 attention)UNet + 后处理 audio 解码器UNet 对 long-range 时序依赖弱、scaling 性能不如 DiT；跨模态共享 attention 是 Stable Diffusion 3 已验证的有效方案
    后训练 RLHF仅 SFT视频质量是高度主观的多维评判，纯 SFT 无法对齐"vivid" / "narrative coherence" 这类 perceptual 维度，必须 RM + RL
    多阶段蒸馏 (用于 Fast 变体)单步生成 (one-step diffusion via consistency)视频 + 音频 joint denoising 一步生成质量损失太大，多阶段蒸馏可以把 NFE 从 ~50 压到 ~5-8 同时保留质量
    480p / 720p 原生输出而非 1080p直接训 1080p1080p 训练算力 + 推理时延 4× 增加，但 Arena Elo 显示 720p 已经能击败 1080p 对手——感知质量 ≠ 分辨率

    核心技术壁垒（推断，论文未明说）

    基于 1.5 pro paper 和 Seedance 系列的演进轨迹，真正难复现的应该是：

    1. 多模态数据配方 — 1.5 pro 明说 "comprehensive audio-visual data framework with multi-stage curation pipeline, advanced captioning system, and curriculum-based scheduling"。视频+音频联合训练数据的清洗、对齐、字幕、配音标注，是 ByteDance 短视频生态的独特资产，是任何学术机构最难复制的部分。
    2. Cross-modal joint module 的具体设计 — paper 没有给细节，但能让 4-15s 双声道音频与视频帧级对齐，需要某种时间对齐的 cross-attention mask + 共享时间编码。这一块即使有 MMDiT 论文做参考，也需要大量内部实验才能调通。
    3. 多任务统一训练 — T2V/I2V/T2VA/I2VA/R2V/Editing/Continuation/Extension 在一个模型里 work，意味着 prompt template、conditioning injection、negative samples 设计需要复杂的 task-aware mixing。这不是简单的 task tag。

      4. 质疑假设

      • 评测有没有偏向自家？论文用了 SeedVideoBench 2.0 (字节自研) + 字节自家专家 + 第三方 Arena.AI 三套评测。Arena.AI 是关键独立证据 (Elo 1450 vs 1371)，但其它分数仍来自字节内部。
      • 30 个 Motion 子类、17 个 Audio 子类的划分是否合理？是否针对性挑选了 Seedance 2.0 强项？没有公开 prompt 集，无法复核。
      • Seedance 2.0 Fast 的质量损失论文完全不提——Fast 通常意味着 NFE 减少和质量退化，但本文不给曲线。
      • 去重/污染检查：与 Arena 用户输入是否有数据泄漏？无说明。

      设计绑定批判

      • 强绑定 ByteDance 内部数据生态：训练数据来自抖音/TikTok/剪映 (Jianying) 的视频+音频 corpus，外部研究者无法复现这一基础。
      • 强绑定 RewardDance / DanceGRPO 系列内部 RM：奖励模型本身是字节系列论文 (RewardDance arXiv:2509.08826)，外部要复现需要先复现 RM 再复现 RL。
      • 强绑定 Volcano Engine 推理栈：Fast 变体的 ≥10× 加速需要字节自研推理栈 (量化 + 并行)，第三方 ckpt 即便发布也很难得到同等速度。
      • 强绑定 Seedream 图像模型生态：image conditioning 大概率复用 Seedream 编码器/VAE (Seedream 2.0/3.0/4.0 也是字节出品)，独立复现 image branch 不容易。

      2g. Training Barrier (核心壁垒)

      最难复现的不是架构 (MMDiT 已开源、SD3 论文公开)，而是：

      1. 音视频联合数据 curation pipeline — 字节有海量短视频+音频对齐数据，外部无法获取等同规模、等同质量、等同 caption density 的语料
      2. 多阶段 distillation 调参 — 多步 diffusion 蒸馏到 5-8 步同时保 quality 是高度炼丹的过程，paper 没给细节
      3. 多任务统一 prompt template — 22 个 multi-modal 任务在同一模型里 work，需要复杂的 task encoding + conditioning 调度

        4. Performance

        详见上方 Key Tables (T1, T9, T25)。补充几个亮点：

        • T2V 30 类 Motion 子类 29 个第一，唯一并列的是"Group Coordinated Motion" (3.29 与 Kling 3.0 持平)
        • I2V 复杂指令 "Compound Multi-Instructions" 上 MQ 4.00 / VPF 3.75，比 Kling 3.0 高 1.0+
        • R2V Subject Reference image-based: task following 2.80 (100% 2-pt rate, 80% 3-pt rate)
        • R2V Video Editing 是最竞争激烈赛道，task following 上 Kling O1 (2.29) 略胜 Seedance (2.20)，但 Editing Consistency Seedance 3.75 vs Kling 3.0 3.09
        • Arena.AI Elo: T2V 1450 (±15) / I2V 1449 (±11)，Rank Spread 1↔1，720p 击败 1080p

        8. Position in Model Landscape

        • vs Sora 2 Pro: Seedance 2.0 在 motion stability、AV 联合、多模态参考全面领先；Sora 强在 abstract/creative prompt 解读
        • vs Veo 3.1: Seedance 在视频 + 音频均领先；但 video extension 任务 Veo 仍领先 0.85 分
        • vs Kling 3.0: 最接近的赛道是 Image Preservation (差 0.13)；其它维度 Seedance 全面领先
        • vs Wan 2.6 (开源对照): Wan 是当前开源最强 AV 模型，但与 Seedance 2.0 差距仍大 (Motion Quality 2.32 vs 3.35)
        • 填补的 gap: 真正"22 任务统一覆盖"的产品级模型；为 AV joint generation 设立了新的产品标杆

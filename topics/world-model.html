1. 主题缘起 #

该主题的历史基线来自 DreamerV3：它将 world-model RL 从“单域精调”推进到“跨域默认超参可用”，把训练稳定性做成可组合机制，成为后续路线比较与迁移的共同参照。 [2301.04104] [2301.04104] [2301.04104]

该主题在 2026 年集中成形，直接原因是世界建模研究从“跨域稳健学习”进一步转向“可执行的世界状态预测”，并且把动作可行性与几何一致性放进同一主问题。 [2301.04104] [2606.16533] [2606.18375] [2607.00678]

一条主线强调跨具身与长时稳定记忆，用课程学习和混合记忆结构提升长滚动一致性。 [2606.16533] [2606.16533]

另一条主线强调多视角几何闭合，把跨视角通信与几何监督视为必须同时满足的条件。 [2606.18375] [2606.18375]

第三条主线强调世界到动作的结构桥接，用 z->m->a 和训推对齐机制缓解 rollout 分布偏移。 [2607.00678] [2607.00678]

与此同时，服务侧论文反复显示瓶颈会从模型本体迁移到内存、调度、批处理和拓扑放置，因此“world model 可部署性”成为同等重要的边界条件。 [2602.21477] [2603.16104] [2604.15186] [2509.02121]

3. 时间线 #

  • 2023-01：DreamerV3 建立跨域 world-model RL 的稳健默认基线，以损失平衡、尺度归一化和分布平滑实现 150+ 任务与 Minecraft 的统一训练配方。 [2301.04104] [2301.04104] [2301.04104]
  • 2025-09：Halo 将批量 agentic workflow 编译为全局执行图，证明 workflow-level 优化可显著影响总时延，建立世界模型上线时的调度基线。 [2509.02121] [2509.02121]
  • 2026-02：Pancake 把多 agent 长期记忆检索开销从主耗时压到低占比，显示外部记忆系统是世界模型闭环中的核心约束。 [2602.21477] [2602.21477]
  • 2026-03：Helium 用 query-plan 思路做跨调用前缀复用和全局排序，给出世界模型多阶段流水的结构化执行范式。 [2603.16104] [2603.16104]
  • 2026-04：Scepsy 用 Aggregate pipeline 建模多 LLM workflow 的资源份额稳定性，补上分数 GPU 与拓扑放置的部署路径。 [2604.15186] [2604.15186]
  • 2026-06：Kairos 提出 learn-maintain-run 原生栈，强化跨具身课程与长时记忆。 [2606.16533] [2606.16533]
  • 2026-06：PAIWorld 将多视角几何一致性定义为显式目标，突出 CVA + Geo-RoPE + REPA 的耦合。 [2606.18375] [2606.18375]
  • 2026-07：ABot-M0.5 把世界潜变量到动作执行的桥接机制前置，强调 Dream Forcing 的训推一致性收益。 [2607.00678] [2607.00678]

5. 技术线交错 #

算法选择会直接约束系统设计：长时记忆与多视角条件使上下文、检索和中间状态显著增大，执行层必须支持跨步骤复用与低成本更新。 [2606.16533] [2606.18375] [2602.21477]

动作桥接方法要求更严格的训推一致条件，系统如果只优化离线吞吐而忽略 rollout 条件匹配，会放大误差累积。 [2607.00678] [2607.00678]

workflow-aware 调度与前缀/图结构复用能放大世界模型收益，否则模型优势会被路由抖动与资源错配抵消。 [2603.16104] [2604.15186] [2509.02121]

内存层与模型层已形成双向耦合：模型越强调长期状态与多 agent 交互，越依赖分层记忆与并发一致性控制。 [2602.21477] [2602.21477]

6. 共识与分歧 #

共识 1：仅靠短时续写不足以支撑复杂任务，必须引入更强的状态持久机制或结构化约束。 [2301.04104] [2606.16533] [2606.18375] [2607.00678]

共识 2：DreamerV3 证明“默认超参跨域可用”可以成为稳定基线，但在具身场景需与几何一致和动作接口机制联合扩展。 [2301.04104] [2301.04104] [2606.18375] [2607.00678]

共识 3：训练和部署不再可分离，系统执行路径会反向决定模型实际效果上限。 [2606.16533] [2606.18375] [2607.00678]

分歧 1：几何一致优先 vs 动作桥接优先。PAIWorld强调跨视角几何闭合是前提，ABot-M0.5强调 z->m->a 与 Dream Forcing 才是闭环关键。 [2606.18375] [2607.00678]

分歧 2：理论解释强度。DreamerV3 明确“无形式化收敛证明，仅实证”；Kairos给出风险差与记忆分解叙述，PAIWorld更依赖实证消融，ABot-M0.5在训推对齐上证据更直接但几何与超长时理论较弱。 [2301.04104] [2606.16533] [2606.18375] [2607.00678]

分歧 3：部署抽象粒度。Scepsy偏向 aggregate 份额建模，Helium/Halo偏向 plan-level 图优化，Pancake偏向记忆检索子系统专精。 [2604.15186] [2603.16104] [2509.02121] [2602.21477]

7. Open challenges (根本性困难) #

困难 1：几何一致、长时一致与动作可执行性的统一目标仍缺可验证闭式框架。 当前方法分别给出局部最优路径，但尚无统一指标可同时约束三者且能稳定训练。 [2606.18375] [2606.16533] [2607.00678]

困难 2：真实闭环性能的主导瓶颈跨层迁移。 即便模型侧改进成立，线上收益常被记忆、路由、拓扑和并发一致性吞噬，这不是单一组件可解问题。 [2606.16533] [2606.18375] [2602.21477] [2604.15186]

困难 3：动态工作流与静态优化之间存在方法鸿沟。 许多优化依赖可预测结构和离线统计，而真实 agent 轨迹常在运行中改变路径。 [2603.16104] [2509.02121] [2604.15186]

困难 4：可复现证据不足。 多数核心结果仍缺完整实现与大规模可审计复现，导致方法可迁移性边界不清晰。 [2606.16533] [2606.18375] [2607.00678]

8. 成熟度判断 #

该主题整体处于“研究前沿向早期工程迁移”阶段，尚未进入可标准化大规模生产状态。 [2606.16533] [2606.18375] [2607.00678]

加速信号包括：世界建模、动作建模和服务调度三条线都已有明确增益证据，且彼此接口逐渐清晰。 [2606.16533] [2606.18375] [2607.00678] [2604.15186]

放缓信号包括：跨层评测标准未统一、动态场景泛化不足、实现开源与生产验证缺口较大。 [2509.02121] [2603.16104] [2604.15186]

趋势判断为“加速但分叉”：短期会继续沿几何一致与动作桥接两支并进，中期在 workflow 级执行层收敛。 [2606.18375] [2607.00678] [2604.15186]

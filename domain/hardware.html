Hardware 品类全景：数据中心 GPU 架构演进 (2020–2025)

1. 现状快照

2025 年数据中心 GPU 进入 AMD CDNA 4 (MI355X) vs NVIDIA Blackwell (B200) 的正面对决时代。双方在 FP8 峰值算力上已基本持平（~5 PFLOPS），竞争焦点从纯算力转向互联规模、显存容量、能效比和软件生态的全栈博弈 [L2:gpu-arch-comparison-2025]。AMD 凭借 chiplet 封装路线和 288 GB 显存在推理市场崛起；NVIDIA 凭借 NVLink5 + NVL72 机架级互联和 CUDA 全栈生态在训练市场保持系统性优势 [L2:nv-blackwell-whitepaper]。精度格式已下探至 FP4/MXFP4 级别，两家均提供硬件原生 4-bit 浮点支持 [L2:amd-cdna4-whitepaper] [L2:nv-blackwell-whitepaper]。

2. 主线与分支

2.1 GPU 架构代际演进

AMD CDNA 路线 (2020–2025)：从 CDNA 1 (MI100) 的单 die Matrix Core 首发，经 CDNA 2 (MI250X) 的 MCM 双 GCD 封装驱动 Frontier Exascale 超算，到 CDNA 3 (MI300X) 的革命性 chiplet 3D 堆叠（8 XCD + 4 IOD），再到 CDNA 4 (MI355X) 全面押注 AI 精度下探 [L2:gpu-arch-comparison-2025]。

  • CDNA 1 (MI100, 2020)：7nm 单 die，120 CU，首次引入 Matrix Core Engine 和 MFMA 指令族，FP16 Matrix 184.6 TFLOPS，32 GB HBM2 [L2:amd-cdna1-whitepaper]。将 GPU 从图形通用架构分化为计算专用架构（移除所有图形固定功能单元）[L2:amd-cdna1-whitepaper]。
  • CDNA 2 (MI250X, 2021)：7nm MCM 双 GCD，220 CU，新增 FP64 Matrix Core（95.7 TFLOPS），128 GB HBM2e (3.2 TB/s)，驱动 Frontier——全球首台 Exascale 超算 [L2:amd-cdna2-whitepaper]。封装内 400 GB/s 双向带宽实现近乎线性的双 GCD 扩展 [L2:amd-cdna2-whitepaper]。
  • CDNA 3 (MI300X/MI325X, 2023–2024)：5nm+6nm chiplet 3D 堆叠，304 CU，首次支持 FP8 (OCP E4M3/E5M2) 和 TF32，FP8 达 2,615 TFLOPS，192–256 GB HBM3/3E，256 MB Infinity Cache 提供 17.2 TB/s 带宽放大 [L2:amd-cdna3-whitepaper]。单 XCD 仅 0.66 TB/s HBM 带宽，数据局部性（L2 hit）至关重要 [L2:amd-cdna3-whitepaper]。
  • CDNA 4 (MI355X, 2025)：3nm+6nm chiplet，256 CU（per-CU 矩阵吞吐翻倍），FP8 达 5.0 PFLOPS，MXFP4/MXFP6 达 10 PFLOPS，LDS 扩容至 160 KB/CU (2.5x)，288 GB HBM3E (8.0 TB/s) [L2:amd-cdna4-whitepaper]。大胆移除 TF32 硬件并将 FP64 Matrix 减半，标志 AI 优先的明确转向 [L2:amd-cdna4-whitepaper]。

NVIDIA 路线 (2020–2025)：

  • Ampere (A100, 2020)：7nm 单 die 826mm²，108 SM，第三代 Tensor Core 引入 TF32 和 FP64 TC，首创 2:4 结构化稀疏和 MIG 多实例 GPU，FP16 TC 312 TFLOPS，80 GB HBM2e (2.04 TB/s)，40 MB L2 [L2:nv-ampere-a100-whitepaper]。TF32 使现有 FP32 模型零代码修改即获得 10x V100 加速 [L2:nv-ampere-a100-whitepaper]。
  • Hopper (H100, 2022)：4N 定制工艺，132 SM，第四代 TC 引入 FP8 + Transformer Engine 自动精度管理，TMA 硬件 tensor 搬运，Thread Block Clusters + DSMEM 跨 SM 协作，FP8 达 1,979 TFLOPS，80 GB HBM3 (3.35 TB/s)，NVLink 4 900 GB/s + NVLink Switch 256-GPU 域 [L2:nv-hopper-h100-whitepaper]。
  • Blackwell (B200, 2024–2025)：4NP 双 die 统一 GPU（NV-HBI 10 TB/s），208B 晶体管，第五代 TC 支持 FP4 + NVFP4 两级微缩放 + TMEM 新存储层次，FP4 达 9,000 TFLOPS，192 GB HBM3e (8.0 TB/s)，NVLink5 1.8 TB/s + NVL72 机架级 72-GPU 域 (130 TB/s) [L2:nv-blackwell-whitepaper]。

2.2 计算单元与 Matrix Core 演进

AMD 的 CU 保持 wavefront=64、4×16-wide SIMD 的基本执行模型 [L2:amd-cdna1-whitepaper]，Matrix Core 从 CDNA 1 的 FP32/FP16/BF16/INT8 逐代扩展到 CDNA 4 的 MXFP8/6/4 全覆盖，per-CU 矩阵吞吐从 1,024 FLOPS/clk/CU (FP16, CDNA 1) 增长到 4,096 FLOPS/clk/CU (FP16, CDNA 4)——四代实现 4x 提升 [L2:amd-cdna4-whitepaper]。

NVIDIA 的 Tensor Core 从 V100 的 warp 级 mma.sync 演进到 Hopper 的 warp-group 级 wgmma（128 线程协作），再到 Blackwell 的 warp 级 tcgen05（32 线程独立发射）[L2:2512.02189]。微基准测试实证：Blackwell tcgen05.mma 单指令延迟仅 11.0–11.4 cycles（近乎恒定），而 Hopper wgmma 延迟随 tile 宽度线性增长 32–128 cycles，证明 Blackwell 采用空间阵列设计而非时域流水线 [L2:2512.02189]。

2.3 显存层次与带宽

五年间 HBM 从 HBM2 (MI100: 32 GB, 1.23 TB/s) 演进到 HBM3E (MI355X/B200: 288/192 GB, 8.0 TB/s)——容量增长 9x，带宽增长 6.5x [L2:gpu-arch-comparison-2025]。

AMD 独有的 Infinity Cache（256 MB, 17.2 TB/s）作为 memory-side cache 提供显著的带宽放大——L2 提供 6.5x HBM 带宽 per XCD，Infinity Cache 再提供 3.2x L2 带宽的全芯片缓冲 [L2:amd-cdna3-whitepaper]。NVIDIA 的对应策略是扩大 L2（A100 40 MB → H100 50 MB → B200 96 MB）和引入 TMEM 专用 tensor 存储 [L2:nv-blackwell-whitepaper]。微基准实测 TMEM cache-miss 延迟 420 cycles，比 Hopper 全局内存路径 ~1000 cycles 降低 58%，per-SM 读带宽达 16 TB/s [L2:2512.02189]。

2.4 互联与规模化

节点内互联：AMD 采用 Infinity Fabric mesh（MI300X: 7 links, 896 GB/s → MI355X: 7 links, 1,075 GB/s），8-GPU 全互联但每对 GPU 仅 1 条链路 [L2:amd-cdna4-whitepaper]。NVIDIA 采用 NVSwitch 提供 non-blocking all-to-all 连接（H100: 900 GB/s → B200: 1,800 GB/s）[L2:nv-blackwell-whitepaper]。

跨节点扩展：NVIDIA 拥有 NVLink Switch System（H100: 256-GPU 域, B200: NVL72 72-GPU 机架域 130 TB/s，可扩展至 576 GPU）[L2:nv-blackwell-whitepaper]，这是 AMD 完全缺失的能力——AMD 跨节点仅依赖标准 InfiniBand/Ethernet [L2:amd-cdna4-whitepaper]。

实测数据验证了拓扑差异的影响：MI300X 8-GPU All-Reduce 达 448 GB/s（~70% 理论峰值），而 H100 NVSwitch 达 ~85% 利用率 [L2:2510.27583]。Infinity Fabric mesh 的通信带宽对 tensor parallelism degree 敏感，NVSwitch 几乎不受影响 [L2:2510.27583]。

2.5 数值格式与精度下探

精度演进路径清晰呈现：FP32/FP16 → +BF16/TF32 → +FP8 → +FP4/MXFP [L2:gpu-arch-comparison-2025]。

  • FP8：Hopper 首发 E4M3/E5M2 双变体 + Transformer Engine per-tensor scaling [L2:nv-hopper-h100-whitepaper]；AMD 在 CDNA 3 跟进 OCP FP8 标准 [L2:amd-cdna3-whitepaper]。
  • FP4/MXFP4：Blackwell 以 NVFP4 两级微块缩放（16 元素 E4M3 scale + 全 tensor FP32 scale）实现 9,000 TFLOPS [L2:nv-blackwell-whitepaper]；MI355X 以 OCP MX 标准 MXFP4（32 元素共享 8-bit 指数）达 10 PFLOPS [L2:amd-cdna4-whitepaper]。
  • MXFP6（AMD 独有）：E3M2/E2M3 中间精度，与 MXFP4 共享相同 10 PFLOPS 吞吐但提供 50% 更多尾数位——这是严格的精度-性能最优点 [L2:amd-cdna4-whitepaper]。

微基准实测 Blackwell FP4 达 7,702.5 TFLOPS（96.3% 峰值利用率），但 perplexity 退化 +7.7–9.1%，MoE 模型从量化中获益更大（2.69x vs 2.50x）[L2:2512.02189]。FP32 accumulator 使 Tensor Core 吞吐减半（1,929 → 965 TFLOPS），accumulator datapath 是吞吐瓶颈 [L2:2512.02189]。

2.6 封装哲学分歧

AMD 走 chiplet 异构集成：多个小 XCD（计算 die，最新 3nm）+ IOD（IO die，成熟 6nm）3D 堆叠，优势是良率高、工艺可独立演进 [L2:amd-cdna3-whitepaper]。CDNA 4 将 IOD 从 4 个精简为 2 个，减少跨 IOD 通信跳数约 14% [L2:amd-cdna4-whitepaper]。

NVIDIA 走 双 die 统一 GPU：两个接近光刻极限的大 die + NV-HBI 10 TB/s 超高速互联，对软件完全透明 [L2:nv-blackwell-whitepaper]。trade-off 是封装成本高但 die 内通信延迟低 [L2:gpu-arch-comparison-2025]。

3. 核心 trade-off 轴

3.1 算力 vs 带宽（Ops:Byte 比率持续攀升）

Ops:Byte 比率从 A100 的 153 (FP16) 攀升到 MI355X/B200 的 ~315/281 (FP16) 和 629/563 (FP8)，MXFP4 更高达 1,258 [L2:gpu-arch-comparison-2025] [L2:amd-cdna4-whitepaper]。这意味着除了大矩阵 GEMM（M≥1,258 for MXFP4），几乎所有操作都是 memory-bound [L2:nv-blackwell-whitepaper]。FP4 的真正价值不在峰值 TFLOPS，而在用更少 bit 传输相同语义信息——同样 8 TB/s 带宽可传输 2x FP4 元素 vs FP8 [L2:nv-blackwell-whitepaper]。

3.2 理论峰值 vs 实际利用率

MI300X 的实测故事最具教育意义：理论 FP8 峰值 2,615 TFLOPS，实际仅达 45%（~1,180 TFLOPS），而 H100/B200 达 93–96% [L2:2510.27583]。频率-效率分离实验精确量化了原因：运行频率从 2100 MHz 降至 ~1200 MHz（~42% 降频）× 软件效率 80–85% ≈ 47% 利用率 [L2:2510.27583]。软件栈成熟度是核心瓶颈，而非硬件设计缺陷 [L2:2510.27583]。对比之下，Blackwell B200 在所有精度上达到 96–99% 理论峰值利用率 [L2:2512.02189]。

3.3 显存容量 vs 互联带宽

AMD 每一代都提供更大显存：MI355X 288 GB vs B200 192 GB (+50%) [L2:gpu-arch-comparison-2025]。NVIDIA 的回答是 NVL72 的 13.5 TB 聚合显存（72 × 192 GB）[L2:nv-blackwell-whitepaper]。对单卡推理，AMD 288 GB 可容纳 ~400B FP8 模型或 ~800B MXFP4 模型 [L2:amd-cdna4-whitepaper]。对大规模训练，NVIDIA NVLink5 (1.8 TB/s per GPU) + NVL72 (130 TB/s 域内带宽) 的互联优势是 AMD 的 Infinity Fabric (1,075 GB/s per GPU, 8-GPU 域) 无法匹配的 [L2:amd-cdna4-whitepaper]。

3.4 AI 算力 vs HPC 性能的取舍

CDNA 4 将 FP64 Matrix 从 163.4 TFLOPS 削减至 78.6 TFLOPS（减半），同时移除 TF32 硬件，腾出面积给 MX 格式和矩阵资源翻倍 [L2:amd-cdna4-whitepaper]。Blackwell 的 FP64 仅 37 TFLOPS（可能已移除 FP64 TC）[L2:nv-blackwell-whitepaper]。两家都在用 HPC 性能换 AI 算力，但 AMD 的 FP64 起点更高（CDNA 3 MI300X: 163.4 TF vs H100: 66.9 TF），因此即使减半后仍保持 78.6 TF [L2:gpu-arch-comparison-2025]。

3.5 功耗与能效

TDP 从 A100 400W → H100 700W → B200 1,000W / MI355X 1,400W，散热从风冷向液冷过渡成为强制要求 [L2:nv-blackwell-whitepaper]。FP8 能效比：B200 4.50 TFLOPS/W vs MI355X 3.60 TFLOPS/W，NVIDIA 领先 25% [L2:amd-cdna4-whitepaper] [L2:nv-blackwell-whitepaper]。MI350X 风冷版 (1,000W) 在 FP8 4.61 TFLOPS/W 上优于 MI355X 液冷版 (1,400W) 的 3.60——额外 400W 功耗仅换来 ~9% 性能提升，深陷收益递减 [L2:amd-cdna4-whitepaper]。

4. 冲突与争议

4.1 MI300X 理论优势 vs 实际性能倒挂

MI300X 在 FP8 理论峰值上领先 H100 32%（2,615 vs 1,979 TFLOPS），但端到端 Llama 70B FP8 推理仅达 H100 的 37–66% [L2:2510.27583]。分离实验表明约一半归因于频率降频（功耗管理），另一半归因于软件效率不足 [L2:2510.27583]。反直觉结论：MI300X 上使用 FP16 反而可能优于 FP8，因 FP16 的更大 memory footprint 恰好落在 MI300X 高带宽区间 [L2:2510.27583]。

4.2 Chiplet vs Monolithic/Dual-Die 的封装路线之争

AMD chiplet 路线（8 个小 die + IOD）的跨 XCD 通信需经过 IOD，引入额外延迟——大矩阵尺寸下 MI300X 出现性能回落（利用率从 50% 降至 ~40%），暗示 chiplet 架构的数据移动瓶颈 [L2:2510.27583]。NVIDIA 双 die 方案（NV-HBI 10 TB/s）保持了对软件的完全透明，但封装复杂度和成本更高 [L2:nv-blackwell-whitepaper]。未解决：哪种路线在下一代（3nm 以下工艺）更具可扩展性。

4.3 精度格式标准化之争

NVIDIA 的 NVFP4（16 元素微块 E4M3 scale + 全 tensor FP32 scale）vs OCP MX 标准的 MXFP4（32 元素 power-of-2 scale）[L2:nv-blackwell-whitepaper]。AMD 完全拥抱 OCP MX 标准，NVIDIA 在兼容 MX 的同时推自有 NVFP4 格式。AMD 独有的 MXFP6 提供 FP8/FP4 之间的精度-性能折中，NVIDIA 无等价格式 [L2:amd-cdna4-whitepaper]。行业标准尚未收敛。

4.4 软件栈成熟度差距——是否可以缩小？

MI300X 实测 45% 利用率 vs NVIDIA 93%+ [L2:2510.27583]，Blackwell 所有精度 96–99% 利用率 [L2:2512.02189]。AMD ROCm 生态正在快速追赶（vLLM/SGLang Day 0 支持 MI350 Series [L2:amd-cdna4-whitepaper]），但 NVIDIA 拥有 CUDA + cuBLAS + cuDNN + TensorRT-LLM + Transformer Engine (硬件级) 的全栈垂直整合 [L2:nv-hopper-h100-whitepaper]。hipBLASLt 的 80–85% 软件效率表明 AMD GEMM kernel 仍有 15–20% 优化空间 [L2:2510.27583]。

5. 未探索角落

  1. MI355X 实际利用率：CDNA 4 已发布但缺乏独立第三方实测数据。MI300X 的 45% 利用率教训 [L2:2510.27583] 能否在 MI355X 上改善，取决于 ROCm 和 MXFP kernel 的成熟度。
  2. Blackwell 双 die NUMA 效应：所有微基准在单 GPU/单 SM 上测量，dual-die NV-HBI 通信的实际延迟和对真实 workload 的影响未被系统量化 [L2:2512.02189]。
  3. FP4 per-layer 精度退化分布：FP4 perplexity 退化 +7.7–9.1% 是模型级平均值，per-layer 退化分布未分析——哪些层可容忍 FP4、哪些需 FP8 的精细粒度指导缺失 [L2:2512.02189]。
  4. MXFP6 实际 workload 验证：MXFP6 理论上是严格最优点（同 MXFP4 吞吐 + 更高精度），但缺乏端到端 LLM 推理的精度-性能 benchmark [L2:amd-cdna4-whitepaper]。
  5. 跨节点 GPU 直联方案：AMD 缺乏 NVL72 等价方案，跨节点仅靠标准网络 [L2:amd-cdna4-whitepaper]。未来是否会推出 Infinity Fabric Switch 是关键未知数。
  6. FP6 软件生态：Blackwell 硬件支持 FP6 但缺乏软件工具链 [L2:2512.02189]，AMD 有 MXFP6 硬件和 ISA 支持 [L2:amd-cdna4-whitepaper]——两家的 6-bit 格式实际可用性均待验证。
  7. 功耗与 TCO 量化：MI300X 性能评测未量化 power/TCO 差异 [L2:2510.27583]，而这对采购决策至关重要。MI355X 1,400W vs B200 1,000W 的差距在大规模部署中对 TCO 影响显著。
  8. MoE 和长上下文推理场景的架构适配：端到端评测仅覆盖 Llama 70B dense 模型 [L2:2510.27583]，MoE 和 long-context workload 对不同架构的偏好尚未系统研究。

    9. 6. 时代定位

    2020–2025 年的数据中心 GPU 架构演进可划分为三个阶段：

    Phase 1: AI 加速器标准奠基（2020–2021）：A100 定义了"数据中心 AI GPU"的完整功能集——TC 全精度覆盖 + 结构化稀疏 + MIG + 大 L2 Cache [L2:nv-ampere-a100-whitepaper]。AMD CDNA 1/2 在 HPC FP64 领域找到差异化定位（MI250X 驱动 Frontier Exascale） [L2:amd-cdna2-whitepaper]。

    Phase 2: FP8 时代与 LLM 推理崛起（2022–2024）：Hopper H100 以 FP8 + Transformer Engine 定义了 LLM 训练的新标准 [L2:nv-hopper-h100-whitepaper]。MI300X 凭 chiplet 架构和 192 GB 显存在推理市场建立竞争力 [L2:amd-cdna3-whitepaper]。这一阶段的核心矛盾是 spec sheet 优势 vs 实际部署性能的鸿沟——MI300X 是最好的案例研究 [L2:2510.27583]。

    Phase 3: FP4/MXFP 时代与系统级竞争（2025+）：Blackwell 和 CDNA 4 同时将精度推至 4-bit。竞争焦点从单卡算力转向系统架构（NVL72 vs 8-GPU 全互联）、能效比（4.50 vs 3.60 FP8 TFLOPS/W）、显存容量（288 vs 192 GB）和软件生态（CUDA vs ROCm）[L2:gpu-arch-comparison-2025]。从架构设计哲学看，Blackwell 的 TMEM + tcgen05 代表了从"训练优先"到"推理优化"的范式转换 [L2:2512.02189]，而 CDNA 4 的 MXFP6 + 160 KB LDS 则提供了独特的精度-性能折中空间 [L2:amd-cdna4-whitepaper]。

    范式转换信号：两家都在主动牺牲 HPC 性能换取 AI 算力——CDNA 4 砍 FP64 Matrix 50%，Blackwell 可能移除 FP64 TC [L2:nv-blackwell-whitepaper] [L2:amd-cdna4-whitepaper]。数据中心 GPU 正从"通用加速器"收敛为"AI 专用推理/训练引擎"。

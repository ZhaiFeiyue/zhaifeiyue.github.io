Kernel 类别全景

1. 现状快照

2024–2026 年 GPU kernel 领域的核心叙事是低比特量化 attention 的代际跃迁与多 chiplet GPU 架构下的调度觉醒。清华朱军团队的 SageAttention 系列 [L2:2410.02367] [L2:2411.10958] [L2:2505.11594] [L2:2505.21136] 在三代迭代中将 attention kernel 从 FP16 推进到 INT8→INT4→FP4，吞吐量从 FlashAttention2 的 1× 提升到 5×。与此同时，GPU 硬件从单 die 走向多 chiplet（MI300X 8 XCD、Blackwell 双 die），迫使 kernel 开发者重新思考数据局部性和资源调度 [L2:2511.02132] [L2:2511.08083]。稀疏-线性混合 attention [L2:2509.24006] 则打开了 95% 稀疏度下无损加速的新范式。

2. 主线与分支

2.1 Attention 量化加速（主线）

这是当前 kernel 领域最活跃的研究方向，由 SageAttention 系列（清华）主导，四篇论文形成完整的技术演进链。

第一代：INT8 混合精度（SageAttention）。核心洞察是 K 矩阵的 channel-wise outlier 本质上是所有 token 共享的大偏置，通过 Smooth K（减去通道均值）消除后，INT8 per-block 量化的余弦相似度从 30.6% 恢复到 99.3% [L2:2410.02367]。$QK^\top$ 用 INT8 Tensor Core，$PV$ 保持 FP16 但使用 FP16 累加器（RTX4090 上 2× 快于 FP32 累加器），整体在 RTX4090 上达到 341 TOPS，2.1× 快于 FlashAttention2 [L2:2410.02367]。

第二代：INT4+FP8（SageAttention2）。将 $QK^\top$ 从 INT8 推进到 INT4（利用 Ada 架构的 mma.m16n8k64 指令），$PV$ 改用 FP8。三个关键创新：(1) 双重 Smoothing（Q 和 K 都减均值），将 INT4 CosSim 从 80% 提升到 99.46% [L2:2411.10958]；(2) Per-thread 量化——利用 MMA 指令的线程-数据映射关系，每个 GPU 线程只需一个量化 scale，精度接近 per-token 但零额外开销 [L2:2411.10958]；(3) 发现 NVIDIA Ada/Hopper 的 FP8 MMA 累加器实际只有 FP22 精度（13 位尾数而非 23 位），通过两级 FP32 buffer 累加消除精度损失 [L2:2411.10958]。RTX4090 上达到 481 TOPS，3× 快于 FlashAttention2 [L2:2411.10958]。

第二代增强：FP16 累加器（SageAttention2++）。发现 mma.f16.f8.f8.f16 指令比 FP32 累加器版本快 2×（总计 4× 快于 FP16），通过缩窄 P 和 V 的 FP8 量化范围（$P_r=224$, $V_r=4.5$，满足 $P_r \times V_r \leq 1023.5$）避免 FP16 累加器溢出。Delayed FP32 Buffering（每两次 MMA 结果在 FP16 中累加后再转 FP32）减半类型转换开销。精度完全不变（CosSim 99.97%），速度提升到 FlashAttention2 的 3.9× [L2:2505.21136]。

第三代：FP4 microscaling（SageAttention3）。利用 Blackwell GPU 的 NVFP4 Tensor Core（E2M1 数据类型，1×16 block，E4M3 scale factor），将两个 matmul 都量化到 FP4。核心创新是 two-level quantization——softmax 输出 P 的值域 $[0,1]$ 导致 FP4 scale factor 仅利用 E4M3 格式 28% 的可表示值，通过先 per-token 归一化到 $[0, 448 \times 6]$ 再做 microscaling，可用表示值从 280 增加到 1016 [L2:2505.11594]。RTX5090 上达到 1038 TOPS，5× 快于 FlashAttention2，端到端加速 HunyuanVideo 3× [L2:2505.11594]。同时首次将 INT8 量化扩展到训练反向传播（SageBwd），微调无损但预训练收敛较慢 [L2:2505.11594]。

2.2 稀疏-线性混合 Attention

SLA（Sparse-Linear Attention） 发现 Diffusion Transformer 的注意力权重天然分为三类：少量大权重（高秩，<10%）、大量小权重（极低秩，>90%）、可忽略权重 [L2:2509.24006]。据此提出三路并行：critical 块用 FlashAttention 精确计算，marginal 块用线性注意力 $O(N)$ 近似，negligible 块直接跳过。三种计算融合在单个 GPU kernel 中 [L2:2509.24006]。在 Wan2.1-1.3B 上实现 95% 稀疏度（52.75T→2.74T FLOPs），attention kernel 加速 13.7×，端到端视频生成加速 2.2× [L2:2509.24006]。仅需 2000 步微调（<预训练成本 0.1%）。

2.3 GPU 架构感知的 Kernel 调度

NUMA-aware Attention（Swizzled Head-first Mapping）。AMD MI300X 的 8 XCD chiplet 架构引入显著 NUMA 效应——每 XCD 仅 4MB 私有 L2 cache，FlashAttention2 的 round-robin 调度将同一 head 的 workgroup 打散到不同 XCD，导致 L2 命中率暴跌至 ~1% [L2:2511.02132]。Swizzled Head-first Mapping 将同一 attention head 的所有 workgroup 映射到同一 XCD，以 ~15 行 Triton swizzle 代码实现 L2 命中率 80-97%、最高 50% 性能提升 [L2:2511.02132]。MHA（如 DeepSeek-V3, H=128）收益最大，GQA 因 KV group 数与 XCD 数天然匹配而收益较小 [L2:2511.02132]。

计算-通信并发（ConCCL）。将 all-gather/all-to-all 通信 offload 到 MI300X 的 14 个 SDMA 引擎（DMA 位于 IOD 层、不经过 L1/L2 cache），消除与 GEMM 的 compute interference [L2:2412.14335]。C3（Concurrent Computation and Communication）加速比从 baseline 的 21% 提升到 72%（理想加速比的比例）[L2:2412.14335]。关键发现是 memory-bound GEMM 在减少 CU 时反而加速（更好的 cache locality），而 communication kernel 仅需 32-64 CU（总 304 中的 10-20%）[L2:2412.14335]。

2.4 AMD GPU 编程框架

HipKittens 是首个系统性的 AMD GPU C++ tile-based 编程框架，将 ThunderKittens 的 tile 抽象（typed tiles + PyTorch 风格算子）移植到 AMD CDNA3/CDNA4 [L2:2511.08083]。核心发现：NVIDIA 的 wave specialization（producer-consumer）在 AMD 上仅达峰值 80%——静态寄存器分配使 producer wave 浪费寄存器 [L2:2511.08083]。替代方案是 8-wave ping-pong（每 SIMD 2 wave 交替 compute/memory），仅 48 行代码即可匹配手写汇编的 GEMM 性能 [L2:2511.08083]。在 MI355X 上，BF16 GEMM 达 1610 TFLOPS（64.4% 峰值），FP8 GEMM 达 3327 TFLOPS（66.5% 峰值），GQA backward 比所有基线快 1.8-2.5× [L2:2511.08083]。显式寄存器 pinning 绕过 HIPCC 限制、直接使用 AGPR 作为 MFMA 输入，在 attention backward 上带来 20% 性能提升 [L2:2511.08083]。

每一代量化精度的提升都伴随着硬件绑定的收紧。SageAttention 的 Triton 实现理论上可通过 HIP backend 移植到 AMD，但 SageAttention3 的 FP4 kernel 完全锁定在 Blackwell 架构上 [L2:2505.11594]。AMD 侧的 HipKittens 则证明了 tile 抽象可以跨厂商统一接口，但底层调度必须根本性地重新设计 [L2:2511.08083]。

轴三：算法复杂度 vs 实际加速

SLA 的 95% FLOPs 缩减（理论 20× 加速）最终转化为 13.7× kernel 加速和 2.2× 端到端加速 [L2:2509.24006]，反映了非 attention 计算（线性层等）的 Amdahl 瓶颈。SageAttention3 的 5× kernel 加速在 HunyuanVideo 上转化为 3× 端到端加速 [L2:2505.11594]，在 LLM 训练上仅 1.1-1.15× [L2:2505.11594]。ConCCL 的 DMA offload 在 microbenchmark 上达到理想加速比的 72%，但缺乏端到端训练验证 [L2:2412.14335]。

4. 冲突与争议

INT 量化 vs FP 量化

SageAttention 系列在 attention 中一贯优先选择 INT 格式而非 FP 格式。SageAttention 在 RTX4090 上证明 INT8 比 FP8（E4M3, E5M2）更精确且速度快 2× [L2:2410.02367]。SageBwd（训练）也选择 INT8 而非 FP8（dQ CosSim: 0.9987 vs 0.9880）[L2:2505.11594]。这与业界（FlashAttention3、NVIDIA 官方推荐）以 FP8 为主的路线形成对比。FlashAttention3 的 FP8 在视频生成模型上严重退化（CogVideoX VQA-t: 2.181 vs 原始 70.928）[L2:2411.10958]，而 SageAttention2 几乎无损。然而，FP4（SageAttention3）的引入说明在最低精度端，浮点格式反而成为唯一选择——INT4 TC 仅在 Ada 上存在，而 FP4 TC 在 Blackwell 上原生支持 [L2:2505.11594]。

结论：尚未完全分出胜负。 INT 在中精度（8-bit/4-bit）上有精度优势，但 FP 在极低精度和新硬件支持上更具前景。

NVIDIA wave specialization vs AMD ping-pong

HipKittens 明确证伪了在 AMD 上照搬 NVIDIA wave specialization 的做法——由于静态寄存器分配，producer wave 浪费寄存器，导致 output tile 尺寸受限、算术强度不足，仅达峰值 80% [L2:2511.08083]。替代的 8-wave ping-pong 以 0 个 producer wave 达到了最佳性能（1610 TFLOPS），与 NVIDIA B200 上 TK/CUTLASS 的 1538/1570 TFLOPS 相当 [L2:2511.08083]。这一发现颠覆了 "wave specialization 是高性能 kernel 必选项" 的普遍认知。

硬件透明 vs 软件调度

NVIDIA Blackwell 选择用硬件 coherency 对软件隐藏多 die 的 NUMA 效应，而 AMD MI300X 显式暴露 NUMA、依赖软件侧的 NUMA-aware 调度 [L2:2511.02132]。Swizzled Head-first 在 MI300X 上的 50% 性能提升证明了软件调度的巨大潜力 [L2:2511.02132]，但也暴露了 AMD 生态对 kernel 开发者的更高要求。未来的 Rubin Ultra（4 die）可能重新需要类似优化 [L2:2511.02132]。

5. 未探索角落

  1. Decode 阶段的低比特 attention：SageAttention 全系列 [L2:2410.02367] [L2:2505.11594] 和 SLA [L2:2509.24006] 均聚焦于 prefill/生成场景（长序列 compute-bound），LLM decode（短序列、大 batch、memory-bound）的量化 attention 效果未知。
    1. 低比特 attention 的预训练收敛：SageBwd 在微调上无损但预训练收敛明显慢于 BF16 [L2:2505.11594]，根因（梯度方差累积？量化噪声分布？）尚未解明。
      1. 跨厂商 kernel 统一：HipKittens 和 ThunderKittens 共享 tile 接口但底层实现完全不同 [L2:2511.08083]。真正的跨厂商 kernel 抽象仍未实现——Triton 在 AMD 上的表现被 HipKittens 超出 1.3-3× [L2:2511.08083]。
        1. All-reduce 的 DMA offload：ConCCL 证明了 DMA offload 对 all-gather/all-to-all 的价值，但最核心的 all-reduce（需算术运算）无法由 DMA 执行 [L2:2412.14335]。是否可以通过增强 DMA 引擎的算术能力来解决，取决于硬件架构演进。
          1. 大模型（14B+）上的稀疏 attention：SLA 仅在 1.3B 模型上验证 [L2:2509.24006]，大模型的注意力权重分布是否保持同样的"高秩-低秩"可分解结构尚无定论。
            1. GQA/MQA 的量化 attention：SageAttention 原始论文未显式处理 GQA [L2:2410.02367]，而 GQA 是当前主流 LLM 架构（Llama 3/4）的标配。

              2. 6. 时代定位

              Kernel 类别正处于硬件驱动的量化精度下探 + 架构复杂性上升的双重拐点。

              从量化加速看，每一代 GPU（Ampere→Ada→Hopper→Blackwell）都带来新的低精度 Tensor Core 指令，SageAttention 系列跟进这些硬件能力实现了 2.1×→3×→3.9×→5× 的加速链。但 FP4 已接近表示能力的物理极限（仅 15 个可表示值），下一步的突破可能需要从精度降低转向计算缩减（如 SLA 的稀疏-线性混合）[L2:2509.24006] 或架构级优化（如 NUMA-aware 调度 [L2:2511.02132]、DMA offload [L2:2412.14335]）。

              从硬件架构看，多 chiplet GPU 的崛起打破了"GPU 内存访问是 uniform"的假设，迫使 kernel 开发者理解 per-XCD L2 cache、SDMA 引擎拓扑、NUMA 效应等底层细节 [L2:2511.02132] [L2:2412.14335]。HipKittens 证明了高层 tile 抽象可以封装这些复杂性 [L2:2511.08083]，但跨厂商统一仍是开放问题。

              Kernel 类别与相邻类别的交叉正在加深：量化 attention 与 algorithm（稀疏注意力、线性注意力）的融合产生了 SLA [L2:2509.24006]；DMA offload 与 cluster（通信优化）的交叉产生了 ConCCL [L2:2412.14335]；HipKittens 与 hardware（AMD 架构）的深度绑定产生了首个可用的 AMD 高性能 kernel 框架 [L2:2511.08083]。

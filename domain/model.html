Model 类综述

1. 现状快照

2025-2026 年的模型设计正经历三重范式转型：(1) MoE 架构主导化——1T 级 MoE 配合 MLA/GQA 压缩 KV cache 已成为开源旗舰的标配 [L2:2507.20534]，稀疏度从 DeepSeek-V3 的 32 推到 K2 的 48 [L2:2507.20534]；(2) 全模态端到端——Qwen 系列证明 Thinker-Talker 框架可将文本/音频/视频/图像统一在一个推理链路中 [L2:2509.17765]，视频生成侧 MMDiT 双分支联合去噪成为 AV 同步的最佳实践 [L2:2604.14148]；(3) 注意力机制多元化——线性注意力首次在公平对比下全面超越全注意力 [L2:2510.26692]，深度维度注意力聚合替代固定残差连接 [L2:2603.15031]，而 RoPE 低频维度的 massive values 被揭示为上下文理解的关键功能信号 [L2:2502.01563]。

2. 主线与分支

2.1 MoE 架构与极限稀疏

主线：以 Kimi K2 为代表的 1T 级 MoE 成为开源 agentic 模型的架构基底。K2 使用 384 routed experts top-8 + 1 shared expert，稀疏度 48 在 scaling law 下比稀疏度 8 节省 1.69× FLOPs [L2:2507.20534]。核心注意力机制为 MLA（Multi-head Latent Attention），每 token KV cache 仅 576 floats（kv_lora_rank=512 + qk_rope_head_dim=64），使得 1T 参数模型在 262K context 下仍可单节点部署 [L2:2602.02276]。

K2.5 在不改动主干的前提下接入 MoonViT-3D 视觉塔，通过"早期融合 + 10:90 低视觉比例"联合预训 15T tokens，证明了 1T 级 MoE + 多模态的可行性 [L2:2602.02276]。K2.6 进一步将工程重心完全放在 post-training 和部署上：默认 INT4 compressed-tensors 发布（仅量化 routed experts，attention/shared/head 保留 FP16），将部署门槛从多节点拉到单 8×H200 节点 [L2:kimi-k2-6]，并通过长程 agentic RL（4000+ 工具调用、12-13 小时连续执行）实现 Toolathlon +80%、MCPMark +89% 的跳跃 [L2:kimi-k2-6]。

Qwen3-Omni 的 Thinker 也采用 MoE 设计（128E top-8，d=2048，30B total / 3B active），但 GQA 比例为 7:1（28Q/4KV），KV cache 每 token 仅 1024 参数，在多模态语境下更倾向于用 GQA 而非 MLA 来压缩 KV [L2:2509.17765]。

分支决策：K2 系列选择 MLA + sigmoid noaux_tc 路由 + 384 experts；Qwen3-Omni 选择 GQA + 128 experts。两者在不同规模和任务类型上各有取舍——MLA 的 KV cache 压缩更激进（576 vs 1024 floats/token），但 kernel 实现复杂度更高 [L2:2507.20534]。

2.2 注意力机制革新

线性注意力突破：Kimi Linear 的 KDA（Kimi Delta Attention）把 Gated DeltaNet 的 scalar forget gate 替换为 per-channel diagonal gate，并通过 DPLR 约束（a=b=k）使 chunkwise kernel 比通用 DPLR 快 ~2× [L2:2510.26692]。层级混合设计中每 3 层 KDA 后插 1 层 MLA（NoPE），实测 3:1 比例最优；纯 MLA（0:1）最差，过疏（15:1）退化 [L2:2510.26692]。MLA 层采用 NoPE（无位置编码），位置信息完全由 KDA 层的 data-dependent gate 和 short convolution 提供，使 context 支持拉到 1M 而无 RoPE 外推问题 [L2:2510.26692]。推理侧 decode 吞吐提升 +75%，KV cache 减少 ~75% [L2:2510.26692]。

深度残差革新：Attention Residuals 把标准 PreNorm 的"恒定权重 1 残差累加"替换为"沿深度做 softmax-attention 加权求和"——每层学习一个伪 query 对所有先前层输出做选择性聚合 [L2:2603.15031]。Block-AttnRes 版本训练 overhead < 4%，在 Kimi Linear 48B-A3B 上验证 GPQA +7.5，scaling law 显示相同 loss 下节省 ~1.25× compute [L2:2603.15031]。DenseFormer 的固定标量聚合被证明无效（val loss 1.767 vs baseline 1.766），input-dependent 的 softmax 选择是关键 [L2:2603.15031]。

RoPE 机理揭示：RoPE 位置编码导致 Q/K 低频维度出现 concentrated massive values，这些值对上下文知识理解至关重要——破坏后 GSM8K 从 76.9% 暴跌至 4.0%，而参数知识任务仅温和下降 [L2:2502.01563]。这一发现直接指导量化策略：AWQ/SmoothQuant（保护 massive values）在上下文任务上优于 GPTQ（不保护）[L2:2502.01563]。所有 RoPE 模型均存在此现象，非 RoPE 模型（GPT-2, OPT, Jamba）均不存在 [L2:2502.01563]。

2.3 全模态端到端模型

Thinker-Talker 框架演进：Qwen2.5-Omni 是首个开源 omni-modal 端到端模型，采用 dense Thinker（7B）+ dense Talker（0.6B）+ 单 codebook + DiT vocoder 架构 [L2:2503.20215]。Qwen3-Omni 将其全面升级为六段专门化流水线——AuT (音频编码) → ViT (视觉编码) → Thinker (MoE 28L) → Talker (MoE+Shared 20L) → Code Predictor (5L dense × 15 AR) → Code2Wav (Causal ConvNet)，每段的 hidden size、head 数、expert 配置都针对自身角色精心定制 [L2:2509.17765]。

关键升级包括：Talker 从 dense 改为 MoE+Shared（128E top-6 + sigmoid-gated shared expert）[L2:2509.17765]；语音编解码从单 codebook 升级到 16 RVQ；vocoder 从 DiT 改为 Causal ConvNet 使首包延迟从 ~640ms 降至 234ms [L2:2509.17765]；Thinker-Talker 从紧耦合（hidden state passing）改为解耦（discrete tokens + multimodal features），使 RAG/safety/FC 能介入 text→speech 路径 [L2:2503.20215]。

"无退化"训练策略：Qwen3-Omni 的 S1 阶段"adapter-first / encoder-second"两步分训 + S2 阶段从一开始就混入跨模态数据，是"同 FLOPs 下 omni-base 在 text/vision 均 ≥ 单模态 base"的关键 [L2:2509.17765]。

2.4 视频与音视频生成

DiT + GRPO 路线：LongCat-Video（13.6B DiT）将所有视频生成任务统一为 video continuation（T2V=0 条件帧，I2V=1 帧，VC=多帧），配合 Block Causal Attention + KV caching 实现分钟级长视频 [L2:2510.22200]。训练创新包括为 Flow Matching 设计改进的 GRPO（固定 SDE 时间步解决 credit assignment、loss 重加权消除梯度消失、multi-reward 防 hacking），推理创新包括 coarse-to-fine（480p→720p 实现 10.6× 加速且质量提升）+ 3D Block Sparse Attention（93.75% sparsity，near-lossless）[L2:2510.22200]。VBench 2.0 Commonsense 维度 70.94% 全场最佳（含闭源）[L2:2510.22200]。

原生音视频联合生成：Seedance 2.0 采用 dual-branch MMDiT 架构，视频和音频各有一支 DiT branch 通过 cross-modal joint module 在每步去噪时深度交互 [L2:2604.14148]。在 SeedVideoBench 2.0 所有维度全面领先（T2V Motion Quality 3.75 vs runner-up 3.10），音频维度是最大护城河（AV Sync 3.75 vs runner-up 2.91），Arena.AI 公开盲测 Elo 1450 以 720p 击败 1080p 对手 [L2:2604.14148]。20/22 多模态任务支持，其中 7 个独家 [L2:2604.14148]。

2.5 训练方法论创新

MuonClip：解决 Muon 优化器在大规模 MoE 训练中的 attention logit 爆炸问题。Muon 的 msign 操作产生 equal singular values 的权重更新，导致 Q/K 的 spectral norm 累加增长被 bilinear form 平方放大。QK-clip 在 max logit 超过阈值 τ 时按 √γ rescale Q/K 投影权重，且是 self-deactivating 的——K2 训练中仅 12.7% heads 在前 70k 步触发，30% 训练后完全自然衰减 [L2:2507.20534]。

长程 agentic RL：K2.6 将 RL rollout horizon 从分钟级推到 4-13 小时（4000+ 工具调用），reward 依赖最终性能指标而非单次对错。核心壁垒在于 trajectory-level reward shaping + 超长 rollout 的基础设施（单条 rollout 占 GPU 数小时）[L2:kimi-k2-6]。

3. 核心 trade-off 轴

轴 1: KV cache 压缩 vs 表达力

MLA 将 KV cache 压到 576 floats/token（K2 系列），GQA 7:1 压到 1024 floats/token（Qwen3-Omni Thinker），KDA 层则为固定 128×128 state（Kimi Linear），三者在 decode 吞吐和模型质量之间走不同取舍 [L2:2507.20534] [L2:2509.17765] [L2:2510.26692]。Kimi Linear 的混合方案（20 KDA + 7 MLA 层）在长 context decode 吞吐 +75% 同时保持质量匹配全注意力 baseline [L2:2510.26692]。

轴 2: MoE 稀疏度 vs 通信开销

K2 的 sparsity 48（384/8）比 sparsity 8 节省 1.69× FLOPs，但带来更多 all-to-all communication cost in expert parallelism [L2:2507.20534]。Qwen3-Omni 选择更温和的 128E top-8（稀疏度 16），降低 EP 通信但需更多 active FLOPs [L2:2509.17765]。

轴 3: 首包延迟 vs 语音质量

Qwen2.5-Omni 的 DiT vocoder 需要 4-block 窗口等待 lookahead，首包 ~640ms+ [L2:2503.20215]；Qwen3-Omni 切换到 Causal ConvNet 降至 234ms，论文承认音质略逊但延迟收益远超质量代价 [L2:2509.17765]。

轴 4: 量化精度 vs 部署门槛

K2.6 的 INT4 routed-only 量化（~258 GB 总权重）把 1T MoE 的部署从 16+ GPU 拉到单 8 卡节点，但 blog 未给出 BF16 vs INT4 在长程 4000 步 rollout 上的消融 [L2:kimi-k2-6]。Massive values 研究表明，保护 Q/K 低频维度的量化方法（AWQ, SmoothQuant）在上下文理解任务上显著优于不保护的方法（GPTQ）[L2:2502.01563]。

轴 5: 多模态联合训练 vs 单模态退化

Qwen3-Omni 通过"S1 adapter-first / encoder-second + S2 早期多模态混入"实现同 FLOPs 下 omni ≥ 单模态 [L2:2509.17765]。K2.5 反直觉地发现 10:90 低视觉比例优于常规 50:50 [L2:2602.02276]，且 zero-vision SFT 优于 text+vision SFT [L2:2602.02276]。但这些结论均仅在特定规模上验证，可迁移性未知。

4. 冲突与争议

位置编码之争：K2 系列沿用 RoPE + YaRN 扩展到 262K context [L2:2602.02276]；Kimi Linear 则完全放弃 RoPE，在 MLA 层用 NoPE，位置信息完全靠 KDA 的 data-dependent gate 提供，context 拉到 1M [L2:2510.26692]。两个方案来自同一团队（Moonshot），暗示 RoPE 并非模型的唯一选项，NoPE + 隐式 recency 可能是长 context 的更优路径。但 K2.6 仍沿用 RoPE [L2:kimi-k2-6]，说明 NoPE 路线尚未完全取代 RoPE。

线性 vs 全注意力：Kimi Linear 在 1.4T 公平对比下首次全面超越全注意力 [L2:2510.26692]，但论文承认 pure linear attention 仍不可行——需要 MLA 层兜底 [L2:2510.26692]。同时 massive values 研究揭示 RoPE 低频维度在全注意力模型中承载核心语义功能 [L2:2502.01563]，这与 KDA 的"不需要 RoPE"哲学形成张力。

Thinker-Talker 耦合 vs 解耦：Qwen2.5-Omni 的紧耦合设计（Thinker hidden + sampled token embed 直接传给 Talker）阻止了外部模块介入 [L2:2503.20215]；Qwen3-Omni 改为解耦（discrete tokens + multimodal features），性能反而提升 [L2:2509.17765]。这表明端到端的信息传递不一定优于 modular 接口设计。

视频生成路径分歧：LongCat-Video 走"统一 video continuation + GRPO 后训练"的开源路线 [L2:2510.22200]；Seedance 2.0 走"闭源 dual-branch MMDiT + 字节内部数据生态 + RewardDance/DanceGRPO"的产品路线 [L2:2604.14148]。两者都验证了 GRPO/RL 在视频生成后训练中的有效性，但 Seedance 2.0 的架构和训练细节完全不披露，可复现性为零 [L2:2604.14148]。

5. 未探索角落

  • KDA 在 100B+ 规模的验证：Kimi Linear 仅在 48B 上验证，大规模 dense / 更大 MoE 模型是否仍能保持质量优势未知 [L2:2510.26692]
  • Attention Residuals 在标准 dense 大模型上的验证：目前仅有 MoE Kimi Linear 一例 production 验证 [L2:2603.15031]
  • "无退化"在其他规模的可复现性：Qwen3-Omni 的多模态无退化仅在 30B-A3B 验证，7B 或 100B+ 上是否成立是 open question [L2:2509.17765]
  • Massive values 的主动利用：目前仅用于量化策略指导，如何在训练中主动塑造 massive value 分布以提升特定能力、非 RoPE 位置编码是否有等价功能信号、可否设计 massive-value-aware 注意力变体均未探索 [L2:2502.01563]
  • INT4 量化对长程 agentic rollout 的累积影响：K2.6 未给出 BF16 vs INT4 在 4000 步 rollout 上的消融 [L2:kimi-k2-6]
  • Code Predictor 的 15 套独立 head/embedding 是否必须：Qwen3-Omni 未尝试 tied weights [L2:2509.17765]
  • GQA/MQA 等 Q/K 共享机制如何与 massive values 交互 [L2:2502.01563]
  • 视频生成模型的 weight quantization：LongCat-Video 13.6B 模型未探索 INT8/INT4 量化以降低部署门槛 [L2:2510.22200]

6. 时代定位

Model 类在 2025-2026 处于"架构饱和 + 训练/部署创新主导"的转折点。K2 系列三次迭代（K2→K2.5→K2.6）全部复用同一 backbone，所有创新放在训练管线（MuonClip、长程 RL、PARL agent swarm）和部署优化（INT4、agent orchestration）上 [L2:kimi-k2-6]。这暗示当前 MoE + MLA/GQA 的架构组合已接近工程最优，下一步的杠杆在后训练和系统层面。

但并行地，Kimi Linear 和 Attention Residuals 代表了架构层面的持续探索——线性注意力 + NoPE 的组合 [L2:2510.26692] 和 input-dependent 深度聚合 [L2:2603.15031] 可能成为下一代架构的标配组件。

多模态侧，Qwen3-Omni 的"六段专门化"证明 omni-modal 模型不应是"一个巨大 backbone 吃一切"，而是让每个子模块针对自身角色定制维度 [L2:2509.17765]。视频生成侧，竞争已从"能否生成"转向"多模态任务覆盖广度 + 细粒度评测维度均衡度" [L2:2604.14148]，开源模型（LongCat-Video）在 Commonsense 维度上已可与闭源竞争 [L2:2510.22200]。

与相邻 category 的交叉：model 的 MoE routing、MLA/KDA kernel 设计直接影响 kernel 层优化空间；MuonClip 和 GRPO 变体是 algorithm 层贡献；Thinker-Talker 解耦和 Agent Swarm 编排跨入 agent 层；而 INT4 量化和 KV cache 压缩直接决定 cluster 层的部署密度。
